## 写作原则

- 可以没有讲解：纯文档治理、极小维护、无新增代码语义时，不必硬写“代码讲解”。
- 有讲解就必须像讲工程，而不是列文件：参照 `D:\nodeproj\orderops-node\代码讲解记录\107-production-readiness-summary-v3-v103.md` 的模式。
- 开头必须讲目标、角色和“不是什么”：尤其说明不是新执行入口、不绕过契约、不写不该写的数据、不突破 mini-kv / Node / Java 边界。
- 中段必须按入口、数据结构、核心流程、边界字段和测试覆盖组织。
- 结尾保留一句话总结，说明这轮给后续松耦合融合带来的价值。
