# v104 归档说明

本版只做 managed audit receipt 的结构拆分，不改运行契约。下面四张图对应本轮关键验证。

1. `01-cmake-configure.png`
   - 命令：`cmake -S . -B cmake-build-v104 -G Ninja`
   - 结果：配置成功，生成 `cmake-build-v104`
   - 意义：确认新的拆分源文件和辅助头文件被 CMake 正确接住

2. `02-cmake-build.png`
   - 命令：`cmake --build cmake-build-v104`
   - 结果：静态库、测试和可执行文件全部链接成功
   - 意义：确认拆分后没有出现重复定义、缺符号或头文件依赖问题

3. `03-ctest.png`
   - 命令：`ctest --test-dir cmake-build-v104 --output-on-failure`
   - 结果：`35/35` 通过
   - 意义：确认 managed audit receipts 回归和整套现有测试都没被结构调整打坏

4. `04-runtime-smoke.png`
   - 命令：启动 `minikv_server.exe 6504 127.0.0.1`，再用 `minikv_client.exe 127.0.0.1 6504` 读取 `INFOJSON`、`SMOKEJSON`、`HEALTH`
   - 结果：返回 `0.102.0`、`OK live_keys=0`，服务正常监听并关闭连接
   - 意义：确认真实运行仍保持只读证据边界，没有引入写入或自动启动行为

验证结束后，临时构建目录和临时日志目录都会清理，只保留正式归档截图和说明文件。
