# mini-kv v90 运行归档说明

## 本版范围

本轮推进的是 mini-kv v90：为 Node v223 managed audit external adapter connection readiness review 提供只读的 `managed_audit_external_adapter_non_participation_receipt`。

这份回执只证明 mini-kv 仍是 runtime evidence provider，不是 external adapter storage backend，不读取 managed audit credentials，不执行 external adapter schema migration，不参与真实外部 adapter，也不改变写/admin/WAL/snapshot/restore 行为。

命令：

```powershell
cmake -S . -B cmake-build-v90 -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug
```

意义：确认 CMake project version 已进入 `0.90.0`，并且本机 MinGW/CLion CMake 工具链可生成本版构建目录。

结果：所有目标构建成功，包括 `minikv_server`、`minikv_client`、`minikv_cli` 和 31 个测试目标。

意义：确认 v90 新增回执代码、fixture 断言和 release 测试都能通过编译。

### 03-ctest.png

命令：

```powershell
ctest --test-dir cmake-build-v90 --output-on-failure
```

结果：31/31 tests passed。

意义：确认 `INFOJSON`、`SMOKEJSON`、release manifest、runtime smoke fixture、历史 restore/retention/digest fixtures 和 v90 外部适配器非参与回执共同闭环。

### 04-real-tcp-smoke-client.png

命令：

```powershell
minikv_server 6429 127.0.0.1
minikv_client 127.0.0.1 6429 5000 --connect-retries 10 --retry-delay-ms 100
```

客户端发送：

```text
SMOKEJSON
INFOJSON
STORAGEJSON
HEALTH
GET restore:real-read-token
QUIT
```

结果：真实 TCP smoke 通过，输出包含：

- `managed_audit_external_adapter_non_participation_receipt`
- `receipt_digest="fnv1a64:0dfb07cd2f8de289"`
- `external_adapter_storage_backend=false`
- `participates_in_external_adapter=false`
- `credential_read_allowed=false`
- `migration_execution_allowed=false`
- `managed_audit_write_executed=false`
- `order_authoritative=false`
- `GET restore:real-read-token` 返回 `(nil)`
- `HEALTH` 返回 `OK live_keys=0`

意义：证明真实运行态只读 smoke 能读到 v90 回执，并且没有写入 restore token、managed audit state、external adapter state 或 order authority 信号。
