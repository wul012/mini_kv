# v69 Release Artifact Digest Package 运行调试归档

## 本版范围

本轮先只读查看 `D:\nodeproj\orderops-node\docs\plans\README.md` 和 `D:\nodeproj\orderops-node\docs\plans\v169-post-production-environment-preflight-roadmap.md`。最新计划已完成 Node v167-v169，下一步推荐并行 Java v60 + mini-kv v69；mini-kv v69 的目标是 release artifact digest package。

本版只推进 mini-kv / C++，没有修改 Node 或 Java。核心新增 `fixtures/release/release-artifact-digest-package.json`，记录 binary、WAL、Snapshot、fixture digest placeholder、restore drill command profile 和 operator confirmation 字段。它不是 restore 执行器，不执行 `LOAD`、`COMPACT`、`SETNXEX`，也不读取生产 secret。

同步修改包括：

- `CMakeLists.txt` 项目版本更新到 `0.69.0`，新增 `minikv_release_artifact_digest_package_tests`
- `fixtures/release/release-artifact-digest-package.json` 新增发布产物摘要包
- `fixtures/release/verification-manifest.json` 更新到 v69，加入 package fixture、目标测试和端口 6409 smoke
- `fixtures/readonly/infojson-empty-inline.json` 更新版本字段
- `tests/release_artifact_digest_package_tests.cpp` 新增 package 契约测试
- `tests/release_verification_manifest_tests.cpp` 等历史证据测试同步当前运行版本
- `README.md` 和 `代码讲解记录_生产雏形阶段/125-version-69-release-artifact-digest-package.md` 补齐说明

### 05-runtime-smoke.png

命令：

```powershell
minikv_server 6409 127.0.0.1
minikv_client 127.0.0.1 6409 5000 --connect-retries 10 --retry-delay-ms 100
```

输入命令：

```text
INFOJSON
CHECKJSON LOAD data/release-artifact-drill.snap
CHECKJSON COMPACT
CHECKJSON SETNXEX release:token 30 value
STORAGEJSON
HEALTH
GET release:token
QUIT
```

结果：真实 server + client smoke 通过。`INFOJSON` 返回 `0.69.0`，`CHECKJSON LOAD` 只说明 `store_replace_from_snapshot` 风险，`CHECKJSON COMPACT` 只说明 WAL rewrite 风险，`CHECKJSON SETNXEX` 只说明 token 写风险，`STORAGEJSON` 保持 `order_authoritative=false`，`GET release:token` 返回 `(nil)`。

服务端进程 PID `25024` 已在验证结束后停止。

## 结论

v69 已从“artifact digest compatibility matrix 可被 production preflight 核对”推进到“release artifact digest package 可被 deployment evidence intake gate 消费”。本版仍然保持只读 evidence 边界：不执行恢复、不写 token、不读取 secret、不启动 Node/Java、不触碰生产数据。
