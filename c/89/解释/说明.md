本目录记录 mini-kv v89 的关键命令证据。v89 范围与 Node 最新计划一致：补充 `adapter_shell_non_storage_guard_receipt`，供 Node v221 managed audit local adapter candidate dry-run 前只读核验 mini-kv 不是 adapter shell storage backend。

## 01-cmake-configure.png

命令：

```powershell
$cmake = 'D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\cmake.exe'
$env:PATH = 'D:\CLION\CLion 2025.1.1\bin\mingw\bin;' + $env:PATH
& $cmake -S . -B cmake-build-v89 -G 'MinGW Makefiles' -DCMAKE_BUILD_TYPE=Debug
```

结果：CMake configure/generate 成功，构建目录为 `cmake-build-v89`。

意义：证明 v89 CMake 工程元数据可以被本机 CLion 工具链识别，`project(mini_kv VERSION 0.89.0)` 能生成当前版本构建。

## 04-real-smoke-client.png

命令：

```powershell
.\cmake-build-v89\minikv_server.exe 6429 127.0.0.1
.\cmake-build-v89\minikv_client.exe 127.0.0.1 6429 5000 --connect-retries 10 --retry-delay-ms 100
```

客户端发送：

```text
SMOKEJSON
INFOJSON
STORAGEJSON
HEALTH
GET restore:real-read-token
QUIT
```

结果：真实 TCP 路径返回 `SMOKEJSON`、`INFOJSON`、`STORAGEJSON`、`HEALTH`，并且 `GET restore:real-read-token` 返回 `(nil)`。

意义：证明 v89 回执不是只存在于 fixture 中，而是能从真实服务端读到；同时没有执行 LOAD/COMPACT/SETNXEX/RESTORE，也没有写出 restore token。
