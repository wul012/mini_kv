# v68 Artifact Digest Compatibility Matrix 运行调试归档

## 本版范围

本轮先只读查看 `D:\nodeproj\orderops-node\docs\plans\README.md` 和 `D:\nodeproj\orderops-node\docs\plans\v166-post-rollback-window-roadmap.md`。最新计划已完成 Java v58、mini-kv v67、Node v167，下一步推荐并行 Java v59 + mini-kv v68；mini-kv v68 的目标是 artifact digest compatibility matrix。

本版只推进 mini-kv / C++，没有修改 Node 或 Java。核心新增 `fixtures/release/artifact-digest-compatibility-matrix.json`，记录 binary、WAL、Snapshot、fixture digest/checksum evidence 的 operator-recorded placeholder 和版本兼容矩阵。它不是 restore 执行器，不执行 `LOAD`、`COMPACT`、`SETNXEX`，也不读取生产 secret。

同步修改包括：

- `CMakeLists.txt` 项目版本更新到 `0.68.0`，新增 `minikv_artifact_digest_compatibility_matrix_tests`
- `fixtures/release/artifact-digest-compatibility-matrix.json` 新增 digest compatibility matrix
- `fixtures/release/verification-manifest.json` 更新到 v68，加入 matrix fixture、目标测试和端口 6408 smoke
- `fixtures/readonly/infojson-empty-inline.json` 更新版本字段
- `tests/artifact_digest_compatibility_matrix_tests.cpp` 新增 matrix 契约测试
- `tests/release_verification_manifest_tests.cpp` 等历史证据测试同步当前运行版本
- `README.md` 和 `代码讲解记录_生产雏形阶段/124-version-68-artifact-digest-compatibility-matrix.md` 补齐说明

### 05-runtime-smoke.png

命令：

```powershell
minikv_server 6408 127.0.0.1
minikv_client 127.0.0.1 6408 5000 --connect-retries 10 --retry-delay-ms 100
```

输入命令：

```text
INFOJSON
CHECKJSON LOAD data/digest-matrix.snap
CHECKJSON COMPACT
CHECKJSON SETNXEX digest:token 30 value
STORAGEJSON
HEALTH
GET digest:token
QUIT
```

结果：真实 server + client smoke 通过。`INFOJSON` 返回 `0.68.0`，`CHECKJSON LOAD` 只说明 `store_replace_from_snapshot` 风险，`CHECKJSON COMPACT` 只说明 WAL rewrite 风险，`CHECKJSON SETNXEX` 只说明 token 写风险，`STORAGEJSON` 保持 `order_authoritative=false`，`GET digest:token` 返回 `(nil)`。

服务端进程 PID `19812` 已在验证结束后停止。

## 结论

v68 已从“restore dry-run operator package 可被 preflight 消费”推进到“artifact digest compatibility matrix 可被 production environment preflight 核对”。本版仍然保持只读 evidence 边界：不执行恢复、不写 token、不读取 secret、不启动 Node/Java、不触碰生产数据。
