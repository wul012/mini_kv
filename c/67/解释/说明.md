# v67 Restore Dry-Run Operator Package 运行调试归档

## 本版范围

本轮先只读查看 `D:\nodeproj\orderops-node\docs\plans\README.md` 和 `D:\nodeproj\orderops-node\docs\plans\v166-post-rollback-window-roadmap.md`。计划要求 Java v58 与 mini-kv v67 并行补齐 rollback SQL review / restore dry-run operator package，Node v167 等两边证据完成后再做只读 preflight contract；这个方向合理，本版只推进 mini-kv / C++。

v67 新增 `fixtures/release/restore-dry-run-operator-package.json`，记录 restore target、operator 记录 digest 的占位字段、WAL/Snapshot compatibility、`CHECKJSON` dry-run 命令和暂停条件。它不是新的运行时命令，不执行 `LOAD`、`COMPACT`、`SETNXEX`，也不把 mini-kv 接入 Java 订单一致性链。

同步修改包括：

- `CMakeLists.txt` 项目版本更新到 `0.67.0`，新增 `minikv_restore_dry_run_operator_package_tests`
- `fixtures/release/verification-manifest.json` 更新到 v67，加入 restore dry-run package 与端口 6407 的 smoke 说明
- `fixtures/readonly/infojson-empty-inline.json` 更新版本字段
- `tests/restore_dry_run_operator_package_tests.cpp` 新增 package 契约测试
- `tests/release_verification_manifest_tests.cpp` 等历史证据测试同步当前运行版本
- `README.md` 和 `代码讲解记录_生产雏形阶段/123-version-67-restore-dry-run-operator-package.md` 补齐说明
- `AGENTS.md` 新增上下文压缩风险规则

### 05-runtime-smoke.png

命令：

```powershell
minikv_server 6407 127.0.0.1
minikv_client 127.0.0.1 6407 5000 --connect-retries 10 --retry-delay-ms 100
```

输入命令：

```text
INFOJSON
CHECKJSON LOAD data/dry-run-restore.snap
CHECKJSON COMPACT
CHECKJSON SETNXEX dryrun:token 30 value
STORAGEJSON
HEALTH
GET dryrun:token
QUIT
```

结果：真实 server + client smoke 通过。`INFOJSON` 返回 `0.67.0`，`CHECKJSON LOAD` 只说明 `store_replace_from_snapshot` 风险，`CHECKJSON COMPACT` 只说明 WAL rewrite 风险，`CHECKJSON SETNXEX` 只说明写风险，`STORAGEJSON` 保持 `order_authoritative=false`，`GET dryrun:token` 返回 `(nil)`。

服务端进程 PID `22824` 已在验证结束后停止。

## 结论

v67 已从“restore compatibility handoff 可人工核对”推进到“restore dry-run operator package 可被 Node v167 preflight 消费”。本版仍然保持只读证据边界：不执行恢复、不写 token、不启动 Node/Java、不触碰生产数据。
