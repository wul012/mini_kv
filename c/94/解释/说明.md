# mini-kv v94 命令归档说明

## 本轮计划来源

本轮推进前已只读查看 Node 计划目录：

- `D:\nodeproj\orderops-node\docs\plans\README.md`
- `D:\nodeproj\orderops-node\docs\plans\v223-post-external-adapter-readiness-roadmap.md`

最新计划中 mini-kv v91 质量门已完成；本轮 v94 是在用户明确同意继续拆分后执行的 C++ 侧质量重构。范围合理：只拆 `COMMANDS` / `COMMANDSJSON` / `EXPLAINJSON` / `CHECKJSON` 的命令契约格式化逻辑，以及 `KEYS` / `INFO` / `STATS` / `HEALTH` / `SMOKEJSON` / `STORAGEJSON` / `WALINFO` 的运行响应格式化逻辑，不修改 Node 或 Java，不新增运行时执行能力。

## 本版目标

v94 把命令目录、命令风险解释、只读执行契约格式化从 `src/command.cpp` 拆到：

- `include/minikv/command_contracts.hpp`
- `src/command_contracts.cpp`
- `include/minikv/command_response_formatters.hpp`
- `src/command_response_formatters.cpp`

`src/command.cpp` 继续负责 dispatch、store/WAL/snapshot 执行路径。拆分后仍保持：

- `CHECKJSON` / `EXPLAINJSON` 字段契约不扩张
- `KEYS` / `STATS` / `HEALTH` / `INFOJSON` / `SMOKEJSON` / `STORAGEJSON` / `WALINFO` 响应契约不扩张
- command digest 算法不改变
- write/admin/WAL/snapshot/restore 处理路径不改变
- mini-kv 仍不是 Java order authority

### 04-tcp-smoke.png

真实运行命令：启动 `minikv_server 6429 127.0.0.1`，通过 `minikv_client 127.0.0.1 6429 5000 --connect-retries 10 --retry-delay-ms 100` 发送：

```text
SMOKEJSON
INFOJSON
STORAGEJSON
HEALTH
GET restore:real-read-token
QUIT
```

结果：SMOKEJSON/INFOJSON/STORAGEJSON/HEALTH 均返回只读运行证据，`GET restore:real-read-token` 返回 `(nil)`，未执行 LOAD、COMPACT、SETNXEX、RESTORE 或 managed audit 写入。验证后服务端进程已停止。
