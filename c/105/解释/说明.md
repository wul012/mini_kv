# v105 归档说明

本版只做 WAL 写路径质量优化：用 command/compaction gate 替代单个 WAL command mutex，保持 append-before-mutation 顺序，同时把 auto/manual compaction 从普通写命令关键区间中分离出来。

1. `01-cmake-configure.png`
   - 命令：`cmake -S . -B cmake-build-v105 -G Ninja`
   - 结果：配置成功
   - 意义：确认新增 gate 代码仍能被当前 CMake/Ninja/MinGW 工具链识别

2. `02-cmake-build.png`
   - 命令：`cmake --build cmake-build-v105`
   - 结果：库、测试和可执行文件全部构建成功
   - 意义：确认 `condition_variable` gate、WAL tests 和现有目标链接正常

3. `03-ctest.png`
   - 命令：`ctest --test-dir cmake-build-v105 --output-on-failure`
   - 结果：`35/35` 通过
   - 意义：确认 WAL、command、TCP、runtime evidence、restore/release fixtures 等现有行为没有回归；新增并发 auto compact replay 回归也通过

4. `04-runtime-smoke.png`
   - 命令：启动 `minikv_server.exe 6505 127.0.0.1 <wal> --auto-compact-wal --wal-compact-min-records 2 --wal-compact-record-ratio 1 --wal-compact-min-bytes 1048576`，再用 `minikv_client.exe` 执行 `SET hot value1`、`SET hot value2`、`WALINFO`、`HEALTH`
   - 结果：`WALINFO` 返回 `wal_records=1`、`auto_compactions=1`、`records_removed=1`，`HEALTH` 返回 `OK live_keys=1`
   - 意义：确认真实运行中 auto compact 仍会触发，WAL 统计正确，服务健康输出保持稳定

验证结束后，临时构建目录和临时日志目录都会清理，只保留正式归档截图和说明文件。
