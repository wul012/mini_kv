# v83 运行调试归档说明

本轮版本目标是给 `INFOJSON` 和 `SMOKEJSON` 补充只读 `binary_provenance` 提示，供 Node v208 在 managed audit persistence boundary candidate 前核对运行二进制、fixture 和 release evidence 路径是否对齐。它不是生产二进制认证，不执行写命令，不执行 `LOAD`、`COMPACT`、`SETNXEX`、`RESTORE`，也不让 mini-kv 进入 Java 订单一致性链路。
