# v81 运行调试归档说明

本轮版本目标是给 `INFOJSON` 和 `SMOKEJSON` 补充只读 `artifact_retention` 证据，供 Node v203 后续检查跨项目 CI artifact retention gate。它不是 GitHub artifact 上传配置，不读取 secret，不打开生产窗口，也不执行 `LOAD`、`COMPACT`、`SETNXEX`、`RESTORE`。

## 截图说明

1. `图片/01-cmake-configure.png`
   - 命令：`cmake -S . -B cmake-build-v81 -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug`
   - 结果：CMake 配置成功，生成 `cmake-build-v81`。
   - 意义：确认 v81 在本机 CLion MinGW/CMake 工具链下可以生成构建文件。

2. `图片/02-cmake-build.png`
   - 命令：`cmake --build cmake-build-v81 --parallel 2`
   - 结果：库、CLI、server、client、benchmark 和全部测试目标构建成功。
   - 意义：确认新增 `artifact_retention` 输出不会破坏 C++ 编译和可执行目标生成。

3. `图片/03-ctest.png`
   - 命令：`ctest --test-dir cmake-build-v81 --output-on-failure`
   - 结果：31/31 测试通过。
   - 意义：确认 `INFOJSON`、`SMOKEJSON`、release manifest、runtime smoke evidence、历史 release fixture 版本联动和边界断言全部通过。

4. `图片/04-real-smoke-client.png`
   - 命令：启动 `minikv_server 6419 127.0.0.1` 后，用 bundled client 执行 `SMOKEJSON`、`INFOJSON`、`STORAGEJSON`、`HEALTH`、`GET restore:real-read-token`、`QUIT`。
   - 结果：真实 TCP 返回 `version="0.81.0"`、`artifact_retention.artifact_root="c/"`、`artifact_retention.artifact_path_hint="c/81/"`、`retention_days=30`、`github_artifact_upload_attempted=false`、`production_window_allowed=false`，并保持 `GET restore:real-read-token` 为 `(nil)`。
   - 意义：确认 Node v203 未来可以从真实运行中的 mini-kv 只读读取 retention gate 所需字段。

5. `图片/05-real-smoke-summary.png`
   - 命令：smoke 脚本检查输出字段并停止 server。
   - 结果：`client_checks=passed`，`server_stopped=True`。
   - 意义：确认本轮启动的后台服务已清理，没有遗留运行进程。
