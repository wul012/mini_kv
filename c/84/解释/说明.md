# v84 运行调试归档说明

本轮按 `D:\nodeproj\orderops-node\docs\plans\v209-managed-audit-dry-run-roadmap.md` 推进 mini-kv v84。版本目标是补充 `retention_provenance_check` 只读证据，让 Node v211 在生成 managed audit identity approval provenance dry-run packet 前，可以核对运行时证据留存路径和二进制来源 digest 是否一致。

本版不执行 `LOAD`、`COMPACT`、`SETNXEX`、`RESTORE`，不写 managed audit，不连接 Java 事务链，也不让 mini-kv 成为订单权威存储。
