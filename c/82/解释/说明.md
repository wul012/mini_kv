# v82 运行调试归档说明

本轮版本目标是给 `SMOKEJSON` 补充只读 `live_read_session` 提示，供 Node v205 在三项目 real-read runtime smoke execution packet 前核对 session echo、server uptime bucket 和 read command list digest。它不是自动启动许可，不执行写命令，不执行 `LOAD`、`COMPACT`、`SETNXEX`、`RESTORE`，也不让 mini-kv 进入 Java 订单一致性链路。

## 截图说明

1. `图片/01-cmake-configure.png`
   - 命令：`cmake -S . -B cmake-build-v82 -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug`
   - 结果：CMake 配置成功，生成 `cmake-build-v82`。
   - 意义：确认 v82 在本机 CLion MinGW/CMake 工具链下可以生成构建文件。

2. `图片/02-cmake-build.png`
   - 命令：`cmake --build cmake-build-v82 --parallel 2`
   - 结果：库、CLI、server、client、benchmark 和全部测试目标构建成功。
   - 意义：确认新增 `live_read_session` 输出不会破坏 C++ 编译和可执行目标生成。

3. `图片/03-ctest.png`
   - 命令：`ctest --test-dir cmake-build-v82 --output-on-failure`
   - 结果：31/31 测试通过。
   - 意义：确认 `SMOKEJSON`、release manifest、runtime smoke evidence、历史 release fixture 版本联动和只读边界断言全部通过。

4. `图片/04-real-smoke-client.png`
   - 命令：启动 `minikv_server 6420 127.0.0.1` 后，用 bundled client 执行 `SMOKEJSON`、`INFOJSON`、`STORAGEJSON`、`HEALTH`、`GET restore:real-read-token`、`QUIT`。
   - 结果：真实 TCP 返回 `version="0.82.0"`、`live_read_session.session_id_echo="mini-kv-live-read-v82"`、`server_uptime_bucket="lt_60s"`、`read_command_list_digest="fnv1a64:5bef33f2fbe65cc5"`、`write_commands_allowed=false`、`auto_start_allowed=false`，并保持 `GET restore:real-read-token` 为 `(nil)`。
   - 意义：确认 Node v205 未来可以从真实运行中的 mini-kv 只读读取 live-read session hint。

5. `图片/05-real-smoke-summary.png`
   - 命令：smoke 脚本检查输出字段并停止 server。
   - 结果：`version 0.82.0`、`live read session object`、`session echo`、`read command digest`、`write commands forbidden`、`auto start forbidden`、`health ok`、`restore token nil` 全部 PASS。
   - 意义：确认本轮启动的后台服务已清理，没有遗留运行进程。
