### 05-runtime-smoke.png

命令：

```powershell
.\cmake-build-v66\minikv_server.exe 6406 127.0.0.1
.\cmake-build-v66\minikv_client.exe 127.0.0.1 6406 5000 --connect-retries 10 --retry-delay-ms 100
```

客户端输入：

```text
INFOJSON
CHECKJSON LOAD data/restore.snap
CHECKJSON COMPACT
CHECKJSON SETNXEX restore:token 30 value
STORAGEJSON
HEALTH
GET restore:token
QUIT
```

结果要点：

```text
INFOJSON -> version=0.66.0
CHECKJSON LOAD -> side_effects=["store_replace_from_snapshot"]
CHECKJSON COMPACT -> side_effects=["wal_rewrite_when_enabled"]
CHECKJSON SETNXEX -> write_command=true
STORAGEJSON -> order_authoritative=false
HEALTH -> live_keys=0
GET restore:token -> (nil)
smoke assertions passed
```

意义：真实 TCP smoke 只解释 restore 敏感命令的风险，不执行 `LOAD`、`COMPACT` 或 `SETNXEX`。`GET restore:token -> (nil)` 证明 token 没被写入。
