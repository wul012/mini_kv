# OSFS 课程设计通俗教程

这套教程面向第一次阅读 mini-kv OSFS 的学生。旧的 v1629-v1636 代码讲解仍然保留，因为它们记录了每个版本当时解决了什么问题；这里不再按版本讲，而是按“磁盘上究竟发生了什么”来组织。读者不需要先懂 mini-kv 的 KV、WAL 或 TCP 主线。

## 怎么读

只剩十分钟时，读 [00-十分钟看懂OSFS.md](00-十分钟看懂OSFS.md)。准备答辩时，建议按 00、01、02、03、04、05、06、07 的顺序读。要跟代码调试，则先读 01，再按其中的源码路径跳转。

| 文档 | 回答的问题 | 最重要的可观察证据 |
|---|---|---|
| [00-十分钟看懂OSFS](00-十分钟看懂OSFS.md) | 这个项目到底做了什么 | 一组命令如何变成磁盘镜像中的结构 |
| [01-CREATE到WRITE完整流程](01-CREATE到WRITE完整流程.md) | 创建和写入经过哪些代码 | `OK created`、fd、偏移、inode、位图 |
| [02-磁盘布局与inode](02-磁盘布局与inode.md) | `.img` 文件里每一段放什么 | 512 字节块、双位图、96 字节 inode |

| [04-文件描述符与区间读写](04-文件描述符与区间读写.md) | fd、SEEK、TELL 怎么工作 | 独立读写偏移和跨块区间 I/O |

## 证据强度

教程里的内容按五级证据区分。概念图只能帮助理解；源码路径说明机制确实存在；自动测试会在行为回退时失败；真实 CLI 运行证明用户能操作；重开镜像和故障注入则用来排除“只在内存里演示”和“FSCK 永远输出 OK”这两种质疑。答辩时不要拿一张截图替代测试，也不要拿测试名称替代对机制的解释。
