# OSFS v1636 课堂演示步骤

## 一、准备

使用新镜像运行，避免旧 v1/v2 镜像影响判断：

```powershell
.\cmake-build-osfs\minikv_osfs.exe --disk osfs-course.img --format --blocks 160 --script .\课程设计交付\v1636-osfs-final\osfs-demo-script.txt
```

开始前建议先打开报告图 2 和图 3：图 2 说明磁盘中超级块、双位图、用户表、inode 表和数据区的位置；图 3 说明 8 个直接块用完后，inode 通过 `indirect_block` 找到后续数据块号。

## 二、按输出讲解

1. `WHOAMI` 返回 `not logged in`，随后 CREATE 被 `ERR login required` 阻断，说明文件命令不能绕过认证。
2. `LOGIN alice wrong` 失败，`LOGIN alice alice123` 成功。用户表来自磁盘 Block 3，不是运行时写死。
3. Alice 创建 `report`，两次 WRITE 后 `TELL` 显示 write offset 为 13；随后分段 READ 得到 `course`、`-`、`design`，证明 fd 有独立读位移。
4. `CHMOD report 0600` 后，DIR 显示 `report ... 0600 ... 13`，说明目录项、保护码、owner 和长度都可见。
5. Alice 创建 `bigfile` 并写入 4291 字节，超过 8 个 512 字节直接块。`SEEK 6 4088` 后读取 64 字节，输出 `DIRECT8|INDIRECT-BEGIN|...`，这是本版最重要的间接块演示点。
6. `FSCK` 输出 `FSCK OK checks=6`，随后列出 superblock、inode bitmap、block bitmap、file sizes、user table、directories 的 OK 明细。
7. Bob 登录后自己的 DIR 为空；`OPEN alice/report r` 被权限拒绝；Bob 执行 `USERADD carol carol123` 被 `ERR USERADD requires root` 拒绝。
8. root 登录后执行 `USERADD carol carol123` 和 `PASSWD carol carol456`，再 `DIR alice` 审查 Alice 的 UFD。
9. Carol 使用新密码登录后 DIR 为空，创建 `carol-note` 后只出现在 Carol 自己的 UFD；最后再跑一次 FSCK，证明用户管理后的磁盘结构仍一致。

## 三、重新打开证明持久化

课堂可在脚本跑完后去掉 `--format`，重新打开同一镜像：

```powershell
.\cmake-build-osfs\minikv_osfs.exe --disk osfs-course.img
```

登录 Carol 后执行 `DIR`，登录 Alice 后执行 `DIR` 和 `OPEN bigfile r`，仍可看到用户、目录和文件内容。这样能说明用户表、MFD/UFD、inode、直接块、间接块和数据块都来自镜像。

## 四、答辩中主动说明的边界

- 这是 MFD/UFD 两级目录，不是任意深度目录树。
- 用户表固定容量，密码哈希是教学简化，不等同生产安全方案。
- 支持一级间接块，不支持二级/三级间接块。
- FSCK 是只读一致性检查，不自动修复磁盘。
- 没有崩溃日志和多进程并发。
- OSFS 是独立课设入口，不复用 KV/WAL/TCP 主线伪装文件系统。
