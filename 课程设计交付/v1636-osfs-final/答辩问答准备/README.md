# OSFS v1636 答辩问答准备

这组材料是给课程设计答辩用的，不是报告正文的重复版。写法按“老师可能怎么追问、你应该怎么把机制讲清楚”来组织，重点放在磁盘布局、MFD/UFD、inode、bitmap、文件描述符位移、一级间接块、FSCK、用户与权限、Linux 运行证据这些能现场解释的内容。

## 阅读顺序

| 文件 | 用途 |
|---|---|
| `00-答辩速记与图解.md` | 先看这一篇，里面有磁盘布局、CREATE+WRITE、FSCK 三张可手画的图 |
| `01-项目定位与价值-QA.md` | 回答“这个项目是什么、为什么适合操作系统课设、输入输出是什么” |
| `02-磁盘布局与目录结构-QA.md` | 回答 superblock、bitmap、user table、inode、MFD/UFD 和持久化问题 |
| `03-命令流程与读写机制-QA.md` | 回答 CREATE、OPEN、READ、WRITE、SEEK、TELL、间接块等运行流程 |
| `04-权限用户与FSCK-QA.md` | 回答登录、权限、USERADD、PASSWD、FSCK 检查点与坏盘发现 |
| `05-测试交付与边界-QA.md` | 回答测试、CI、Linux 证据、报告材料、未做项和老师常见反问 |

共 60 个 Q&A。建议不要逐字背，先背三条主线：

1. 这个课设的输入是命令和二进制磁盘镜像，输出是目录、文件内容、权限状态、FSCK 报告和可复现证据。
2. 真正落盘的是 superblock、双 bitmap、user table、inode table、MFD/UFD 目录块和文件数据块，不是内存里临时拼出来的演示。
3. 已实现的机制要大胆讲，边界也要主动说：一级间接块和只读 FSCK 已经是真实现；EXT2 group descriptor、自动修复、多进程并发、二级/三级间接块是教学边界。
