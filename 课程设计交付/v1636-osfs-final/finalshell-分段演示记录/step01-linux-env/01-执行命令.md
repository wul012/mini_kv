# Step 01：Linux 环境、项目版本、工具链

用途：确认本次演示不是在 Windows 本机模拟，而是在远程 Ubuntu Linux 上，对指定版本的 `mini-kv` 项目进行真实编译/运行前检查。

在 FinalShell 中粘贴执行：

```bash
clear

cd /home/azureuser/mini-kv

echo
echo "========== [1] Linux host =========="
echo
hostname

echo
uname -srmo

echo
head -4 /etc/os-release

echo
echo "========== [2] Project =========="
echo
pwd

echo
git branch --show-current

echo
git rev-parse --short HEAD

echo
echo "========== [3] Toolchain =========="
echo
cmake --version | head -1

echo
ninja --version

echo
g++ --version | head -1

echo
echo "========== Step 1 done =========="
```

预期观察点：

- 能看到 Ubuntu/Linux 内核信息。
- 当前目录应为 `/home/azureuser/mini-kv`。
- Git 提交号应能固定本次演示版本。
- CMake、Ninja、G++ 均能输出版本号。
