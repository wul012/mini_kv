# Step 09：执行结果与解释

状态：待 FinalShell 实跑。

本文件不预填伪造终端输出。完成后至少应原样保存四组信号：

```text
name=corrupt-me ... physical=<真实块号> ... size=14
FSCK OK checks=6
cleared block bitmap bit for block <同一块号> at byte offset <偏移>
FSCK ERROR checks=6
ERROR block bitmap: block bitmap mismatch at block <同一块号>
```

这一步的机理是：文件 inode 仍引用数据块，数据也仍在镜像中，只是 block bitmap 谎称该块空闲。FSCK 从 inode 可达关系重建期望位图，发现磁盘位图与期望不一致，因此报告错误。它不会自动把位图改回去。

验收失败条件：破坏前镜像已经不干净、块号来自硬编码而非 STAT、改错了 inode bitmap 或数据内容、破坏后仍输出 OK、运行时重新带 `--format` 把损坏镜像覆盖，均需重新执行。
