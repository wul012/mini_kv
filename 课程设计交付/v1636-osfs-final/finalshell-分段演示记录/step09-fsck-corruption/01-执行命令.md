# Step 09：破坏 block bitmap 并运行 FSCK

状态：已在 FinalShell 实跑通过；本命令保留用于再次复现。该步骤会故意修改 `/tmp/osfs-step09.img`，只用于一次性演示镜像。

```bash
cd /home/azureuser/mini-kv
rm -f /tmp/osfs-step09.img /tmp/osfs-step09-setup.txt /tmp/osfs-step09-check.txt \
      /tmp/osfs-step09-before.log /tmp/osfs-step09-after.log

cat > /tmp/osfs-step09-setup.txt <<'EOF'
LOGIN alice alice123
CREATE corrupt-me
OPEN corrupt-me w
WRITE 3 integrity-data
CLOSE 3
STAT corrupt-me
FSCK
QUIT
EOF

./build-osfs-linux/minikv_osfs \
  --disk /tmp/osfs-step09.img \
  --format --blocks 96 \
  --script /tmp/osfs-step09-setup.txt \
  > /tmp/osfs-step09-before.log

echo
echo "[1] Before corruption"
grep -E '^(name=corrupt-me|FSCK OK)' /tmp/osfs-step09-before.log

DATA_BLOCK=$(sed -n 's/.*physical=\([0-9][0-9]*\).*/\1/p' /tmp/osfs-step09-before.log | head -n 1)
test -n "$DATA_BLOCK"
echo "corrupting_data_block=$DATA_BLOCK"

python3 - /tmp/osfs-step09.img "$DATA_BLOCK" <<'PY'
from pathlib import Path
import sys

path = Path(sys.argv[1])
block = int(sys.argv[2])
image = bytearray(path.read_bytes())
block_size = 512
block_bitmap_block = 1
offset = block_bitmap_block * block_size + block // 8
image[offset] &= ~(1 << (block % 8))
path.write_bytes(image)
print(f"cleared block bitmap bit for block {block} at byte offset {offset}")
PY

cat > /tmp/osfs-step09-check.txt <<'EOF'
LOGIN root root
FSCK
QUIT
EOF

./build-osfs-linux/minikv_osfs \
  --disk /tmp/osfs-step09.img \
  --script /tmp/osfs-step09-check.txt \
  > /tmp/osfs-step09-after.log

echo
echo "[2] After corruption"
grep -E '^(FSCK ERROR|ERROR block bitmap|BYE)' /tmp/osfs-step09-after.log

rm -f /tmp/osfs-step09.img /tmp/osfs-step09-setup.txt /tmp/osfs-step09-check.txt \
      /tmp/osfs-step09-before.log /tmp/osfs-step09-after.log

echo
echo "========== Step 9 done =========="
```

观察点：破坏前必须是 `FSCK OK`；脚本从真实 STAT 输出解析物理块号，不依赖写死块号；清掉对应 block bitmap 位后，再打开同一镜像必须得到 `FSCK ERROR` 和 `block bitmap mismatch`。
