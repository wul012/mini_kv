# Step 06：fd 偏移与持久化

用途：演示两个关键点：文件描述符维护读写偏移；退出程序后重新打开磁盘镜像，文件内容仍然存在。

在 FinalShell 中粘贴执行：

```bash
clear

cd /home/azureuser/mini-kv

echo "========== Step 6: fd offset and persistence =========="

rm -f /tmp/osfs-step06.img /tmp/osfs-step06-a.txt /tmp/osfs-step06-b.txt

cat > /tmp/osfs-step06-a.txt <<'EOF'
LOGIN alice alice123
CREATE note
OPEN note rw
WRITE 3 HELLO
TELL 3
WRITE 3 -LINUX
TELL 3
SEEK 3 0
READ 3 20
CLOSE 3
QUIT
EOF

echo
echo "[1] fd offset in one run"
./build-osfs-linux/minikv_osfs --disk /tmp/osfs-step06.img --format --script /tmp/osfs-step06-a.txt \
  | sed -n '/OPEN note/,/BYE/p'

cat > /tmp/osfs-step06-b.txt <<'EOF'
LOGIN alice alice123
OPEN note r
READ 3 20
FSCK
QUIT
EOF

echo
echo "[2] reopen same disk image"
./build-osfs-linux/minikv_osfs --disk /tmp/osfs-step06.img --script /tmp/osfs-step06-b.txt \
  | grep -E 'osfs>|OK login|OK fd|HELLO|FSCK OK|BYE'

rm -f /tmp/osfs-step06.img /tmp/osfs-step06-a.txt /tmp/osfs-step06-b.txt

echo
echo "========== Step 6 done =========="
```

预期观察点：

- 第一次 `WRITE 3 HELLO` 后应显示 `offset=5`。
- Alice 是格式化时的预置账号，可以直接登录。
- 第二次 `WRITE 3 -LINUX` 后应显示 `offset=11`。
- `TELL 3` 应显示 fd 的读写偏移。
- `SEEK 3 0` 后 `READ 3 20` 应读出 `HELLO-LINUX`。
- 第二次不加 `--format` 重新打开同一个磁盘镜像，仍能读出 `HELLO-LINUX`。
- `FSCK OK checks=6` 表示磁盘结构检查通过。
