# Step 07：删除、DIR 字段与写权限拒绝

用途：补齐前 6 段没有显式覆盖的点：`DELETE` 删除文件、`DIR` 表头字段含义、以及“写权限”被拒绝。

在 FinalShell 中粘贴执行：

```bash
clear

cd /home/azureuser/mini-kv

echo "========== Step 7: delete, DIR fields, write denial =========="

rm -f /tmp/osfs-step07.img /tmp/osfs-step07.txt

cat > /tmp/osfs-step07.txt <<'EOF'
LOGIN root root
USERADD deluser del123
USERADD other other123
LOGIN deluser del123
CREATE delete-me
OPEN delete-me rw
WRITE 3 DELETE-ME
CLOSE 3
DIR
DELETE delete-me
DIR
OPEN delete-me r
CREATE secret
CHMOD secret 0600
DIR
LOGIN other other123
OPEN deluser/secret w
OPEN deluser/secret rw
FSCK
QUIT
EOF

./build-osfs-linux/minikv_osfs --disk /tmp/osfs-step07.img --format --script /tmp/osfs-step07.txt \
  | grep -E 'osfs>|OK login|OK created|OK fd|OK wrote|OK closed|name inode physical mode owner size|delete-me|OK deleted|empty|ERR|secret|permission denied|FSCK OK|BYE'

rm -f /tmp/osfs-step07.img /tmp/osfs-step07.txt

echo
echo "========== Step 7 done =========="
```

预期观察点：

- `DIR` 会显示表头：`name inode physical mode owner size`。
- 写入 `DELETE-ME` 后，`DIR` 中应出现 `delete-me ... 0644 ... 9`，其中 `physical` 是物理块位置，`mode` 是保护码，`size` 是文件长度。
- `DELETE delete-me` 后应出现 `OK deleted delete-me`。
- 删除后再次 `DIR` 应显示 `(empty)`。
- 再次 `OPEN delete-me r` 应出现 `ERR file not found`。
- `secret` 设置为 `0600` 后，另一个用户以 `w` 和 `rw` 打开都应出现 `ERR permission denied`。
- 最后 `FSCK OK checks=6`，说明删除和权限测试后文件系统结构仍一致。
