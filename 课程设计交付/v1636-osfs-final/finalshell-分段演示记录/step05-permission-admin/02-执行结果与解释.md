## 关键解释

### 1. Alice 创建私有文件

```text
osfs> CREATE secret
OK created secret
osfs> CHMOD secret 0600
OK chmod secret 0600
osfs> DIR
secret 4 0 0600 1000 0
```

Alice 创建了文件 `secret`，并把权限设置为 `0600`。`0600` 的含义是：文件所有者可以读写，其他用户没有读写权限。目录输出中的 `1000` 是 Alice 的 uid，说明这个文件归 Alice 所有。

### 2. Bob 访问 Alice 私有文件被拒绝

```text
osfs> LOGIN bob bob123
OK login bob uid=1001
osfs> OPEN alice/secret r
ERR permission denied
```

Bob 登录后尝试以只读方式打开 `alice/secret`，系统返回 `ERR permission denied`。这说明 OSFS 的权限位不只是显示在目录里，而是在打开文件时参与了实际权限判断。

### 3. 普通用户不能新增账号

```text
osfs> USERADD carol carol123
ERR USERADD requires root
```

Bob 尝试创建新用户 Carol，被系统拒绝。这里证明用户管理操作受 root 身份限制，普通用户不能直接改用户表。

### 4. root 可以创建用户，Carol 可以登录

```text
osfs> LOGIN root root
OK login root uid=0
osfs> USERADD carol carol123
osfs> LOGIN carol carol123
OK login carol uid=1002
osfs> WHOAMI
carol uid=1002
```

切回 root 后再创建 Carol，随后 Carol 能正常登录，并通过 `WHOAMI` 显示 `carol uid=1002`。这说明用户表、登录和身份切换都在 OSFS 中真实生效。

## 本段结论

第 5 段证明：OSFS 已经实现了基本权限检查和 root 管理边界。普通用户不能读取别人没有授权的文件，也不能随意创建新账号；root 则可以进行用户管理。
