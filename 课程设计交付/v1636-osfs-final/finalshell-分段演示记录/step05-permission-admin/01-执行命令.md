# Step 05：权限拒绝与 root 管理限制

用途：演示两个关键点：普通用户不能读别人无权限文件；普通用户不能执行 `USERADD`，必须由 root 管理用户。

在 FinalShell 中粘贴执行：

```bash
clear

cd /home/azureuser/mini-kv

echo "========== Step 5: permission and admin gate =========="

rm -f /tmp/osfs-step05.img /tmp/osfs-step05.txt

cat > /tmp/osfs-step05.txt <<'EOF'
LOGIN root root
LOGIN alice alice123
CREATE secret
CHMOD secret 0600
DIR
LOGIN bob bob123
OPEN alice/secret r
USERADD carol carol123
LOGIN root root
USERADD carol carol123
LOGIN carol carol123
WHOAMI
QUIT
EOF

./build-osfs-linux/minikv_osfs --disk /tmp/osfs-step05.img --format --script /tmp/osfs-step05.txt \
  | grep -E 'osfs>|OK login|OK useradd|OK created|OK chmod|secret|permission denied|USERADD requires root|carol uid|BYE'

rm -f /tmp/osfs-step05.img /tmp/osfs-step05.txt

echo
echo "========== Step 5 done =========="
```

预期观察点：

- Alice 创建 `secret` 并设置为 `0600`。
- Alice 和 Bob 是格式化时的预置账号，不需要重复创建。
- Bob 执行 `OPEN alice/secret r` 时应出现 `ERR permission denied`。
- Bob 执行 `USERADD carol carol123` 时应出现 `ERR USERADD requires root`。
- root 执行 `USERADD carol carol123` 后，Carol 可以正常登录。
