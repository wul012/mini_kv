# Step 03：OSFS 自动测试验证

用途：只运行本次课设 OSFS 相关测试，避免全量 `ctest` 把仓库里其它历史测试一起跑出来。

在 FinalShell 中粘贴执行：

```bash
clear

cd /home/azureuser/mini-kv

echo "========== Step 3: OSFS tests =========="

ctest --test-dir build-osfs-linux -R '^osfs_' --output-on-failure

echo
echo "========== Step 3 done =========="
```

预期观察点：

- 当前提交应运行 3 个测试：
  - `osfs_tests`
  - `osfs_resilience_tests`
  - `osfs_cli_smoke`
- 末尾应出现：

```text
100% tests passed, 0 tests failed out of 3
```

说明：

- 这里必须加 `-R '^osfs_'`。
- Step 02 必须先构建三项 OSFS 目标，其中包括当前版本新增的 `minikv_osfs_resilience_tests`。
- 不建议直接执行全量 `ctest --test-dir build-osfs-linux`，因为当前构建目录只构建了 OSFS 目标，而仓库里还注册了大量非 OSFS 测试；全量跑会输出很多“找不到可执行文件”的无关失败。
