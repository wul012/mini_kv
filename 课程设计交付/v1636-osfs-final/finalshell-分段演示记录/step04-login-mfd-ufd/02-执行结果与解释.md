## 关键解释

### 1. 预置用户复核

```text
osfs> LOGIN root root
OK login root uid=0
osfs> USERADD alice alice123
osfs> USERADD bob bob123
```

格式化阶段已经建立 `alice` 和 `bob`。这里两条重复 `USERADD` 实际会返回 `ERR user already exists`，但原 grep 没有保留该错误，所以这段历史输出不能作为用户创建成功证据。后续 Alice/Bob 均能以预置密码登录，且进入不同 UFD；二级目录隔离的有效证据来自后面的登录、CREATE 和 DIR，而不是这两条重复命令。

### 2. Alice 登录后进入自己的 UFD

```text
osfs> LOGIN alice alice123
OK login alice uid=1000
osfs> WHOAMI
alice uid=1000
osfs> CREATE alice-note
OK created alice-note
osfs> DIR
alice-note 4 0 0644 1000 0
```

Alice 登录成功后，`WHOAMI` 显示当前身份是 `alice uid=1000`。随后创建 `alice-note`，再执行 `DIR` 可以看到这个文件，说明文件被放进 Alice 自己的用户文件目录中。

### 3. Bob 登录后看到的是另一个目录

```text
osfs> LOGIN bob bob123
OK login bob uid=1001
osfs> WHOAMI
bob uid=1001
osfs> DIR
(empty)
```

Bob 登录后，当前身份变为 `bob uid=1001`。此时执行 `DIR` 显示 `(empty)`，没有看到 Alice 创建的 `alice-note`。这正是二级目录隔离的关键证据：不同用户进入的是各自的 UFD，而不是同一个共享根目录。

### 4. 本段结论

第 4 段证明：OSFS 已经实现了用户登录和按用户划分的二级目录结构。Alice 的文件留在 Alice 的 UFD 中，Bob 登录后进入 Bob 自己的 UFD，因此不会直接看到 Alice 的文件。
