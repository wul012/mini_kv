# Step 04：登录与二级目录隔离

用途：用一个很短的脚本演示“用户登录 + MFD/UFD 二级目录”。重点看：不同用户登录后看到的是自己的目录。

在 FinalShell 中粘贴执行：

```bash
clear

cd /home/azureuser/mini-kv

echo "========== Step 4: login and two-level directory =========="

rm -f /tmp/osfs-step04.img /tmp/osfs-step04.txt

cat > /tmp/osfs-step04.txt <<'EOF'
WHOAMI
LOGIN root root
LOGIN alice alice123
WHOAMI
CREATE alice-note
DIR
LOGIN bob bob123
WHOAMI
DIR
QUIT
EOF

./build-osfs-linux/minikv_osfs --disk /tmp/osfs-step04.img --format --script /tmp/osfs-step04.txt \
  | grep -E 'osfs>|OK login|alice uid|bob uid|alice-note|empty|BYE'

rm -f /tmp/osfs-step04.img /tmp/osfs-step04.txt

echo
echo "========== Step 4 done =========="
```

预期观察点：

- `LOGIN alice alice123` 后，`WHOAMI` 应显示 `alice uid=1000`。
- `--format` 会预置 root、Alice 和 Bob，因此这里直接登录，不再重复执行 `USERADD`。
- Alice 创建 `alice-note` 后，`DIR` 能看到 `alice-note`。
- `LOGIN bob bob123` 后，`WHOAMI` 应显示 `bob uid=1001`。
- Bob 执行 `DIR` 应显示空目录，即 Bob 看不到 Alice 自己 UFD 中的文件。
