# Step 08：执行结果与解释

状态：待 FinalShell 实跑。

这里暂不放“预期输出截图”，以免与实际 Linux 证据混淆。实跑后应原样保留命令、退出状态和以下关键行：

```text
payload_bytes= 4191
OK wrote 4191 bytes offset=4191
fd=3 read_offset=0 write_offset=4191
OK seek 4 4088
DIRECT8|INDIRECT-BEGIN|...
FSCK OK checks=6
BYE
```

讲解时要指出：offset 4088 到 4095 位于第八个直接块的末尾，从 offset 4096 开始进入逻辑块 8。逻辑块 8 的物理块号不是保存在 `direct[8]`，因为 direct 只有 0-7；它保存在 `indirect_block` 指向的块号数组第 0 项。

验收失败条件：写入长度不超过 4096、读取没有跨边界标记、使用旧二进制、FSCK 报错、或没有保存真实终端结果，都不能把本步骤标为完成。
