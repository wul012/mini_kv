# Step 08：直接块到一级间接块边界

状态：已在 FinalShell 实跑通过；本命令保留用于再次复现。执行前确认 `build-osfs-linux/minikv_osfs` 已由当前提交构建。

```bash
cd /home/azureuser/mini-kv
rm -f /tmp/osfs-step08.img /tmp/osfs-step08.txt /tmp/osfs-step08.log

python3 - <<'PY'
from pathlib import Path

payload = "X" * 4088 + "DIRECT8|INDIRECT-BEGIN|" + "Y" * 80
commands = [
    "LOGIN alice alice123",
    "CREATE bigfile",
    "OPEN bigfile w",
    f"WRITE 3 {payload}",
    "TELL 3",
    "CLOSE 3",
    "OPEN bigfile r",
    "SEEK 4 4088",
    "READ 4 64",
    "CLOSE 4",
    "FSCK",
    "QUIT",
]
Path("/tmp/osfs-step08.txt").write_text("\n".join(commands) + "\n", encoding="utf-8")
print("payload_bytes=", len(payload))
PY

./build-osfs-linux/minikv_osfs \
  --disk /tmp/osfs-step08.img \
  --format --blocks 192 \
  --script /tmp/osfs-step08.txt \
  > /tmp/osfs-step08.log

echo
echo "[1] Direct-to-indirect boundary evidence"
grep -E '^(OK wrote|fd=3 |OK seek|DIRECT8\|INDIRECT-BEGIN|FSCK OK|BYE)' \
  /tmp/osfs-step08.log

rm -f /tmp/osfs-step08.img /tmp/osfs-step08.txt /tmp/osfs-step08.log

echo
echo "========== Step 8 done =========="
```

观察点：payload 共 4191 字节，超过八个 512 字节直接块的 4096 字节容量。offset 4088 开始的 64 字节读取应连续包含 `DIRECT8|INDIRECT-BEGIN|`，最后 FSCK 仍为 OK。程序的完整回显先写入临时日志，再仅筛选关键行，避免 4191 字节的 `WRITE` 命令挤满终端。
