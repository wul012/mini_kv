# Step 02：CMake / Ninja 构建验证

用途：在远程 Linux 上重新运行 CMake 配置和 Ninja 构建，证明 `minikv_osfs` 与 `minikv_osfs_tests` 可以真实编译生成。

在 FinalShell 中粘贴执行：

```bash
clear

cd /home/azureuser/mini-kv

echo
echo "========== [1] Configure =========="
echo
cmake -S . -B build-osfs-linux -G Ninja -DCMAKE_BUILD_TYPE=Debug | tail -8

echo
echo "========== [2] Build =========="
echo
cmake --build build-osfs-linux --target minikv_osfs minikv_osfs_tests --parallel 1

echo
echo "========== [3] Binaries =========="
echo
ls -lh build-osfs-linux/minikv_osfs build-osfs-linux/minikv_osfs_tests

echo
file build-osfs-linux/minikv_osfs build-osfs-linux/minikv_osfs_tests

echo
echo "========== Step 2 done =========="
```

预期观察点：

- CMake 应显示配置/生成完成，或者显示构建目录已写入。
- Ninja 构建应成功；如果之前已经构建过，可能显示 `ninja: no work to do.`，这也是正常的。
- `ls -lh` 应能看到 `minikv_osfs` 和 `minikv_osfs_tests` 两个可执行文件。
- `file` 应显示它们是 Linux ELF 可执行文件。
