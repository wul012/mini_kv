# OSFS v1633 课堂演示步骤

## 一、准备

使用新路径格式化磁盘，避免旧 v1 镜像触发版本不兼容：

```powershell
.\cmake-build-osfs\minikv_osfs.exe --disk osfs-course.img --format --blocks 96 --script .\课程设计交付\v1633-osfs-final\osfs-demo-script.txt
```

开始前可以先打开报告图 2，说明磁盘中依次存在超级块、block bitmap、inode bitmap、用户表、inode 表和数据区。

## 二、按输出讲解

1. `WHOAMI` 返回 `not logged in`，随后 CREATE 被 `ERR login required` 阻断。文件命令不能绕过认证。
2. `LOGIN alice wrong` 返回认证失败；`LOGIN alice alice123` 成功。用户名、uid、gid、密码哈希和 UFD inode 位于磁盘用户表。
3. Alice 的首次 `DIR` 为空。CREATE 后，目录项进入 Alice UFD，不进入共享根目录。
4. `OPEN report w` 得到 fd 3。两次 WRITE 后 write offset 依次变为 7、13，TELL 显示句柄状态。
5. 关闭后以 fd 4 只读打开。三次 READ 分别得到 `course`、`-`、`design`，read offset 最终为 13。
6. CHMOD 0600 后 DIR 行显示文件名、inode、物理块号、保护码、owner 和长度 13。
7. Bob 登录后自己的 DIR 为空，说明 UFD 隔离。`OPEN alice/report r` 被权限拒绝；Bob 创建的 `bob-note` 只出现在 Bob UFD。
8. root 登录后可执行 `DIR alice`，再读取 `alice/report`。这说明 root 通过用户表/MFD 找到目标 UFD，而不是依赖 Alice 会话内存。

## 三、重新打开证明持久化

第二次启动不要使用 `--format`：

```powershell
.\cmake-build-osfs\minikv_osfs.exe --disk osfs-course.img
```

登录 Alice 后执行 `DIR` 和 `OPEN report r`，仍能看到并读取文件。用户表、MFD/UFD、inode 和数据块都来自镜像。

## 四、答辩中主动说明的边界

- 这是 MFD/UFD 两级目录，不是任意深度目录树。
- 用户表固定容量，密码哈希是教学简化。
- inode 只有八个直接块，不含间接块。
- 没有 fsck、日志恢复和多进程并发。
- OSFS 是独立课设入口，不复用 KV/WAL 伪装文件系统。
