# OSFS 课堂演示步骤

## 1. 先说明选题

本项目选择课程要求中的“实验二 Linux 二级文件系统”。实现方式是用二进制文件模拟磁盘空间，在固定大小的 block 上组织超级块、块位图、inode 表、目录项和数据块。

演示前可以先说清楚一点：OSFS 是 mini-kv 项目里的独立旁路模块，专门用于操作系统课设，不会把原来的 KV/WAL/TCP 逻辑改成文件系统。

## 2. 构建目标

Linux 环境可执行：

```bash
cmake -S . -B build
cmake --build build --target minikv_osfs
```

Windows 本机验证可使用 CLion 捆绑 CMake/MinGW，命令形态类似：

```powershell
$env:PATH = 'D:\CLION\CLion 2025.1.1\bin\mingw\bin;' + $env:PATH
$cmake = 'D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\cmake.exe'
& $cmake -S . -B cmake-build-v1630 -G "MinGW Makefiles"
& $cmake --build cmake-build-v1630 --target minikv_osfs minikv_osfs_tests
```

## 3. 运行脚本演示

```bash
./build/minikv_osfs --disk data/osfs-course.img --format --blocks 96 --script 课程设计交付/v1630-osfs-final/osfs-demo-script.txt
```

Windows 本机路径示例：

```powershell
.\cmake-build-v1630\minikv_osfs.exe --disk .\cmake-build-v1630\osfs-course.img --format --blocks 96 --script .\课程设计交付\v1630-osfs-final\osfs-demo-script.txt
```

## 4. 演示时重点解释的输出

1. `OK created report`：说明目录项和 inode 分配成功。
2. `OK fd=3`：说明 open 命令返回句柄，后续 read/write 不直接操作文件名。
3. `OK wrote ... bytes`：说明数据写入虚拟磁盘的数据块。
4. `course design storage layer`：说明 read 通过 fd 读回刚才写入的内容。
5. `OK chmod report 0600`：说明保护码被修改。
6. `OK login guest ...` 后的 `ERR permission denied`：说明普通用户没有 other read 位时不能读 root 创建的文件。
7. `DIR` 输出中的文件名、inode、mode、owner、size：对应课程要求“dir 列目录时列出文件名、物理地址、保护码和文件长度”。

## 5. 建议答辩顺序

先讲磁盘布局，再讲 create/open/write/read 的路径，最后讲权限拒绝。不要一开始就展开全部源码；老师通常更关心你是否理解文件系统内部结构，而不是背函数名。
