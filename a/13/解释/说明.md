# 第十三版开发调试运行归档说明

第十三版在第十二版请求上限和客户端 timeout 基础上，补强 TCP 网络兼容性测试：

- 项目版本更新到 `0.13.0`。
- `tcp_server_tests` 的测试客户端不再写死 IPv4 `127.0.0.1` socket，而是使用 `getaddrinfo(AF_UNSPEC)` 解析并尝试连接。
- `tcp_server_tests` 新增 `localhost` 绑定和连接场景，覆盖 Windows 上常见的 IPv4 / IPv6 / hostname 解析差异。
- README 更新为 Version 13，并补充 `localhost` 启动示例和测试覆盖说明。
- 真实 smoke 使用 `minikv_server.exe 6413 localhost` 和 `minikv_client.exe localhost 6413 2000` 验证 hostname 链路。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证前已把 MinGW 运行时目录加入当前 PowerShell 进程的 `PATH`，避免测试程序因找不到运行时 DLL 误报 `0xc0000135`。

## 04-localhost-network-test.png

- 命令：`.\cmake-build-debug\minikv_tcp_server_tests.exe`
- 结果：退出码为 0，结果为 `PASS`。该测试程序成功时不输出正文，所以截图记录了命令和 PASS 结论。
- 意义：单独复跑增强后的 TCP server 测试，确认：
  - IPv4 loopback 场景仍然通过。
  - 请求上限 `max_request_bytes=8` 和 `event=tcp_request_rejected` 仍然通过。
  - `localhost` 绑定和 `localhost` 连接能通过 `getaddrinfo(AF_UNSPEC)` 正常通信。
  - `localhost` 场景下连接统计仍然是 `total=1`、`active=0`、`peak=1`。

## 05-localhost-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6413 localhost --max-request-bytes 16 --accept-poll-ms 50`。
- bundled client 验证：用 `.\cmake-build-debug\minikv_client.exe localhost 6413 2000` 发送 `PING v13-localhost` / `QUIT`。
- over-limit 验证：用 Python `socket.create_connection(('localhost', 6413))` 发送 17 字节无换行请求 `01234567890123456`。
- 结果：
  - client 输出 `socket timeout: 2000 ms`。
  - `PING v13-localhost` 返回 `v13-localhost`，`QUIT` 返回 `BYE`。
  - over-limit socket 收到 `ERR line too long`。
  - server stdout 出现 `event=server_start host=localhost ... max_request_bytes=16 accept_poll_ms=50`。
  - server stdout 出现 `event=tcp_request_rejected ... pending_bytes=17 max_request_bytes=16`。
- 意义：确认第十三版真实 TCP 链路能使用 `localhost` hostname 启动和连接；同时第十二版请求上限、客户端 timeout 和拒绝日志能力仍然有效。
