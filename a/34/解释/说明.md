# v34 运行调试归档说明

本轮版本目标：

```text
在 v31 终端行编辑能力基础上，为 bundled TCP client 增加 Tab 命令补全。
```

v34 重点验证：

```text
LineEditorCompletion 支持命令前缀补全。
Tab 按键接入 read_client_line 的交互式按键循环。
唯一匹配补全为完整命令并追加空格。
多个匹配时只扩展最长公共前缀。
参数位置和 token 中间不做补全，避免误改 key / value。
非交互输入仍走 fallback_read_line，不影响脚本和 smoke test。
代码讲解记录在开发调试同一轮内同步补齐。
```

## 04-line-editor-tests.png

- 命令：`.\cmake-build-debug\minikv_line_editor_tests.exe`
- 结果：退出码为 0。
- 意义：专项验证行编辑和补全逻辑：
  - `PI` 补全为 `PING `。
  - `  pi` 保留前导空白并补全为 `  PING `。
  - `:h` 补全为 `:history `。
  - `HEA` 补全为 `HEALTH `。
  - `E` 在 `EXIT` / `EXPIRE` 之间扩展为公共前缀 `EX`。
  - `S` 歧义过大且公共前缀不变，不修改。
  - `BAD` 无匹配，不修改。
  - 光标在参数中或 token 中间时不补全。

## 05-client-completion-smoke.png

- 命令：真实启动 `minikv_server.exe`，再用真实 `minikv_client.exe` 通过重定向 stdin 输入命令。
- client 输入流程：

```text
PING
:history
QUIT
```

- 关键结果：
  - client 收到 `PONG`。
  - 本地 `:history` 输出 `1  PING`。
  - client 收到 `BYE`。
  - server 输出 `event=tcp_listen` 和 `event=tcp_client_accepted`。
  - 验证结束后停止本轮启动的 server 进程。

调试意义：

```text
Tab 补全是交互式终端能力，纯补全逻辑由 line_editor_tests 覆盖。
真实 smoke 继续验证非交互重定向输入仍走 fallback_read_line，确保脚本和自动化流程没有被 Tab 支持破坏。
```
