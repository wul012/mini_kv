# 第十九版开发调试运行归档说明

第十九版在第十八版并发 RESP-over-TCP 客户端覆盖基础上，转向持久化恢复可靠性加固：

- 项目版本更新到 `0.19.0`。
- 新增 `WalReplayReport`，记录 WAL replay 的 `applied_records`、`skipped_records` 和 `truncated_records`。
- `WriteAheadLog::replay()` 保持兼容，仍返回成功应用的记录数。
- 新增 `WriteAheadLog::replay_with_report()`，用于需要观测跳过和截断记录的入口。
- WAL replay 会跳过空白行和坏记录。
- WAL replay 会把末尾无换行的最后一条记录视为疑似部分写入，跳过并计入 `truncated_records`。
- `minikv_cli` 启动时会显示 replayed / skipped / truncated 数量。
- `minikv_server` 的 `event=wal_replay` 日志会显示 `records`、`skipped` 和 `truncated`。
- `wal_tests` 新增坏 WAL 和尾部半条 WAL 记录验证。
- `snapshot_tests` 新增坏 snapshot 不替换现有 Store 的验证。
- README 更新为 Version 19，并补充 WAL replay report 和 corrupt snapshot 保护说明。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`

  - `BROKEN record`
  - 末尾无换行的 `SET partial value`
- 外部客户端验证：Python raw socket 发送 inline `GET kept` / `GET partial` / `QUIT`。
- 结果：
  - `GET kept` 返回 `value`。
  - `GET partial` 返回 `(nil)`。
  - `MATCH replayed_state True`。

  - `MATCH wal_replay_report True`。
  - 服务端 stderr 为空。
- 意义：确认真实 `minikv_server.exe` 启动时会跳过坏 WAL 行和尾部半条 WAL，只恢复完整有效记录，并把恢复报告输出到结构化日志。
