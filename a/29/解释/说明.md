# v29 运行调试归档说明

本轮版本目标：

```text
在 v28 可配置 WAL maintenance 的基础上，新增运行期观测命令 STATS / HEALTH。
```

v29 重点验证：

```text
STATS 可以查询 live key、WAL 状态、compact 阈值、compact counters 和 TCP 连接统计。
HEALTH 可以返回简短 OK 状态，适合快速探活。
CLI 模式没有 server 连接统计时返回 connection_stats_available=no。
TCP server 模式会把 active / total / peak 连接统计注入命令层。
```

## 06-cli-stats-health-smoke.png

- 命令：真实运行 `minikv_cli.exe <wal_path> --auto-compact-wal --wal-compact-min-records 4 --wal-compact-record-ratio 2 --wal-compact-min-bytes 1048576`。
- 输入流程：

```text
SET observed one
STATS
HEALTH
WALINFO
QUIT
```

- 关键结果：
  - `SET observed one` 返回 `OK inserted`。
  - `STATS` 返回 `live_keys=1 wal_enabled=yes ... connection_stats_available=no`。
  - `HEALTH` 返回 `OK live_keys=1 wal_enabled=yes compact_recommended=no ...`。
  - `WALINFO` 仍能输出 v28 的 compact 阈值和 compact counters。
- 意义：确认本地 CLI 模式下新命令可用，并且没有 server 连接统计时会明确标记不可用。

## 07-server-stats-health-smoke.png

- 命令：真实启动 `minikv_server.exe`，再用真实 `minikv_client.exe` 连接。
- server 参数包含：

```text
--auto-compact-wal
--wal-compact-min-records 4
--wal-compact-record-ratio 2
--wal-compact-min-bytes 1048576
--accept-poll-ms 25
```

- client 输入流程：

```text
SET srv value
STATS
HEALTH
QUIT
```

- 关键结果：
  - `SET srv value` 返回 `OK inserted`。
  - `STATS` 返回 `connection_stats_available=yes active_connections=1 total_connections=1 peak_connections=1`。
  - `HEALTH` 返回 `OK live_keys=1 ... connection_stats_available=yes`。
  - 验证结束后停止本轮启动的 server 进程。
- 意义：确认 TCP server 会把已有连接统计通过 `CommandProcessorOptions::connection_stats` 注入命令层，运行期查询能看到真实连接状态。
