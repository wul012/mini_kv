# 第十八版开发调试运行归档说明

第十八版在第十七版 RESP-over-TCP 兼容边界测试基础上，继续补并发 RESP 客户端覆盖：

- 项目版本更新到 `0.18.0`。
- 新增 `tests/tcp_resp_concurrency_tests.cpp`。
- 新增构建目标 `minikv_tcp_resp_concurrency_tests`。
- 新增 CTest 用例 `tcp_resp_concurrency_tests`。
- 新测试启动 4 个 raw-socket RESP 客户端，并让它们同时连接同一个 `TcpServer`。
- 测试等待服务端统计到 `active_connections=4` 后统一放行客户端发送 pipeline。
- 每个客户端发送独立 key 的 `SET` / `GET` / `PING` / `TTL` / `DEL` / `GET` / `QUIT`。
- 测试精确校验每个客户端的 RESP 响应，并检查 `total_connections=4`、`active_connections=0`、`peak_connections=4`。
- 修复 `server_main` 并发 stdout 日志输出：`TcpServer::Options::logger` 现在加锁写入，避免多个连接线程同时输出时结构化日志粘行。
- README 更新为 Version 18，并补充并发 RESP 客户端覆盖和线程安全日志说明。

本轮验证继续使用 CLion 捆绑工具链：

## 04-tcp-resp-concurrency-tests.png

- 命令：`.\cmake-build-debug\minikv_tcp_resp_concurrency_tests.exe`
- 结果：退出码为 0，结果为 `PASS`。该测试程序成功时不输出正文，所以截图记录了命令和 PASS 结论。
- 意义：单独复跑新增测试，确认：
  - 测试内启动 `TcpServer` 并绑定临时端口。
  - 4 个 raw-socket RESP 客户端先同时连接并保持打开。
  - 服务端连接统计达到 `active_connections=4` 后，测试统一放行客户端发送 pipeline。

  - 每个客户端收到精确 RESP 响应：`+OK inserted`、bulk value、bulk ping message、`:-1`、`:1`、`$-1`、`+BYE`。
  - 最终 `Store` 大小回到 0。
  - 连接统计满足 `total_connections=4`、`active_connections=0`、`peak_connections=4`。

## 05-external-resp-concurrency-smoke.png

- 服务端命令：`.\cmake-build-debug\minikv_server.exe 6419 127.0.0.1 --accept-poll-ms 50`
- 外部客户端验证：Python 启动 4 个线程，每个线程创建一个 raw socket RESP 客户端。
- 客户端行为：4 个客户端先连接并在 barrier 等待，然后一起发送 `SET` / `GET` / `PING` / `TTL` / `DEL` / `GET` / `QUIT` pipeline。
- 结果：
  - `client 0 match True`
  - `client 1 match True`
  - `client 2 match True`
  - `client 3 match True`
  - `MATCH concurrent_clients True`
  - 服务端日志出现 `active_connections=4` 和 `peak_connections=4`。
  - 服务端 stderr 为空。
  - 并发关闭日志逐行完整输出，没有再出现结构化日志粘行。
- 意义：确认不只测试进程内可用，真实 `minikv_server.exe` 也能同时服务多个外部 RESP 客户端，并且连接指标和 stdout 日志在并发场景下保持可读。
