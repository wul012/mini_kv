# v36 运行调试归档说明

本轮版本目标：

```text
为 bundled TCP client 增加本会话 key 补全能力。
```

v36 重点验证：

```text
LineEditorCompletionOptions 将命令补全和 key 补全候选拆开。
命令 token 仍然走大小写不敏感补全。
GET / SET / DEL / EXPIRE / TTL 的第二个 token 支持 key 补全。
key 补全保持大小写敏感，且不会补第三个 token 或光标中间位置。
minikv_client 在成功 SET 后记住 key，在 DEL 后移除 key，在 LOAD 后清空本地 key 候选。
非交互脚本输入仍然走 getline fallback，真实 server + client smoke 继续可用。
代码讲解记录和开发调试同轮补齐。
```

## 04-line-editor-tests.png

- 命令：`.\cmake-build-debug\minikv_line_editor_tests.exe`
- 结果：退出码为 0。
- 意义：专项验证客户端补全语义：
  - `GET na` 能补成已知 key `name`。
  - `GET al` 对 `alpha` / `alpine` 只扩展到公共前缀 `alp`。
  - `SET`、`EXPIRE`、`TTL` 的 key 参数同样能补全。
  - `GET AL` 不匹配 `name`，说明 key 补全大小写敏感。
  - `PING na`、`GET name now`、光标在 key token 中间时不补全。

## 05-client-key-smoke.png

- 命令：真实启动 `minikv_server.exe`，再用真实 `minikv_client.exe` 通过重定向 stdin 输入命令。
- client 输入流程：

```text
SET alpha one
SET alpine two
GET alpha
DEL alpha
GET alpha
GET alpine
QUIT
```

- 关键结果：
  - 两次 `SET` 返回 `OK inserted`。
  - `GET alpha` 返回 `one`。
  - `DEL alpha` 返回 `1`。
  - 再次 `GET alpha` 返回 `(nil)`。
  - `GET alpine` 返回 `two`。
  - `QUIT` 返回 `BYE`。
  - server 日志出现 `event=tcp_listen`、`event=tcp_client_accepted`、`event=tcp_client_closed`。

调试意义：

```text
单元测试覆盖 Tab 补全核心逻辑，真实 smoke 证明 v36 改造没有破坏脚本输入和 server/client 运行闭环。
```
