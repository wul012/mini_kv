# 第二十七版开发调试运行归档说明

第二十七版在第二十六版 `WALINFO` 和 compact hint 的基础上，补上可选自动 WAL compact。核心目标是：当用户明确开启 `--auto-compact-wal` 后，CLI 和 server 可以在启动 replay 后、以及后续 WAL 写命令后，根据同一套 maintenance 阈值自动把长 WAL 重写成短 WAL。

本轮主要改动：

- 项目版本更新到 `0.27.0`。
- 新增 `WalAutoCompactReport`，记录自动 compact 前后 WAL 状态、compacted key 数和是否真正执行 compact。
- `WriteAheadLog` 新增 `compact_if_recommended(const Store&, WalAutoCompactReport*)`。
- `CommandProcessorOptions` 新增 `auto_compact_wal`。
- WAL 模式下的 `SET` / `DEL` / `EXPIRE` 在写入成功后可自动检查并 compact。
- CLI 新增 `--auto-compact-wal`。
- server 新增 `--auto-compact-wal`，并在启动日志中输出 `event=wal_auto_compact` 或 `event=wal_auto_compact_skipped`。
- `TcpServer::Options` 新增 `auto_compact_wal`，让真实 TCP 请求也能复用自动 compact。
- `wal_tests` 增加自动 compact 覆盖。
- README 更新为 Version 27，并补充自动 compact 用法、测试说明和 Roadmap。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`

## 05-cli-auto-compact-smoke.png

- 命令：真实运行 `minikv_cli.exe`。
- 验证流程：
  - 先不带自动 compact，连续 10 次写入同一个 key，制造长 WAL。
  - 再使用 `--auto-compact-wal` 启动 CLI，确认启动期自动 compact。
  - 再使用新的 WAL 文件，启动后连续 8 次写入同一个 key，确认运行期写命令触发自动 compact。
- 关键结果：
  - 启动期输出 `WAL auto compact: compacted=1 records=10->1 bytes=380->38`。
  - 启动期 `WALINFO` 输出 `wal_records=1 live_keys=1 compact_recommended=no`。
  - 运行期连续写入后 `WALINFO` 输出 `wal_records=1 live_keys=1 compact_recommended=no`。
  - 最终两个 WAL 文件都只剩 1 条 WAL2 记录。

## 06-server-auto-compact-smoke.png

- 命令：真实启动 `minikv_server.exe <port> 127.0.0.1 <wal_path> --auto-compact-wal --accept-poll-ms 25`，再用真实 `minikv_client.exe` 连接。
- 验证流程：
  - 先用 CLI 构造 10 条同 key WAL 历史。
  - 启动 server 并检查启动日志。
  - 客户端执行 `WALINFO`、连续 7 次 `SET srv liveN`、`WALINFO`、`GET srv`、`QUIT`。
  - 验证结束后停止 server 进程。
- 关键结果：
  - server 启动日志包含 `event=wal_auto_compact ... before_wal_records=10 after_wal_records=1`。
  - server 启动日志包含 `auto_compact_wal=true`。
  - 客户端第一次和第二次 `WALINFO` 都显示 `wal_records=1 live_keys=1 compact_recommended=no`。
  - `GET srv` 返回最后写入的 `live6`。
  - 最终 WAL 文件只剩 1 条 WAL2 记录。
