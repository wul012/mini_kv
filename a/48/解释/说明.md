# 第四十八版运行调试归档说明

本轮版本目标：

```text
按最新跨项目计划新增 EXPLAINJSON <command>，让 mini-kv 提供结构化 raw command 风险解释。
```

本轮只做 `EXPLAINJSON`，不执行被 explain 的命令，不做 ACL，不做事务，不新增后台扫描，不让 mini-kv 承担订单、库存、支付的权威存储。

---

# 5. 05-explainjson-smoke.png

对应流程：

```powershell
启动 minikv_server.exe 6411 127.0.0.1
使用 minikv_client.exe 发送：
  EXPLAINJSON SET orderops:1 value
  GET orderops:1
  EXPLAINJSON GET orderops:1 extra
  QUIT
```

结果：

```text
SMOKE OK
SET explain 返回 mutates_store=true 和 touches_wal=true；
GET orderops:1 仍然返回 (nil)；
错误参数返回 allowed_by_parser=false 和 usage warning。
```

意义：

```text
真实 server/client 链路证明 EXPLAINJSON 能解释 raw command 风险，而且不会执行被解释的写命令。
```
