# 第二十版开发调试运行归档说明

第二十版在第十九版持久化恢复可靠性加固基础上，补齐项目级持续集成入口，让当前本地可通过的构建和测试流程可以在 GitHub Actions 上跨平台重复执行：

- 项目版本更新到 `0.20.0`。
- 新增 `.github/workflows/ci.yml`。
- CI 触发入口包括 `push` 到 `main` / `master`、`pull_request` 和手动 `workflow_dispatch`。
- CI 矩阵覆盖 `ubuntu-latest`、`macos-latest` 和 `windows-latest`。
- 每个 CI job 执行 `actions/checkout@v4`、CMake 配置、CMake 构建和 CTest。
- README 更新为 Version 20，并新增 CI 说明。
- README Roadmap 移除已经完成的 CI 项，后续重点转为客户端体验和更强恢复保护。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证前已把 MinGW 运行时目录加入当前 PowerShell 进程的 `PATH`，避免测试程序因为找不到运行时 DLL 误报 `0xc0000135`。

## 01-cmake-configure.png

- 命令：`cmake -S . -B cmake-build-debug`
- 结果：配置成功，构建文件写入 `cmake-build-debug`。
- 意义：确认第二十版 CMake 版本号 `0.20.0` 能被 CMake 正确识别，现有目标仍能正常生成。

## 02-cmake-build.png

- 命令：`cmake --build cmake-build-debug --clean-first`
- 结果：先清理旧构建产物，再完整编译并链接 38 个构建步骤。
- 意义：确认新增 CI 工作流文件和 README 调整没有影响 C++20 / MinGW 本地构建，已有服务端、客户端、benchmark 和全部测试目标仍能正常编译。

## 03-ctest.png

- 命令：`ctest --test-dir cmake-build-debug --output-on-failure`
- 结果：11 个测试全部通过：
  - `store_tests`
  - `command_tests`
  - `wal_tests`
  - `snapshot_tests`
  - `stress_tests`
  - `resp_tests`
  - `tcp_server_tests`
  - `tcp_resp_tests`
  - `tcp_resp_compat_tests`
  - `tcp_resp_concurrency_tests`
  - `client_history_tests`
- 意义：确认第二十版在只新增 CI 接入和文档更新的情况下没有引入功能回归，Store、CommandProcessor、WAL、Snapshot、Stress、RESP parser、TCP server、RESP-over-TCP 和客户端历史能力仍保持稳定。
