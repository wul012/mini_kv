第十七版在第十六版外部 RESP-over-TCP pipeline 验证基础上，继续扩大 RESP 兼容性覆盖：

- 项目版本更新到 `0.17.0`。
- 新增 `tests/tcp_resp_compat_tests.cpp`。
- 新增构建目标 `minikv_tcp_resp_compat_tests`。

- 新测试覆盖 RESP null bulk reply、integer reply、command error、protocol error。
- 新测试覆盖 `GET missing`、`SET`、`TTL`、`EXPIRE`、`DEL`、未知命令、错误参数数量和非法 null bulk 请求。

## 04-tcp-resp-compat-tests.png

- 命令：`.\cmake-build-debug\minikv_tcp_resp_compat_tests.exe`
- 结果：退出码为 0，结果为 `PASS`。该测试程序成功时不输出正文，所以截图记录了命令和 PASS 结论。
- 意义：单独复跑新增测试，确认：
  - 测试内启动 `TcpServer` 并绑定临时端口。
  - 第一条连接发送 RESP compatibility pipeline。
  - `GET missing` 返回 `$-1` null bulk。
  - `SET durable value` 返回 `+OK inserted`。
  - `TTL durable` 在无过期时返回 `:-1`。
  - `EXPIRE durable 120` 返回 `:1`。
  - `DEL durable` 返回 `:1`。
  - 删除后 `TTL durable` 返回 `:-2`。
  - 未知命令返回 `-ERR unknown command`。
  - 错误参数数量返回 `-ERR usage: GET key`。
  - `QUIT` 返回 `+BYE` 并关闭连接。
  - 第二条连接发送非法 RESP null bulk 请求，返回 `-ERR null RESP bulk strings are not supported`。
  - 连接统计满足 `total_connections=2`、`active_connections=0`、`peak_connections=1`。

## 05-external-resp-compat-smoke.png

- 服务端命令：`.\cmake-build-debug\minikv_server.exe 6418 127.0.0.1 --accept-poll-ms 50`
- 外部客户端验证：用 Python 原始 socket 连接 `127.0.0.1:6418`。
- 第一段请求：`GET missing` / `SET durable value` / `TTL durable` / `EXPIRE durable 120` / `DEL durable` / `TTL durable` / `NOPE` / `GET durable extra` / `QUIT`。
- 第二段请求：非法 RESP null bulk：`*1\r\n$-1\r\n`。
- 结果：
  - compatibility pipeline 的原始字节匹配结果为 `MATCH compatibility True`。
  - protocol error 的原始字节匹配结果为 `MATCH protocol_error True`。
  - 服务端日志出现两次 `event=tcp_client_accepted` 和两次 `event=tcp_client_closed`。
  - 服务端 stderr 为空。
- 意义：确认不只测试进程内可用，真实 `minikv_server.exe` 也能按 RESP 协议返回 null bulk、integer、error 和 protocol error 帧。
