# 第四十版运行调试归档说明

本轮版本目标：

```text
新增服务端 KEYS 命令，返回当前仍然有效的 key 列表。
```

本轮刻意只做服务端 key-listing 基础能力，不把客户端自动刷新 key cache 一起塞进来，符合“每版工作量适中”的规则。

# 5. 05-keys-smoke.png

对应流程：

```powershell
启动 minikv_server.exe 6402 127.0.0.1
使用 minikv_client.exe 发送：
  SET beta two
  SET alpha one
  KEYS
  DEL beta
  KEYS
  QUIT
```

结果：

```text
SMOKE OK
第一次 KEYS 返回 key_count=2 keys=alpha beta
删除 beta 后第二次 KEYS 返回 key_count=1 keys=alpha
QUIT 返回 BYE
```

意义：

```text
KEYS 不只在单测里可用，也能通过真实 TCP server/client 链路工作，并且输出是稳定排序的一行文本。
```

后台进程处理：

```text
smoke 结束后已停止本轮启动的 minikv_server.exe。
tmp-v40-smoke-archive 已删除。
```
