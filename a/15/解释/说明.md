# 第十五版开发调试运行归档说明

第十五版在第十四版客户端连接重试基础上，补上 bundled TCP client 的本地会话历史能力：

- 项目版本更新到 `0.15.0`。
- 新增 `include/minikv/client_history.hpp` 和 `src/client_history.cpp`。
- `minikv_client` 支持本地命令 `:history`，用于显示本次会话已经发送到服务端的命令。
- `minikv_client` 支持本地命令 `!!`，用于重复上一条已发送命令。
- `minikv_client` 支持本地命令 `!N`，用于重复第 N 条历史命令，索引从 1 开始。
- 本地历史命令不会发送给服务端；重复命令会解析成真实服务端命令后再发送。
- 新增 `minikv_client_history_tests`，验证历史记录、重复上一条、重复指定编号、容量裁剪和空历史错误提示。
- README 更新为 Version 15，并补充客户端本地历史命令说明。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

## 04-client-history-tests.png

- 命令：`.\cmake-build-debug\minikv_client_history_tests.exe`
- 结果：退出码为 0，结果为 `PASS`。该测试程序成功时不输出正文，所以截图记录了命令和 PASS 结论。
- 意义：单独复跑客户端历史测试，确认：
  - 空历史执行 `:history` 返回 `history is empty`。
  - 普通命令会进入历史。
  - `:history` 会输出编号和命令。
  - `!!` 会重复上一条命令。
  - `!1` 会重复第 1 条命令。
  - 超出容量后历史会裁剪旧记录。
  - `ClientHistory{0}` 会被拒绝。

## 05-client-history-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6416 127.0.0.1 --accept-poll-ms 50`。
- bundled client 验证：用 `.\cmake-build-debug\minikv_client.exe 127.0.0.1 6416 2000`。
- 客户端输入：`PING first` / `:history` / `!!` / `!1` / `QUIT`。
- 结果：
  - `PING first` 返回 `first`。
  - `:history` 输出 `1  PING first`。
  - `!!` 输出 `repeating: PING first`，并再次收到 `first`。
  - `!1` 输出 `repeating: PING first`，并再次收到 `first`。
  - `QUIT` 返回 `BYE`。
  - 服务端日志出现 `event=tcp_client_accepted` 和 `event=tcp_client_closed`。
- 意义：确认第十五版真实进程链路中，客户端本地历史命令不会发给服务端；重复命令会被解析成真实 inline 命令并发送给服务端执行。
