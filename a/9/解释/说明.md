# 第九版开发调试运行归档说明

第九版在第八版 TCP RESP 接入的基础上，补齐一个更贴近 Redis 客户端习惯的兼容命令，并给 RESP parser 加上明确边界：

- 新增 `PING [message]` 命令：无参数返回 `PONG`，带消息时原样返回消息。
- RESP 输出中 `PONG` 使用 simple string：`+PONG\r\n`。
- RESP array 参数数量上限为 `1024`。
- RESP bulk string 长度上限为 `65536` 字节。
- 超限 RESP 请求会立即返回 RESP error，避免服务端一直等待异常大 payload。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证前已把 MinGW 运行时目录加入当前 PowerShell 进程的 `PATH`，避免测试程序因找不到运行时 DLL 误报 `0xc0000135`。

## 04-inline-tcp-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6399 127.0.0.1`，再用 `minikv_client.exe 127.0.0.1 6399` 发送 inline 命令。
- 输入流程：`PING`、`PING inline v9 smoke`、`QUIT`。
- 结果：客户端收到 `PONG`、`inline v9 smoke`、`BYE`。
- 意义：确认新增 `PING` 命令在旧的 inline TCP 协议和 bundled client 中可用。

## 05-resp-tcp-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6399 127.0.0.1`，再用 Python socket 发送原始 RESP 字节。
- 输入流程：
  - RESP `PING`
  - RESP `PING hello from resp ping`
  - RESP `QUIT`
  - 单独发送超限 array 声明 `*1025\r\n`
- 结果：
  - `PING` 返回 `+PONG\r\n`
  - echo ping 返回 `$20\r\nhello from resp ping\r\n`
  - `QUIT` 返回 `+BYE\r\n`
  - 超限 array 返回 `-ERR RESP command array has too many arguments\r\n`
- 意义：确认第九版 RESP 兼容路径和协议加固路径都能在真实 TCP 连接上工作。
