# 第三十九版运行调试归档说明

本轮版本目标：

```text
为服务端 metrics 文件导出增加大小轮转和保留数量控制。
```

新增能力：

```text
--metrics-file-max-bytes bytes
 -> 当下一条 metrics 行会让主文件超过阈值时，先轮转再写入。

--metrics-file-keep count
 -> 控制保留多少份旧 metrics 文件，命名为 <path>.1、<path>.2 等。
```

本轮还把“每个版本工作量要适中，不能多也不能少，除非不得已”写入 `AGENTS.md`，作为后续版本推进规则。

# 5. 05-metrics-rotation-smoke.png

对应流程：

```powershell
启动 minikv_server.exe 6400 127.0.0.1
参数：
  --metrics-interval-ms 50
  --metrics-file tmp-v39-smoke-archive\server.metrics.log
  --metrics-file-max-bytes 240
  --metrics-file-keep 2

再用 minikv_client.exe 发送：
  PING v39
  SET smoke-key smoke-value
  STATSJSON
  QUIT
```

结果：

```text
SMOKE OK
client 收到 v39、OK inserted、STATSJSON 和 BYE
生成 server.metrics.log、server.metrics.log.1、server.metrics.log.2
没有生成 server.metrics.log.3
```

意义：

```text
真实 server/client 路径可用，metrics 文件确实发生轮转，且 keep=2 生效。
```

后台进程处理：

```text
smoke 结束后已停止本轮启动的 minikv_server.exe。
```
