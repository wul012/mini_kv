# 第十四版开发调试运行归档说明

第十四版在第十三版 `localhost` 网络兼容测试基础上，补上 bundled TCP client 的可配置连接重试能力：

- 项目版本更新到 `0.14.0`。
- `minikv_client` 新增参数：`--connect-retries count`。
- `minikv_client` 新增参数：`--retry-delay-ms ms`，默认重试间隔为 `250 ms`。
- 客户端默认仍然不重试，旧用法 `minikv_client.exe [host] [port] [timeout_ms]` 保持兼容。
- 连接失败时，客户端会输出 `connect attempt current/total failed: ...` 和下一次重试等待时间。
- 连接错误现在会在 `connect` 失败当下保存，避免 Windows 下后续日志丢失真实 WSA 错误码。
- README 更新为 Version 14，并补充客户端等待服务端上线的示例。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证前已把 MinGW 运行时目录加入当前 PowerShell 进程的 `PATH`，避免测试程序因找不到运行时 DLL 误报 `0xc0000135`。

## 04-client-retry-smoke.png

- 命令：先启动客户端，再启动服务端。
- 客户端命令：`.\cmake-build-debug\minikv_client.exe 127.0.0.1 6415 --connect-retries 20 --retry-delay-ms 200`。
- 客户端输入：`PING v14-retry` / `QUIT`。
- 服务端延迟启动命令：`.\cmake-build-debug\minikv_server.exe 6415 127.0.0.1 --accept-poll-ms 50`。
- 结果：
  - 客户端在服务端启动前先输出 `connect attempt 1/21 failed: connect failed: WSA error 10061`。
  - 客户端输出 `retrying in 200 ms`。
  - 服务端启动后，客户端成功连接并输出 `connect retries: 20, retry delay: 200 ms`。
  - `PING v14-retry` 返回 `v14-retry`。
  - `QUIT` 返回 `BYE`。
  - 服务端日志出现 `event=tcp_client_accepted` 和 `event=tcp_client_closed`。
- 意义：确认第十四版真实进程链路中，客户端可以先于服务端启动，通过配置重试等待服务端上线，并在连接成功后继续正常执行 inline 命令。
