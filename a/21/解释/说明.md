# 第二十一版开发调试运行归档说明

第二十一版在第二十版跨平台 CI 基础上，回到 TCP 客户端体验改进：把第十五版已有的进程内本地历史扩展为可选持久化历史文件。

本版主要改动：

- 项目版本更新到 `0.21.0`。
- `ClientHistory` 新增 `load_from_file()` 和 `save_to_file()`。
- 历史文件缺失时视为 0 条历史，不报错。
- 历史文件加载时跳过空行，并兼容 Windows 文本里的行尾 `\r`。
- 继续复用 `ClientHistory::add()` 的容量裁剪逻辑，只保留最近的 bounded history。
- `save_to_file()` 会按需创建父目录，并用当前 bounded history 覆盖写入历史文件。
- `minikv_client` 新增 `--history-file path` 参数。
- 客户端启动时加载历史文件，发送命令并收到响应后保存历史文件。
- `client_history_tests` 新增缺失文件、保存加载、父目录创建、容量裁剪和 CRLF 兼容验证。
- README 更新为 Version 21，并补充 `--history-file` 用法和 Roadmap。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证前已把 MinGW 运行时目录加入当前 PowerShell 进程的 `PATH`，避免测试程序因为找不到运行时 DLL 误报 `0xc0000135`。

## 05-client-history-smoke.png

- 服务端命令：真实启动 `.\cmake-build-debug\minikv_server.exe <port> 127.0.0.1 --accept-poll-ms 50`
- 第一次客户端命令：真实运行 `.\cmake-build-debug\minikv_client.exe 127.0.0.1 <port> --history-file tmp-v21-run\history-smoke\client.history`
- 第一次输入：
  - `PING first`
  - `SET smoke history`
  - `QUIT`
- 第二次客户端命令：再次使用同一个 `--history-file`
- 第二次输入：
  - `:history`
  - `!2`
  - `GET smoke`
  - `QUIT`
- 结果：
  - 第一次会话创建历史文件。
  - 第二次会话显示 `3 entries available, 3 loaded`。
  - `:history` 显示第 2 条为 `SET smoke history`。
  - `!2` 输出 `repeating: SET smoke history` 并执行成功。
  - `GET smoke` 返回 `history`。
  - 保存后的历史文件包含两次会话发送过的命令。
  - 服务端接受了两个客户端连接，stderr 为空。
- 意义：确认新功能不只是单元测试可用，真实 server/client 进程也能跨会话复用历史文件。
