# 第四十三版运行调试归档说明

本轮版本目标：

```text
让 KEYS 支持可选 prefix 过滤，同时保护客户端全量 key cache 刷新语义。
```

本轮只做 `KEYS [prefix]`，不做分页、limit、通配符或模糊匹配，保持版本工作量适中。

# 5. 05-keys-prefix-smoke.png

对应流程：

```powershell
创建 client.keys，内容为 stale
启动 minikv_server.exe 6406 127.0.0.1
使用 minikv_client.exe --key-cache-file client.keys 发送：
  SET alpha one
  SET alpine two
  SET beta three
  KEYS
  KEYS alp
  QUIT
检查 client.keys
```

结果：

```text
SMOKE OK
KEYS 输出 key_count=3 keys=alpha alpine beta
KEYS alp 输出 key_count=2 prefix=alp keys=alpha alpine
client.keys 最终保存：
  alpha
  alpine
  beta
```

意义：

```text
真实 server/client 链路证明 prefix filter 生效；
无参数 KEYS 仍能刷新全量 key cache；
带 prefix 的 KEYS 不会把 full cache 缩成过滤子集。
```

后台进程处理：

```text
smoke 结束后已停止本轮启动的 minikv_server.exe。
```
