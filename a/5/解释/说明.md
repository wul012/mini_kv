# 第五版开发调试运行归档说明

第五版把 mini-kv 从「WAL 增量持久化」推进到「手动快照持久化」。本轮验证使用 CLion 自带工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证时临时把 MinGW 运行时目录加入本次 PowerShell 进程的 `PATH`，用于加载测试程序和可执行文件依赖的 DLL。

## 01-cmake-configure.png

- 命令：`cmake -S . -B cmake-build-debug`
- 结果：配置成功，构建目录为 `cmake-build-debug`。
- 意义：确认第五版新增的 `snapshot.cpp`、`snapshot_tests.cpp` 和 CMake 目标都能被 CMake 正确识别。

## 02-cmake-build.png

- 命令：`cmake --build cmake-build-debug`
- 结果：编译成功，生成 `minikv_cli.exe`、`minikv_server.exe`、`minikv_client.exe` 以及测试程序。
- 意义：确认快照模块、命令层、Store TTL 数据结构调整可以通过 C++20 编译。

## 03-ctest.png

- 命令：`ctest --test-dir cmake-build-debug --output-on-failure`
- 结果：4 个测试全部通过：
  - `store_tests`
  - `command_tests`
  - `wal_tests`
  - `snapshot_tests`
- 意义：确认原有内存 KV、命令解析、WAL 行为未被破坏，并验证第五版快照保存、加载、TTL 保留逻辑可用。

## 04-cli-snapshot-smoke.png

- 命令：通过管道向 `.\cmake-build-debug\minikv_cli.exe` 输入快照命令。
- 输入流程：`SET` 写入数据，`SAVE` 保存快照，`DEL` 删除数据，`LOAD` 加载快照，再用 `GET` 读取。
- 结果：出现 `OK saved 1`、`OK loaded 1`，最终 `GET v5` 返回 `first snapshot value`。
- 意义：确认交互式 CLI 中的 `SAVE path` 和 `LOAD path` 可以完成手动快照恢复。

## 05-tcp-snapshot-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6395`，再通过 `minikv_client.exe 127.0.0.1 6395` 输入快照命令。
- 输入流程：`SET` 写入 TCP 会话数据，`SAVE` 保存快照，`DEL` 删除数据，`LOAD` 加载快照，再用 `GET` 读取。
- 结果：客户端收到 `OK saved 1`、`OK loaded 1`，最终 `GET tcp` 返回 `from network snapshot`。
- 意义：确认 TCP 服务端复用同一套命令处理器后，也支持第五版快照功能。

## 进程与清理

- TCP smoke test 启动的服务端监听端口为 `6395`，截图中记录了当次 PID，验证结束后已停止。
- 验证期间生成的临时快照文件 `a\5\mini-kv-v5-cli.snapshot` 和 `a\5\mini-kv-v5-tcp.snapshot` 已删除。
- 验证期间生成的临时目录 `.tmp-v5-smoke` 已删除。
