# 第八版开发调试运行归档说明

第八版把第七版已经完成的 RESP parser 接入 TCP server。本轮新增能力：

- TCP server 自动识别 inline 文本命令和 RESP array 请求。
- RESP 请求返回 RESP 格式响应，包括 simple string、integer、bulk string、null bulk string 和 error。
- bundled `minikv_client` 不再依赖服务端启动问候语，避免干扰 RESP 客户端。

本轮验证使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证时临时把 MinGW 运行时目录加入当前 PowerShell 进程的 `PATH`，避免测试程序因找不到运行时 DLL 报 `0xc0000135`。

## 01-cmake-configure.png

- 命令：`cmake -S . -B cmake-build-debug`
- 结果：配置成功，构建目录为 `cmake-build-debug`。
- 意义：确认第八版更新后的 CMake 版本、RESP/TCP 相关源码和测试目标都能被 CMake 正确识别。

## 02-cmake-build.png

- 命令：`cmake --build cmake-build-debug`
- 结果：编译成功。
- 意义：确认 RESP response formatting、TCP server 协议自动识别、client 去问候语依赖等改动可以通过 C++20 编译并成功链接。

## 03-ctest.png

- 命令：`ctest --test-dir cmake-build-debug --output-on-failure`
- 结果：6 个测试全部通过：
  - `store_tests`
  - `command_tests`
  - `wal_tests`
  - `snapshot_tests`
  - `stress_tests`
  - `resp_tests`
- 意义：确认第八版新增 TCP RESP 接入后，原有 Store、CommandProcessor、WAL、Snapshot、Stress、RESP parser 行为未被破坏。

## 04-inline-tcp-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6398`，再用 `minikv_client.exe 127.0.0.1 6398` 发送 inline 命令。
- 输入流程：`SET inline v8-ok`，`GET inline`，`QUIT`。
- 结果：客户端收到 `OK inserted`、`v8-ok`、`BYE`。
- 意义：确认去掉服务端启动问候语后，旧的 inline TCP 协议和 bundled client 仍然可用。

## 05-resp-tcp-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6398`，再用 Python socket 发送原始 RESP 字节。
- 输入流程：RESP pipeline 发送 `SET resp v8-wire`、`GET resp`、`SIZE`、`QUIT`。
- 结果：服务端返回 `+OK inserted\r\n`、`$7\r\nv8-wire\r\n`、`:1\r\n`、`+BYE\r\n`。
- 意义：确认 TCP server 已经能接收 RESP 请求，并按 RESP 格式返回 simple string、bulk string、integer 和 close 响应。
