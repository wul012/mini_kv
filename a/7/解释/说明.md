# 第七版开发调试运行归档说明

第七版把 mini-kv 从「inline 文本协议」向 Redis 协议兼容方向推进了一步。本轮新增：

- `include/minikv/resp.hpp`：RESP parser 对外接口。
- `src/resp.cpp`：RESP array-of-bulk-strings 请求解析实现。
- `tests/resp_tests.cpp`：RESP parser 单元测试。

当前 TCP server 仍然使用一行一命令的 inline 文本协议；第七版先把 RESP parser 独立跑通，为下一版接入 TCP server 做准备。

本轮验证使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证时临时把 MinGW 运行时目录加入当前 PowerShell 进程的 `PATH`，避免测试程序因找不到运行时 DLL 报 `0xc0000135`。

## 01-cmake-configure.png

- 命令：`cmake -S . -B cmake-build-debug`
- 结果：配置成功，构建目录为 `cmake-build-debug`。
- 意义：确认第七版新增的 `src/resp.cpp`、`tests/resp_tests.cpp` 和 `minikv_resp_tests` 目标都能被 CMake 正确识别。

## 02-cmake-build.png

- 命令：`cmake --build cmake-build-debug`
- 结果：编译成功。
- 意义：确认 RESP parser 模块、测试目标以及原有 CLI/server/client/benchmark/test 目标都能通过 C++20 编译并链接。

## 03-ctest.png

- 命令：`ctest --test-dir cmake-build-debug --output-on-failure`
- 结果：6 个测试全部通过：
  - `store_tests`
  - `command_tests`
  - `wal_tests`
  - `snapshot_tests`
  - `stress_tests`
  - `resp_tests`
- 意义：确认第七版新增 RESP parser 后，原有 Store、CommandProcessor、WAL、Snapshot、Stress 行为未被破坏，并且新增 RESP 测试通过。

## 04-resp-tests-verbose.png

- 命令：`ctest --test-dir cmake-build-debug -R resp_tests --verbose`
- 结果：`resp_tests` 单项测试通过。
- 意义：单独验证 RESP parser 的核心行为：解析 array-of-bulk-strings、处理带空格 value、记录 consumed 字节数、支持 pipelined 输入切分、区分 incomplete 和 error，并能桥接到当前 inline `CommandProcessor`。

## 05-cli-regression-smoke.png

- 命令：通过管道向 `.\cmake-build-debug\minikv_cli.exe` 输入 inline 命令。
- 输入流程：`SET resp seventh-version`，`GET resp`，`QUIT`。
- 结果：出现 `OK inserted`，并能读回 `seventh-version`。
- 意义：确认第七版新增 RESP parser 不影响现有 CLI inline 协议运行。

## 进程与清理

- 第七版验证没有启动 TCP server 或其他长期后台进程。
- 本轮没有生成需要删除的临时快照、WAL 或日志文件。
- 归档保留 `a/7/图片/` 下 5 张截图和本说明文件。
