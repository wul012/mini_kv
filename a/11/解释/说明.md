# 第十一版开发调试运行归档说明

第十一版在第十版 graceful shutdown 和结构化日志基础上，补上 TCP 连接生命周期追踪和并发连接指标：

- `TcpServer` 新增 `bound_port()`，测试和嵌入式场景可以拿到端口 `0` 绑定后的真实端口。
- `TcpServer` 新增 `connection_stats()`，返回累计连接数、当前活跃连接数和历史峰值连接数。
- 每个 TCP 客户端连接分配递增 `connection_id`。
- `event=tcp_client_accepted` 日志新增 `connection_id`、`active_connections`、`total_connections`、`peak_connections`。
- 新增 `event=tcp_client_closed` 日志，客户端线程退出时减少活跃连接并记录关闭后的指标。
- `event=tcp_stop` 日志新增最终连接指标。
- `tcp_server_tests` 从只验证启动/停止，升级为真实连接一次 TCP 并验证响应、日志和指标。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

## 04-connection-metrics-test.png

- 命令：`.\cmake-build-debug\minikv_tcp_server_tests.exe`
- 结果：退出码为 0，结果为 `PASS`。
- 意义：这个测试会在临时端口启动 `TcpServer`，读取 `bound_port()`，用真实 TCP socket 发送 `PING` / `QUIT`，验证收到 `PONG` / `BYE`，再断言：
  - 出现 `event=tcp_client_accepted`
  - 出现 `event=tcp_client_closed`
  - 出现 `connection_id=1`
  - `total_connections=1`
  - `active_connections` 从 1 回到 0
  - `peak_connections=1`

## 05-concurrent-tcp-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6411 127.0.0.1`，再用 Python socket 建立两个重叠 TCP 连接。
- 输入流程：
  - 第一个连接发送 `PING` 并保持连接。
  - 第二个连接发送 `PING`、`QUIT` 后关闭。
  - 第一个连接再发送 `QUIT` 后关闭。
- 结果：
  - 两个连接都收到正确的 `PONG` / `BYE` 响应。
  - server stdout 出现 `connection_id=1` 和 `connection_id=2`。
  - 第二个连接接入时日志显示 `active_connections=2`、`total_connections=2`、`peak_connections=2`。
  - 两个连接关闭后日志显示 `active_connections=0`。
- 意义：确认第十一版真实 TCP server 可以追踪连接生命周期，并能记录并发连接峰值。
