# 第五十一版运行调试归档说明

本轮版本目标：

```text
按最新跨项目计划，为 mini-kv 写命令提供只读 execution-contract / CHECKJSON 响应。
它用于在真实写入前暴露 parser、side_effects、command_digest 和 WAL 相关检查。
```

本轮只做 `CHECKJSON` 只读执行契约，不新增事务，不新增集群，不让 mini-kv 承担订单、库存、支付的权威存储，也不开放 Node 对 mini-kv 的真实写执行入口。

# 5. 05-checkjson-smoke.png

对应流程：

```powershell
启动 minikv_server.exe 6414 127.0.0.1
使用 minikv_client.exe 发送：
  CHECKJSON SET orderops:1 value
  GET orderops:1
  CHECKJSON GET orderops:1
  QUIT
```

结果：

```text
SMOKE OK
CHECKJSON SET 返回 read_only=true、execution_allowed=false、write_command=true。
wal.enabled=false 时 durability=memory_only，并给出 wal disabled warning。
GET orderops:1 仍然返回 (nil)，证明 CHECKJSON SET 没有执行写入。
CHECKJSON GET 返回 write_command=false、durability=not_applicable。
```

意义：

```text
真实 server + client 链路证明 CHECKJSON 可以给 Node 后续版本当作执行前证据，同时不会改变 mini-kv 数据。
```
