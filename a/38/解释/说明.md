# v38 运行调试归档说明

本轮版本目标：

```text
为 bundled TCP client 增加可选持久化 key cache，让 key-oriented Tab 补全候选可以跨客户端会话延续。
```

v38 重点验证：

```text
ClientKeyCache 支持 add/remove/clear、去重、容量裁剪、文件加载和保存。
minikv_client 支持 --key-cache-file path。
客户端启动时可加载已有 key cache。
成功 SET 后保存 key，DEL 后移除 key，LOAD 后清空 key cache。
真实 server + 两个 client session 能验证跨会话 key cache。
自动化 heartbeat 已改成每 15 分钟检查一次。
代码讲解记录在开发调试同一轮内同步补齐。
```

## 05-key-cache-smoke.png

- 命令：真实启动 `minikv_server.exe`，再用同一个 `--key-cache-file tmp-v38-run/client.keys` 运行两个真实 `minikv_client.exe` 会话。
- session 1 输入：

```text
SET alpha one
SET alpine two
QUIT
```

- session 1 后 key cache：

```text
alpha
alpine
```

- session 2 输入：

```text
GET alpha
DEL alpha
QUIT
```

- 关键结果：
  - session 2 启动时输出 `2 keys available, 2 loaded`。
  - `GET alpha` 返回 `one`。
  - `DEL alpha` 返回 `1`。
  - session 2 后 key cache 只剩 `alpine`。

调试意义：

```text
单元测试覆盖 key cache 文件算法，真实 smoke 证明两个 client 会话之间能通过同一个文件延续 key 补全候选。
```
