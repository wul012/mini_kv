# 第二十三版开发调试运行归档说明

第二十三版在第二十二版 Snapshot 原子保存基础上，转向 WAL 恢复可靠性加固：新写入的 WAL 记录带 checksum，replay 时仍兼容旧版裸文本 WAL，并在 checksum 不匹配时跳过坏记录。

本版主要改动：

- 项目版本更新到 `0.23.0`。
- 新增 WAL2 记录格式：`WAL2 <16位checksum> <原始WAL记录>`。
- `WriteAheadLog::append()` 写入 WAL2 格式。
- checksum 使用 64-bit FNV-1a，对原始 WAL 记录文本计算。
- `WriteAheadLog::replay_with_report()` 兼容旧格式 `SET` / `DEL` / `EXPIREAT` 裸记录。
- `WriteAheadLog::replay_with_report()` 遇到 WAL2 checksum 不匹配时跳过该记录。
- `WalReplayReport` 新增 `checksum_failed_records`。
- CLI 启动 WAL replay 时输出 checksum failed 数量。
- server 启动 `event=wal_replay` 日志新增 `checksum_failed=` 字段。
- `wal_tests` 新增 WAL2 写入、checksum mismatch 跳过、旧格式兼容验证。
- README 更新为 Version 23，并补充 WAL2 checksum 说明。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`

## 05-cli-wal-checksum-smoke.png

- 命令：真实运行 2 次 `.\cmake-build-debug\minikv_cli.exe <wal_path>`。
- 第一次输入：
  - `SET good value`
  - `SET bad original`
  - `EXIT`
- 中间处理：
  - 确认 WAL 文件包含 `WAL2 ` 记录。
  - 把第二条 WAL2 payload 从 `SET bad original` 篡改为 `SET bad tampered`，但不更新 checksum。
  - 追加一条旧格式裸记录：`SET legacy old`。
- 第二次输入：
  - `GET good`
  - `GET bad`
  - `GET legacy`
  - `EXIT`
- 结果：
  - 第二次启动输出 `2 records replayed, 1 skipped, 0 truncated, 1 checksum failed`。
  - `GET good` 返回 `value`，说明未篡改 WAL2 正常恢复。
  - `GET bad` 返回 `(nil)`，说明篡改 WAL2 被跳过。
  - `GET legacy` 返回 `old`，说明旧格式裸 WAL 仍兼容。
  - 最终 WAL 文件保留 WAL2 记录和旧格式裸记录。
- 意义：确认新功能不只在单元测试可用，真实 CLI 启动 replay 也能同时处理 WAL2、checksum mismatch 和旧格式记录。
