# v30 运行调试归档说明

本轮版本目标：

```text
在 v29 STATS / HEALTH 的基础上，新增可选周期性 server metrics 结构化日志。
```

v30 重点验证：

```text
--metrics-interval-ms 可以开启周期性 server metrics。
默认不启用周期 metrics。
开启后 server 会输出 event=server_metrics。
metrics 日志包含 active_connections / total_connections / peak_connections。
metrics 日志包含 metrics_interval_ms，方便确认当前周期。
```

## 05-server-metrics-smoke.png

- 命令：真实启动 `minikv_server.exe`，再用真实 `minikv_client.exe` 连接。
- server 参数：

```text
--accept-poll-ms 25
--metrics-interval-ms 50
```

- client 输入流程：

```text
PING
STATS
HEALTH
QUIT
```

- 关键结果：
  - client 收到 `PONG`。
  - client 的 `STATS` 输出包含 `connection_stats_available=yes`。
  - client 的 `HEALTH` 输出包含 `OK live_keys=0`。
  - server stdout 出现 `event=server_metrics`。
  - server metrics 日志包含 `metrics_interval_ms=50`。
  - 验证结束后停止本轮启动的 server 进程。

意义：

```text
真实 server 运行时能够按配置周期输出结构化 metrics 日志。
```
