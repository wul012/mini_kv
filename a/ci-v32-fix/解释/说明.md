# v32 CI 问题修复归档说明

本轮目标：

```text
修复远端 v32 提交在 GitHub Actions 上 Linux / macOS / Windows 检查失败的问题。
```

定位结果：

```text
远端 master 停在 v32 提交 5be0e89。
GitHub Actions 的失败 run 为 CI / 25629695531。
Ubuntu 和 macOS 失败在 Test 步骤。
Docker 复现 v32 后，wal_tests 在 Release 配置下崩溃：
basic_string::replace: __pos == npos
```

根因：

```text
CI 原来使用 Release 构建。
Release 会定义 NDEBUG，导致 assert(...) 被编译移除。
wal_tests 里有 assert(wal.append(...))、assert(wal.repair(...)) 这类带副作用的断言。
这些表达式在 Release 下不会执行，WAL 文件没有写入，后续测试再去替换不存在的字符串就会崩溃。
```

修复内容：

```text
.github/workflows/ci.yml
 -> CI 改为 Debug 配置，让 assert 驱动的测试在 CI 中保持启用。

tests/wal_tests.cpp
 -> 把带副作用的 wal.append / wal.repair / wal.compact_if_recommended 先执行到 bool 变量，再 assert 结果。

src/tcp_server.cpp
 -> 对非 Windows 下为空实现的 NetworkRuntime 做 (void)runtime，消除 Linux unused variable warning。

README.md
 -> 补充说明 CI 使用 Debug，因为当前测试体系是 assert-driven。
```
