# 第四十七版运行调试归档说明

本轮版本目标：

```text
按最新跨项目计划新增 KEYSJSON [prefix]，让 mini-kv 提供结构化、只读、有上限的 key inventory。
```

本轮只做 `KEYSJSON`，不做 ACL、不做分页游标、不做 glob 查询、不读取 value、不改变现有 `KEYS` 文本响应、不接 Node。

# 5. 05-keysjson-smoke.png

对应流程：

```powershell
启动 minikv_server.exe 6410 127.0.0.1
使用 minikv_client.exe 发送：
  SET user:1 one
  SET user:2 two
  SET order:1 paid
  KEYSJSON user:
  QUIT
```

结果：

```text
SMOKE OK
KEYSJSON 能被 ConvertFrom-Json 解析；
prefix=user:；
key_count=2；
keys 包含 user:1 和 user:2；
truncated=false；
limit=1000。
```

意义：

```text
真实 server/client 链路证明 KEYSJSON 可以给外部控制面提供稳定 key inventory。
```
