# 第二十二版开发调试运行归档说明

第二十二版在第二十一版客户端持久化历史基础上，转向 Snapshot 持久化安全加固：把 `SnapshotFile::save()` 从直接截断写目标文件，改为同目录临时文件写完整后再替换目标快照。

本版主要改动：

- 项目版本更新到 `0.22.0`。
- `SnapshotFile::save()` 先写入同目录临时文件。
- 临时文件命名使用目标文件名加 `.tmp.`、时间戳和尝试序号。
- 写入、flush、close 任一步失败时返回 false，并由 RAII 清理临时文件。
- 写入完整后再替换目标 snapshot。
- Windows 下使用 `MoveFileExW(..., MOVEFILE_REPLACE_EXISTING | MOVEFILE_WRITE_THROUGH)` 替换目标文件。
- 非 Windows 下使用 `std::filesystem::rename()` 替换目标文件。
- `snapshot_tests` 新增同一路径二次保存、加载替换后内容、临时文件不残留验证。
- README 更新为 Version 22，并补充 atomic snapshot saves 说明。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证前已把 MinGW 运行时目录加入当前 PowerShell 进程的 `PATH`，避免测试程序因为找不到运行时 DLL 误报 `0xc0000135`。

## 05-cli-snapshot-smoke.png

- 命令：真实运行 3 次 `.\cmake-build-debug\minikv_cli.exe`。
- 第一次输入：
  - `SET name old`
  - `SAVE tmp-v22-run/snapshot-smoke/state.snap`
  - `EXIT`
- 第二次输入：
  - `SET name new`
  - `SET extra value`
  - `SAVE tmp-v22-run/snapshot-smoke/state.snap`
  - `EXIT`
- 第三次输入：
  - `LOAD tmp-v22-run/snapshot-smoke/state.snap`
  - `GET name`
  - `GET extra`
  - `EXIT`
- 结果：
  - 第一次 `SAVE` 返回 `OK saved 1`。
  - 第二次同路径 `SAVE` 返回 `OK saved 2`。
  - 第三次 `LOAD` 返回 `OK loaded 2`。
  - `GET name` 返回 `new`。
  - `GET extra` 返回 `value`。
  - 最终 snapshot 文件内容包含 `"name" "new"` 和 `"extra" "value"`。
  - smoke 目录没有 `.tmp.` 临时文件残留。
- 意义：确认新保存路径不只在单元测试里可用，真实 CLI 的 `SAVE` / `LOAD` 命令也能覆盖旧 snapshot 并加载新内容。
