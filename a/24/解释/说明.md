# 第二十四版开发调试运行归档说明

第二十四版在第二十三版 WAL2 checksum 的基础上，补上手动 WAL 压缩能力。核心目标是让长时间运行后的 WAL 可以被重写为“当前仍然存活的数据状态”，删除已经被覆盖、删除或过期的历史记录，减少后续启动 replay 成本。

本轮主要改动：

- 项目版本更新到 `0.24.0`。
- `WriteAheadLog` 新增 `compact(const Store&, std::size_t*)`。
- WAL compact 会从 `Store::snapshot_items()` 读取当前存活键值。
- compact 输出仍使用 `WAL2 <checksum> <record>` 格式。
- 有 TTL 的 key 会写成 `SET key value` 加 `EXPIREAT key epoch_millis`。
- compact 先写临时文件，再替换目标 WAL 文件。
- `CommandProcessor` 新增 `COMPACT` 命令。
- 无 WAL 启动时执行 `COMPACT` 返回 `ERR WAL not enabled`。
- WAL 命令路径增加互斥保护，避免 `COMPACT` 与同一命令层的 WAL 写入交错。
- `wal_tests` 增加 compact 后文件内容、replay 结果和 deleted key 不恢复的覆盖。
- `command_tests` 增加 `COMPACT` 参数错误、无 WAL 错误和 HELP 文本覆盖。
- README 更新为 Version 24，并补充 WAL compact 使用说明。

## 05-cli-wal-compact-smoke.png

- 命令：真实运行 2 次 `.\cmake-build-debug\minikv_cli.exe tmp-v24-run\compact-smoke.wal`。
- 第一次输入：
  - `SET keep value`
  - `SET phrase hello from compact`
  - `EXPIRE keep 60`
  - `SET gone remove`
  - `DEL gone`
  - `COMPACT`
  - `EXIT`
- 中间检查：
  - `COMPACT` 返回 `OK compacted 2`。
  - compact 后 WAL 文件只有 3 行。
  - 每一行均以 `WAL2 ` 开头。
  - 文件包含 `SET keep value`、`EXPIREAT keep ...`、`SET phrase hello from compact`。
  - 文件不再包含 `gone` 或 `DEL` 历史。
- 第二次输入：
  - `GET keep`
  - `GET phrase`
  - `GET gone`
  - `TTL keep`
  - `EXIT`
- 结果：
  - 启动 replay 输出 `3 records replayed, 0 skipped, 0 truncated, 0 checksum failed`。
  - `GET keep` 返回 `value`。
  - `GET phrase` 返回 `hello from compact`。
  - `GET gone` 返回 `(nil)`。
  - `TTL keep` 返回剩余 TTL。
- 意义：确认新功能不只在单元测试中可用，真实 CLI 启动、compact、落盘、重启 replay 的完整路径也通过。
