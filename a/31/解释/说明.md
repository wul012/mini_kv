# v31 运行调试归档说明

本轮版本目标：

```text
在 v30 周期 server metrics 的基础上，为 bundled TCP client 增加终端交互式行编辑能力。
```

v31 重点验证：

```text
LineEditorBuffer 支持插入、退格、Delete、Home/End 和左右移动。
LineEditorHistoryNavigator 支持 Up/Down 风格历史浏览，并能回到当前草稿。
minikv_client 在交互终端可使用 read_client_line。
stdin 被重定向时，minikv_client 仍保持原来的逐行输入行为，方便脚本和 smoke test。
真实 server + redirected client smoke 仍能执行 PING / SET / GET / :history / QUIT。
```

## 05-client-line-editor-smoke.png

- 命令：真实启动 `minikv_server.exe`，再用真实 `minikv_client.exe` 通过重定向 stdin 输入命令。
- client 输入流程：

```text
PING
SET lineedit ok
GET lineedit
:history
QUIT
```

- 关键结果：
  - client 收到 `PONG`。
  - `SET lineedit ok` 返回 `OK inserted`。
  - `GET lineedit` 返回 `ok`。
  - `:history` 输出 `PING`、`SET lineedit ok`、`GET lineedit`。
  - `--history-file` 保存了发送过的命令。
  - 验证结束后停止本轮启动的 server 进程。

意义：

```text
stdin 重定向时，v31 新增的 read_client_line 会回退到普通 getline，不破坏脚本和自动化 smoke。
```
