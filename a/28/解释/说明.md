# 第二十八版开发调试运行归档说明

第二十八版在第二十七版可选自动 WAL compact 的基础上，补上可配置阈值和运行期 compact 统计。核心目标是：让用户可以调节 compact 推荐/自动 compact 的触发条件，并通过 `WALINFO`、CLI stats、server `event=wal_stats` 看到 compact 次数、写回 key 数、移除记录数和节省字节数。

本轮主要改动：

- 项目版本更新到 `0.28.0`。
- 新增 `WalMaintenanceOptions`，包含：
  - `compact_min_records`
  - `compact_record_ratio`
  - `compact_min_bytes`
- 新增 `WalCompactionStats`，包含：
  - `manual_compactions`
  - `auto_compactions`
  - `repair_compactions`
  - `compacted_keys`
  - `records_removed`
  - `bytes_saved`
- `WalMaintenanceReport` 新增 options 和 compaction stats。
- `WriteAheadLog` 构造函数支持传入 WAL maintenance options。
- `COMPACT`、`--auto-compact-wal` 和 `--repair-wal` 会分别累计 manual / auto / repair compact 统计。
- CLI 新增：
  - `--wal-compact-min-records`
  - `--wal-compact-record-ratio`
  - `--wal-compact-min-bytes`
- server 新增同样三个 WAL compact 阈值参数。
- `WALINFO` 输出 compact 阈值和 compact 统计。
- README 更新为 Version 28。
- `wal_tests` 增加自定义阈值和 compact counters 覆盖。

## 04-wal-tests.png

- 命令：`.\cmake-build-debug\minikv_wal_tests.exe`
- 结果：退出码为 0，`minikv_wal_tests.exe: PASS`。
- 意义：单独复跑 WAL 测试，重点确认：
  - 默认 compact 阈值仍然是 `8 / 4 / 65536`。
  - 手动 `COMPACT` 会累计 `manual_compactions`。

  - repair 会累计 `repair_compactions`。
  - 自定义阈值 `4 / 2 / 1048576` 可以让第 4 次同 key 写入触发自动 compact。
  - `records_removed` 和 `bytes_saved` 会记录 compact 收益。

## 05-cli-custom-thresholds-smoke.png

- 命令：真实运行 `minikv_cli.exe <wal_path> --auto-compact-wal --wal-compact-min-records 4 --wal-compact-record-ratio 2 --wal-compact-min-bytes 1048576`。
- 输入：
  - 连续 4 次 `SET tuned valueN`
  - `WALINFO`
  - `GET tuned`
  - `COMPACT`
  - `WALINFO`
  - `EXIT`
- 关键结果：
  - 启动 stats 显示自定义阈值 `compact_min_records=4 compact_record_ratio=2 compact_min_bytes=1048576`。
  - 第一次 `WALINFO` 显示 `auto_compactions=1 records_removed=3 bytes_saved=120`。
  - `GET tuned` 返回 `value3`。
  - 手动 `COMPACT` 后第二次 `WALINFO` 显示 `manual_compactions=1 auto_compactions=1`。
  - 最终 WAL 文件只剩 1 条 WAL2 记录。

## 06-server-custom-thresholds-smoke.png

- 命令：真实启动 `minikv_server.exe <port> 127.0.0.1 <wal_path> --auto-compact-wal --wal-compact-min-records 4 --wal-compact-record-ratio 2 --wal-compact-min-bytes 1048576 --accept-poll-ms 25`，再用真实 `minikv_client.exe` 连接。
- 验证流程：
  - server 启动并输出 `event=wal_stats`。
  - client 连续 4 次执行 `SET srv valueN`。
  - client 执行 `WALINFO` 和 `GET srv`。

- 关键结果：

  - `GET srv` 返回 `value3`。
  - 最终 WAL 文件只剩 1 条 WAL2 记录。
