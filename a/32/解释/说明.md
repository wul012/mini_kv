# v32 运行调试归档说明

本轮版本目标：

```text
在 v31 终端行编辑和历史能力的基础上，为命令处理链路增加运行时命令计数。
```

v32 重点验证：

```text
CommandProcessorMetrics 记录 total_commands、successful_commands、error_commands。
CommandMetricsTracker 用原子计数器支撑单进程和 TCP 多连接共享统计。
空输入不计入命令统计。
ERR 开头的响应计入 error_commands，其他有效响应计入 successful_commands。
STATS 和 HEALTH 输出命令计数。
TCP server 周期性 server_metrics 日志和停止日志复用同一份命令计数。
代码讲解记录在开发调试同一轮内同步补齐。
```

## 06-command-metrics-smoke.png

- 命令：真实启动 `minikv_server.exe`，再用真实 `minikv_client.exe` 通过重定向 stdin 输入命令。
- client 输入流程：

```text
PING
BADCOMMAND
STATS
HEALTH
QUIT
```

- 关键结果：
  - client 收到 `PONG`。
  - `BADCOMMAND` 返回 `ERR unknown command`。
  - `STATS` 返回 `total_commands=2 successful_commands=1 error_commands=1`。
  - `HEALTH` 返回 `total_commands=3 successful_commands=2 error_commands=1`。
  - server 周期性 `event=server_metrics` 日志包含命令总数和错误数。
  - 验证结束后停止本轮启动的 server 进程。

调试备注：

```text
smoke 输入文件必须用 UTF-8 无 BOM 写入。
否则第一条 PING 前会带 BOM，服务端会把它识别成未知命令。
本轮最终 smoke 已使用无 BOM 输入重新验证通过。
```

另一个现象：

```text
smoke 脚本会先做一次 TCP readiness probe，所以 server 日志里的 total_connections 可能比真实 client 连接多 1。
这个探测连接不发送命令，因此不会影响 total_commands、successful_commands、error_commands。
```
