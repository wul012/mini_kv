# 第十二版开发调试运行归档说明

第十二版在第十一版连接生命周期追踪基础上，补上可配置 TCP 请求限制和 bundled client socket timeout：

- `TcpServer::Options` 新增 `max_request_bytes`，默认 `65536`。
- `serve_client` 不再使用硬编码 `64 * 1024`，改用 `options.max_request_bytes`。
- 请求缓冲超过上限时，服务端返回原有协议错误，并新增 `event=tcp_request_rejected` 结构化日志。
- `minikv_server` 新增 CLI 参数：`--max-request-bytes bytes` 和 `--accept-poll-ms ms`。
- `event=server_start` 和 `event=tcp_listen` 日志会输出当前请求上限。
- `minikv_client` 新增可选第三个参数 `timeout_ms`，用于设置 socket 接收和发送超时。
- `tcp_server_tests` 增加低请求上限场景，验证超限 inline 请求被拒绝并记录日志。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`

## 04-request-limit-test.png

- 命令：`.\cmake-build-debug\minikv_tcp_server_tests.exe`
- 结果：退出码为 0，结果为 `PASS`。
- 意义：这个测试会在临时端口启动 `TcpServer`，设置 `max_request_bytes=8`，先验证 `PING` / `QUIT` 正常，再发送 10 字节无换行 inline 请求，断言收到 `ERR line too long`，并检查：
  - `event=tcp_request_rejected`
  - `pending_bytes=10`
  - `max_request_bytes=8`
  - `total_connections=2`

## 05-limit-timeout-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6412 127.0.0.1 --max-request-bytes 8 --accept-poll-ms 50`。
- bundled client 验证：用 `.\cmake-build-debug\minikv_client.exe 127.0.0.1 6412 2000` 发送 `PING` / `QUIT`。
- over-limit 验证：用 Python socket 发送 10 字节无换行请求 `0123456789`。
- 结果：
  - client 输出 `socket timeout: 2000 ms`，并收到 `PONG` / `BYE`。
  - over-limit socket 收到 `ERR line too long\n`。
  - server stdout 出现 `max_request_bytes=8`、`accept_poll_ms=50`、`event=tcp_request_rejected`、`pending_bytes=10`、`reason=request_too_long`。
- 意义：确认第十二版真实 TCP server 能使用 CLI 配置请求上限，bundled client 能使用 socket timeout 参数，超限请求能被拒绝并记录结构化日志。
