## 04-tcp-resp-tests.png

- 命令：`.\cmake-build-debug\minikv_tcp_resp_tests.exe`
- 结果：退出码为 0，结果为 `PASS`。该测试程序成功时不输出正文，所以截图记录了命令和 PASS 结论。
- 意义：单独复跑新增测试，确认：
  - 测试内启动 `TcpServer` 并绑定临时端口。
  - 原始 socket 发送 pipelined RESP 请求。
  - 服务端连续处理 `PING` / `SET` / `GET` / `SIZE` / `QUIT`。
  - 响应精确匹配 `$8\r\nexternal\r\n+OK inserted\r\n$11\r\nresp-client\r\n:1\r\n+BYE\r\n`。
  - 连接统计满足 `total_connections=1`、`active_connections=0`、`peak_connections=1`。
  - 日志包含 listen、accepted、closed、stop 等关键事件。

## 05-external-resp-smoke.png

- 服务端命令：`.\cmake-build-debug\minikv_server.exe 6417 127.0.0.1 --accept-poll-ms 50`
- 外部客户端验证：用 Python 原始 socket 连接 `127.0.0.1:6417`，发送 RESP pipeline。
- 客户端请求：`PING external` / `SET name resp-client` / `GET name` / `SIZE` / `QUIT`。
- 结果：
  - 收到 RESP bulk string：`external`。
  - `SET` 返回 `+OK inserted`。
  - `GET name` 返回 `resp-client`。
  - `SIZE` 返回 `:1`。
  - `QUIT` 返回 `+BYE`。
  - 原始字节匹配结果为 `MATCH True`。
  - 服务端日志出现 `event=tcp_client_accepted` 和 `event=tcp_client_closed`。
- 意义：确认不只测试进程内可用，真实 `minikv_server.exe` 也能被外部 RESP 客户端按 pipeline 方式访问。
