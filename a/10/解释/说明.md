# 第十版开发调试运行归档说明

第十版在第九版 RESP 兼容加固之后，补上 TCP server 的运行管理能力：

- `TcpServer` 新增 `request_stop()` 和 `stop_requested()`。
- `TcpServer::Options` 新增 `should_stop` 外部停止条件、`accept_poll_interval` 轮询间隔和 `logger` 日志回调。
- TCP accept loop 改为可周期性醒来的 `select` + `accept` 模式，能观察停止请求后退出 `run()`。
- `server_main.cpp` 安装 `Ctrl+C` / `SIGTERM` 信号处理，只设置停止标记，不在信号处理函数里做复杂工作。
- 服务端日志改为结构化 key-value 风格，例如 `event=server_start host=127.0.0.1 port=6400`。
- 新增 `tcp_server_tests`，验证服务启动、请求停止、退出和结构化 stop 日志。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`
- MinGW 运行时：`D:\CLION\CLion 2025.1.1\bin\mingw\bin`

验证前已把 MinGW 运行时目录加入当前 PowerShell 进程的 `PATH`，避免测试程序因找不到运行时 DLL 误报 `0xc0000135`。

## 04-graceful-stop-test.png

- 命令：`.\cmake-build-debug\minikv_tcp_server_tests.exe`
- 结果：退出码为 0，结果为 `PASS`。
- 意义：这个测试会在临时端口启动 `TcpServer`，等待 `event=tcp_listen`，调用 `request_stop()`，等待 `run()` 线程退出，并断言产生 `event=tcp_stop`。这是第十版“可请求停止”能力的核心验证。

## 05-structured-tcp-smoke.png

- 命令：启动 `.\cmake-build-debug\minikv_server.exe 6400 127.0.0.1`，再分别用 bundled client 和 Python socket 验证 inline / RESP 请求。
- inline 输入流程：`PING`、`QUIT`。
- RESP 输入流程：RESP `PING`、RESP `QUIT`。
- 结果：
  - inline client 收到 `PONG` 和 `BYE`。
  - RESP socket 收到 `+PONG\r\n+BYE\r\n`。
  - server stdout 出现结构化日志：`event=server_start`、`event=server_hint`、`event=tcp_listen`、`event=tcp_client_accepted`。
- 意义：确认第十版真实 TCP server 仍然兼容 inline / RESP 协议，并能输出结构化生命周期日志。
