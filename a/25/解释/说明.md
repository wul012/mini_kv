# 第二十五版开发调试运行归档说明

第二十五版在第二十四版 WAL compact 的基础上，补上启动期 WAL repair 工具。核心目标是：当 WAL 里混有 checksum mismatch、坏行、尾部半条记录时，可以在 CLI 或 server 启动时用 `--repair-wal` 先 replay 可恢复记录，再把 WAL 重写为干净的短 WAL2 文件。

本轮主要改动：

- 项目版本更新到 `0.25.0`。
- `WalRepairReport` 新增，用于记录 repair 前 replay 的 applied / skipped / truncated / checksum_failed 和 repair 后 compacted key 数量。
- `WriteAheadLog` 新增 `repair(Store&, WalRepairReport*)`。
- `repair()` 会先调用 `replay_with_report()` 恢复可用记录，再调用 `compact()` 重写 WAL。
- `minikv_cli` 新增启动参数 `--repair-wal`。
- `minikv_server` 新增启动参数 `--repair-wal`。
- CLI repair 启动输出新增 `repaired to N keys`。
- server repair 启动日志新增 `event=wal_repair ... compacted_keys=N`。
- `wal_tests` 增加 repair 场景：checksum mismatch、坏行、尾部半条记录、旧格式裸记录、TTL 恢复和 repair 后干净 WAL replay。
- README 更新为 Version 25，并补充 CLI / server `--repair-wal` 使用说明。

本轮验证继续使用 CLion 捆绑工具链：

- CMake/CTest：`D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin`

## 05-cli-wal-repair-smoke.png

- 命令：真实运行 2 次 `.\cmake-build-debug\minikv_cli.exe tmp-v25-run\cli-repair-smoke.wal`。
- 第一次输入：
  - `SET good value`
  - `SET bad original`
  - `EXIT`
- 中间处理：
  - 把 `SET bad original` 篡改为 `SET bad tampered`，制造 checksum mismatch。
  - 追加旧格式裸记录 `SET legacy old`。
  - 追加坏行 `GARBAGE record`。
  - 追加无结尾换行的 `SET partial value`，制造尾部半条记录。
- 第二次启动：
  - 使用 `--repair-wal`。
  - 输入 `GET good`、`GET bad`、`GET legacy`、`EXIT`。
- 结果：
  - 启动输出 `2 records replayed, 3 skipped, 1 truncated, 1 checksum failed; repaired to 2 keys`。
  - `GET good` 返回 `value`。
  - `GET bad` 返回 `(nil)`。
  - `GET legacy` 返回 `old`。
  - repair 后 WAL 文件只有 2 行，且每行都是 `WAL2`。
  - repair 后 WAL 保留 `SET good value` 和 `SET legacy old`，删除坏 checksum、坏行和尾部半条记录。
- 意义：确认 CLI 用户入口可以真实修复 damaged WAL，并在同一次启动后提供修复后的数据。
