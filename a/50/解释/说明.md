```text
按最新跨项目计划在 EXPLAINJSON 响应中增加 schema_version、command_digest 和 side_effect_count。
```

# 5. 05-explainjson-digest-smoke.png

对应流程：

```powershell
启动 minikv_server.exe 6413 127.0.0.1
使用 minikv_client.exe 发送：
  EXPLAINJSON SET orderops:1 value
  EXPLAINJSON SET orderops:1 value
  GET orderops:1
  EXPLAINJSON RESETSTATS
  QUIT
```

结果：

```text
SMOKE OK
schema_version=1；
command_digest 符合 fnv1a64:<16位hex>；
重复 SET explain 的 digest 一致；
side_effect_count 与 side_effects 数组长度一致；
GET orderops:1 仍然返回 (nil)。
```

意义：

```text
真实 server/client 链路证明 EXPLAINJSON 输出可以作为稳定 evidence 被 Node 后续复核，而且 explain 仍然不执行被解释的写命令。
```
