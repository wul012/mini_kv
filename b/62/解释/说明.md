## 本版目标

新增 TTL token 恢复证据：

```text
fixtures/ttl-token/recovery-evidence.json
tests/ttl_token_recovery_tests.cpp
```

覆盖语义：

```text
未过期 SETEXAT replay -> token 恢复且 TTL 存在
已过期 SETEXAT replay -> token 不复活
已过期 snapshot entry load -> token 不复活
snapshot baseline + WAL delta -> restart 后合并恢复
compact 后 SET + EXPIREAT -> token 和 TTL 仍可恢复
```

## 边界

本版不做：

```text
不修改 Node
不修改 Java
不启动 Node 或 Java
不把 mini-kv 接进 Java 订单交易链路
不让 mini-kv 成为订单权威存储
不定义生产幂等归属、备份、留存或多租户策略
```

`SETNXEX` 仍是写命令；`recovery-evidence.json` 是只读证据文件。Node v161 可以消费本版 evidence 做 controlled idempotency drill runbook，但默认仍应只读或 dry-run。
