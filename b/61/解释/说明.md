## 本版目标

新增短 TTL token 原语：

```text
SETNXEX key seconds value
```

语义：

```text
第一次 claim -> 1
重复 live token -> 0
过期后再次 claim -> 1
TTL 必须为正数
成功 claim 在 WAL 中写 SETEXAT key epoch_millis value
```

```text
SETNXEX idem:token 1 reclaimed
GET idem:token
TTL idem:token
QUIT
```

结果：

```text
第一次 claim 返回 1
重复 claim 返回 0
GET 返回 claimed
过期后 reclaim 返回 1
GET 返回 reclaimed
WAL 中出现 SETEXAT idem:token ... claimed / reclaimed
server 已停止
```

## 边界

本版不做：

```text
不修改 Node
不修改 Java
不启动 Node 或 Java
不把 mini-kv 接进 Java 订单交易链路
不让 mini-kv 成为订单权威存储
不打开 UPSTREAM_ACTIONS_ENABLED=true
```

`SETNXEX` 是 mini-kv 内部 token primitive，不是订单幂等的最终生产实现。Node v160 可以消费本版 evidence 做 readiness review，但默认仍应只读或 dry-run。
