## 04-cli-boundaries.png

真实输入顺序：

```text
SMOKEJSON
CHECKJSON LOAD data/prod.snap
QUIT
```

原始输出由 Windows PowerShell `ConvertFrom-Json` 解析后再断言。`SMOKEJSON` 保持 `read_only=true`、`execution_allowed=false`、`restore_execution_allowed=false`、`order_authoritative=false`，且 write/admin/runtime-write 三个执行信号均为 false。`CHECKJSON` 正确识别 `LOAD` 是写命令，但自身仍是只读检查，`execution_allowed=false`、`wal.touches_wal=false`。`QUIT` 返回 `BYE`。

第一次摘要解析误用了 PowerShell 7 才支持的 `ConvertFrom-Json -Depth`，在本机 Windows PowerShell 5 参数解析阶段失败；随后移除该参数，对同一份原始 CLI 日志重跑并通过。这是校验脚本兼容性偏差，不是产品失败，最终归档只使用兼容命令的成功结果。
