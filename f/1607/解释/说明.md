# v1607 命令归档说明

本轮归档对应 mini-kv v1607 的最终收口状态。v1607 按用户新的粒度要求，不再拆成五个很小的版本，而是把“f 文件夹解释质量”“中文讲解三千字门槛”“截图目录必须有真实截图”“只读边界”和“CI 可诊断性”合并成一个足够厚的版本闭环。

## 01-cmake-configure-build.png

命令：`cmake -S . -B cmake-build-v1607-ci -G "MinGW Makefiles"`，随后执行 `cmake --build cmake-build-v1607-ci -- -j 4`。

结果：CMake 配置成功，构建成功。新增的 f-folder 解释质量收口模块、命令目录、fixture 测试、中文讲解字数测试、CI workflow 防卡配置以及 CLI 程序均已生成。

意义：证明 v1607 不是只改文档，而是完成了 C++ 证据模块、命令目录、测试目标和可运行二进制的构建闭环。配置阶段仍出现既有的 Windows object path 长度警告，但生成与构建均成功。

## 02-targeted-ctest.png

命令：定向 CTest，覆盖 `f_folder_explanation_quality_closeout`、`dependabot_config`、`command_catalog`、`shard_readiness` 以及 current fixture 兼容测试，并带上 `--timeout 120 --progress`。

结果：12/12 通过。

意义：证明新增 `SHARDFFOLDEREXPLANATIONQUALITYCLOSEOUTJSON` 已接入命令目录、SHARDJSON、当前 fixture、规则目录、检查目录、validation 失败路径和中文讲解三千字测试。walkthrough 测试已改为按 ASCII 文件名发现讲解文件，避免 Windows/MSVC Debug 下中文路径编码失败后触发 assert 卡住。

## 03-full-ctest.png

命令：`ctest --test-dir cmake-build-v1607-ci --output-on-failure --timeout 120 --progress`。

结果：334/334 通过。

意义：证明 v1607 没有破坏既有 WAL、TCP、restore、credential、archive、runtime receipt、command catalog 和各类只读证据测试。新增解释质量规则保持 read-only，不影响存储行为。`--progress` 与 `--timeout 120` 也让 CI 将来遇到单测卡住时能暴露最后执行到的测试，而不是长时间无日志挂起。

## 04-cli-smoke.png

命令：通过 UTF-8 无 BOM 输入文件重定向给 `minikv_cli.exe`，依次执行：

```text
SHARDFFOLDEREXPLANATIONQUALITYCLOSEOUTJSON
CHECKJSON SHARDFFOLDEREXPLANATIONQUALITYCLOSEOUTJSON
QUIT
```

结果：CLI smoke 标记全部通过，包括 contract、v1607 release、`minimumWalkthroughChineseCharacterCount=3000`、`fillerPaddingAllowed=false`、validation passed、CHECKJSON read-only/no execution/no write/no WAL 和正常 `BYE`。

意义：证明用户真实 CLI 入口可以读到 v1607 单大版本解释质量证据，且该证据不启动服务、不写 store、不触 WAL、不打开 router、不允许执行。PowerShell 直接管道给 native 程序时会污染首行输入，连 `PING` 都会被识别为 unknown；因此最终 smoke 使用 UTF-8 无 BOM 文件重定向，避免把 shell 输入编码问题误判成 mini-kv 行为问题。

## CI 修复说明

第一次和第二次远端 CI 都在 Windows `Test` 步骤长时间停住，Ubuntu 和 macOS 已通过，Windows 的 configure/build 也通过。原因判断为新增 walkthrough 测试直接拼中文目录路径，在 Windows/MSVC Debug 路径编码失败时可能触发 debug assert 等待，导致 CTest 无日志挂起。

本轮补了两层修复：

- `f_folder_explanation_quality_closeout_walkthrough_tests` 改为按 ASCII 文件名递归发现讲解文件，仍然实际读取中文讲解正文并统计中文字符数。
- `.github/workflows/ci.yml` 的 Test 步骤增加 `timeout-minutes: 30`，CTest 增加 `--timeout 120 --progress`，以后即使某条测试卡住，也会失败并暴露位置。
