# mini-kv v1616 命令归档说明

## 01-build-focused-logger-tests.png

对应命令：

```powershell
$cmake = 'D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\cmake.exe'
$ctest = 'D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\ctest.exe'
$env:PATH = 'D:\CLION\CLion 2025.1.1\bin\mingw\bin;' + $env:PATH
& $cmake --build cmake-build-v1616 --config Debug --parallel 4
& $ctest --test-dir cmake-build-v1616 -R '^(log_tests|cli_log_level_flag_smoke|server_log_level_invalid_smoke)$' --output-on-failure
```

截图意义：

- 证明 v1616 新增的 `src/log.cpp`、`include/minikv/log.hpp`、CLI/server 入口和 `tests/log_tests.cpp` 已经进入同一个 CMake 构建闭环。
- focused CTest 只跑 K3 直接相关测试：`log_tests` 保护 logger 级别解析、过滤和格式字段；`cli_log_level_flag_smoke` 保护 `minikv_cli --log-level debug` 能正常启动并从 stdin EOF 退出；`server_log_level_invalid_smoke` 保护 server 对非法 log level 快速拒绝。
- 这张截图是本版最小回归面证据，说明 K3 的核心行为不是只靠文档描述。

## 03-real-tcp-smoke-stderr-stdout.png

对应命令：

```powershell
.\cmake-build-v1616\minikv_server.exe 6617 127.0.0.1 --log-level info --accept-poll-ms 10 --metrics-interval-ms 1000
# 测试脚本通过 TCP 发送 PING，并在验证后停止临时 server 进程。
```

截图意义：

- 真实 TCP smoke 返回 `response=PONG`，说明 server 仍能处理实际请求。
- `stdout_has_server_start=True` 且 `stdout_has_tcp_runtime_log=False`，说明 server startup/hint 证据仍在 stdout，而 TCP listen/accepted/closed/metrics 运行日志没有污染 stdout。
- `stderr_has_timestamp=True`、`stderr_has_tcp_listen=True`、`stderr_has_tcp_client_accepted=True`，说明 `--log-level info` 下运行日志进入 stderr，并带有 K3 要求的 `ts=`、`level=`、`thread=` 字段。
- 临时 server 进程在验证结束后已停止，没有保留后台服务。

## 04-format-diff-check.png

对应命令：

```powershell
$clang = 'D:\CLION\CLion 2025.1.1\bin\clang\win\x64\bin\clang-format.exe'
& $clang --dry-run --Werror include\minikv\log.hpp src\log.cpp src\main.cpp src\server_main.cpp tests\log_tests.cpp
git diff --check
```

截图意义：

- `clang_format_exit=0`，说明 v1616 触碰的 C++/header/test 文件符合本仓库 clang-format 风格。
- `git_diff_check_exit=0`，说明没有 trailing whitespace 或 whitespace-error。
- Git 输出的 LF/CRLF warning 是 Windows 工作区的行尾提示，不是 `git diff --check` 失败项；本轮仍把它记录下来，避免把提示误判成代码质量错误。
