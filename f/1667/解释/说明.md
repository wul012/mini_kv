# mini-kv v1667 命令证据说明

## 范围与边界

v1667 处理一条真实的 MinGW CMake 对象路径警告。它只缩短一个私有构建 target，公开
CTest 名和 v1662 已冻结的长测试源文件路径保持不变；`src/`、`include/`、
`fixtures/release/` 均为零改动。本版新增的 277 项 baseline 和 197 分风险上限只约束
内部测试构建身份，不改变命令、fixture、WAL、snapshot、OSFS 或执行权限。

## 01-object-path-warning.png

- 命令：在空的 `cmake-build-v1667` 中用 CLion MinGW 执行 CMake configure，随后扫描
  configure 与 full-build 日志中的 `CMAKE_OBJECT_PATH_MAX`、`cannot be safely placed`、
  `object file directory`。
- 结果：三类警告共 0 条；长内部 target 数 278→277，最高 target+source 分数 200→197。
- 意义：修复的是形成警告的真实路径组合，不是提高上限或过滤 stderr；v1662 的 132 字符
  源文件名没有被偷偷重命名。

## 02-target-path-contract.png

- 命令：运行 `minikv_test_architecture_tests.exe`，并聚焦执行架构合同与完整公开 CTest 名。
- 结果：扫描 361 个测试源、344 个 helper 注册、277 个存量长 target，最高分 197；
  聚焦 CTest 2/2。合同内部依次验证新增长 target、stale baseline、198 分路径三条红灯。
- 意义：当前修复已经变成只减不增的门；公开
  `text_package_review_closeout_audit_validation_tests` 仍然可检索且只注册一次。

## 03-full-verification.png

- 命令：完整 `cmake --build cmake-build-v1667 --parallel 4`，随后完整运行 353 条 CTest。
- 结果：构建 exit 0，耗时 328.2 秒；CTest 353/353，失败 0，耗时 210.67 秒。
- 意义：验证完整 CMake 生成图与所有行为合同，聚焦目标没有被拿来替代最终收口。

## 04-cli-boundaries.png

- 命令：对本版新编译的 `minikv_cli.exe` 输入 `SMOKEJSON`、`CHECKJSON GET demo`、
  `HEALTH`、`QUIT`，用 PowerShell `ConvertFrom-Json` 解析两段 JSON。
- 结果：`read_only=true`、`execution_allowed=false`、`order_authoritative=false`、
  `write_command=false`、`wal.touches_wal=false`，HEALTH 与 BYE 正常。
- 意义：内部构建身份变化没有扩大 router、write、WAL、network、credential、restore 或
  order authority 边界；判断来自真实新二进制，不是冻结截图。
