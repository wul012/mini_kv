# v1634 命令归档说明

v1634 的主题是给 OSFS 课设层补上一级间接块寻址。它不改变 mini-kv 原有 KV、WAL、snapshot、TCP/RESP、治理 receipt 或 Node/Java 只读证据链；本目录只归档 OSFS 本版本的构建、测试和真实命令演示。构建过程中的原始日志、临时脚本和临时磁盘镜像保存在 `cmake-build-v1634/_evidence-v1634/`，属于任务中间产物，最终清理时会删除；这里保留的是压缩后的可读 PNG 证据。

## 01-configure-build.png

命令：`cmake -S . -B cmake-build-v1634 ...`，随后执行 `cmake --build cmake-build-v1634 -j 4`。

这张图证明本版用 CLion 捆绑的 CMake、Ninja、MinGW 工具链重新生成构建系统，并在最终测试前重新确认目标已处于最新状态。CMake 输出里的长路径 warning 是本仓库历史测试文件名过长导致的已知 Windows 本地提醒，不是 v1634 的新增失败；构建阶段返回 `ninja: no work to do`，说明截图归档前的源码已经和现有构建产物一致。

## 02-focused-osfs-tests.png

命令：`ctest --test-dir cmake-build-v1634 -R '^osfs_tests$|^osfs_cli_smoke$' --output-on-failure`。

这张图证明 OSFS 的聚焦测试 2/2 通过。`osfs_tests` 覆盖 v1634 新增的跨直接块到间接块边界写入、读回、重开镜像持久化、超新容量上限拒绝、删除后释放数据块和间接块本身；`osfs_cli_smoke` 继续证明 shell 层仍能按原来的 login/create/open/read/write/close 流程运行。本版没有用文档声明代替可执行断言。

## 03-full-ctest.png

命令：`ctest --test-dir cmake-build-v1634 --output-on-failure -j 4`。

这张图证明完整回归 344/344 通过。v1634 修改了磁盘格式版本、inode 存储布局和底层 block 分配/释放路径，理论上可能波及 OSFS 之外的静态库链接与测试构建；全量 CTest 用来确认 KV 命令、WAL、snapshot、RESP/TCP、只读 JSON 证据、治理 receipt、归档质量门和 OSFS 测试都没有被破坏。

## 04-indirect-cli-smoke.png

命令：`minikv_osfs --disk indirect-smoke.img --format --blocks 192 --script indirect-smoke-script.txt`。

这张图来自真实 `minikv_osfs` 命令，不是测试进程内部 mock。脚本以 Alice 登录，创建 `large`，写入 4204 字节，其中第 4096 字节之后必须进入一级间接块；随后从 offset 4090 读取 64 字节，输出包含 `D8|INDIRECT-BEGIN|`，证明直接块末尾和间接块开头之间的连续读没有断裂。最后删除文件并 `DIR` 为空，说明文件可删除，目录项和存储释放路径没有留下可见残留。

## 进程与临时文件

本版没有留下需要继续运行的后台服务。`cmake-build-v1634/_evidence-v1634/` 中的 `indirect-smoke-script.txt`、`indirect-smoke-output.txt`、`indirect-smoke.img` 和 CTest 日志是生成本说明截图的临时材料，不作为长期交付物保留。长期保留的材料是本目录四张 PNG、本说明、v1634 代码讲解、README/CHANGELOG/OSFS 指南更新、源代码和测试修改。
