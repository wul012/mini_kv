# v1624 运行归档说明

## 版本范围

v1624 是 Post-K6 之后的导览可信度加固版本：把 v1623 的通俗项目说明中最小输入输出故事接入真实 `Store` + `CommandProcessor` 路径，新增 `project_orientation_examples_tests`，并让 `project_orientation_doc_contract` 要求文档继续保留该可执行示例。

本版没有修改 Node、Java 或 aiproj；没有新增运行时命令、TCP 路由、写入路由、WAL 行为、snapshot 行为、managed audit 写入、credential 读取、raw endpoint、服务自动启动或执行授权。

截图由临时本地 HTML 证据页通过 Playwright MCP 捕获。截图时临时启动了只监听 `127.0.0.1:8764` 的静态 HTTP server；归档完成后按 cleanup gate 停止并删除临时 HTML/日志目录。

## 边界复核

`project_orientation_examples_tests` 先执行 `SET order:100 paid` 和 `GET order:100`，确认普通命令确实能写入和读取内存 Store；随后执行 `INFOJSON`，确认 `read_only=true`、`execution_allowed=false`、`order_authoritative=false` 仍然存在；再执行 `CHECKJSON LOAD data/prod.snap`，确认它只解释 `LOAD` 的风险，仍然声明 `execution_allowed=false`，并标出 `store_replace_from_snapshot`。

测试随后再次读取 `order:100` 并确认仍为 `paid`，这一步证明 `CHECKJSON LOAD` 没有真正执行 snapshot load，没有替换 Store，也没有触碰 WAL。最后 `SMOKEJSON` 继续证明 forbidden commands、no auto-start、no managed audit write、no restore/compact execution 等只读边界仍然存在。
