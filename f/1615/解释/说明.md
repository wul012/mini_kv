# mini-kv v1615 命令归档说明

本目录记录 v1615 post-K2 review GitHub Actions runtime advisory closure 的关键证据。本版只更新 mini-kv 自身 GitHub Actions workflow 的 action tag，并用 CTest 合同测试防止旧 Node 20 runtime action tag 回归；不新增运行时命令，不改 WAL / snapshot / TCP / RESP，不改 K2 coverage floor，不启动长期服务，不读取 credential，也不修改 Node 或 Java 仓库。

## 01-action-release-verification.png

- 命令：
  - `gh api repos/actions/checkout/releases/latest --jq '{tag_name, name, html_url}'`
  - `gh api repos/actions/upload-artifact/releases/latest --jq '{tag_name, name, html_url}'`
- 结果：
  - `actions/checkout` 最新 release 为 `v6.0.3`。
  - `actions/upload-artifact` 最新 release 为 `v7.0.1`。
- 意义：本版 action tag 不是凭记忆猜测，而是来自 GitHub API 的官方 release 查询，用来收掉 K2 review 后提出的 Node 20 action runtime deprecation 时间敏感提醒。

## 02-action-tag-scan.png

- 命令：对 `.github/workflows/ci.yml` 和 `tests/dependabot_config_tests.cpp` 执行 action tag 扫描。
- 结果：
  - workflow 中 4 个 checkout step 全部为 `actions/checkout@v6.0.3`。
  - coverage artifact upload step 为 `actions/upload-artifact@v7.0.1`。
  - CTest 合同测试断言新 tag 存在，并断言 `actions/checkout@v4` / `actions/upload-artifact@v4` 不存在。
- 意义：升级覆盖 clang-format、三平台构建、sanitizer、coverage 四条 checkout 路径，且不会遗漏 coverage artifact 上传路径。

## 03-build-focused-test.png

- 命令：
  - `cmake -S . -B cmake-build-v1615 -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug`
  - `cmake --build cmake-build-v1615 --config Debug --parallel 4`
  - `ctest --test-dir cmake-build-v1615 -C Debug -R dependabot_config_tests --output-on-failure --timeout 120 --progress`
- 结果：构建完成，focused `dependabot_config_tests` 1/1 通过。
- 说明：第一次完整构建因为本地超时限制没有即时回传最终输出，但子进程继续运行并自然结束；随后执行短 build 复查确认全部目标已构建完成。
