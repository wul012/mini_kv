# mini-kv v1613 命令归档说明

## 本版范围

v1613 是 production-excellence K2 coverage report hardening。v1612 的 Ubuntu coverage job 已经完成配置、构建和 334 条 coverage CTest，但 `gcovr 7.0` 在 `src/command.cpp` 的 GCC/gcov negative branch-hit 输出上失败。本版按 `gcovr` 日志建议加入 `--gcov-ignore-parse-errors negative_hits.warn_once_per_file`，并用 CTest 保护这个参数。

本版不新增运行时命令，不修改 store、WAL、snapshot、RESP、TCP 或 fixture 行为，不打开 router、write、WAL 或 execution 权限。coverage job 仍然是 required，不使用 `continue-on-error`。

## 截图清单

### 01-remote-coverage-negative-hits-failure.png

命令：

```powershell
gh run view 27407287210 --repo wul012/mini_kv --job 80999471334 --log-failed
```

结果：v1612 远端 coverage job 在 `Generate core coverage report` 失败，核心错误为 `NegativeHits: Got negative hit value ... branch 7 taken -2`，日志建议使用 `--gcov-ignore-parse-errors negative_hits.warn_once_per_file`。

意义：这证明失败来自 gcov/gcovr 解析层，不是 CMake、构建或 CTest 失败。v1613 的修复范围因此只处理 report parser policy，不削弱 coverage job。

### 02-default-configure-build.png

命令：

```powershell

& 'D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\cmake.exe' -S . -B cmake-build-v1613 -G 'MinGW Makefiles' -DCMAKE_BUILD_TYPE=Debug
& 'D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\cmake.exe' --build cmake-build-v1613 -- -j 4
```

结果：默认 Debug configure/build 成功。构建保留既有长 target 的 CMake path warning，但没有构建失败。

意义：这证明 v1613 只影响 CI coverage report 命令，不影响默认 C++ 构建路径。

### 03-focused-ci-contract-test.png

命令：


结果：`dependabot_config_tests` 1/1 通过。

意义：这条测试确认 CI workflow 仍包含 required coverage job、core filter、artifact 上传，并且显式保护 `--gcov-ignore-parse-errors negative_hits.warn_once_per_file`。

## 下一步

推送 v1613 后，需要等待 GitHub Actions 的 `ubuntu coverage` job 生成 `mini-kv-core-coverage` artifact。若 report 产出成功，下一版读取真实 baseline，设置 `gcovr --fail-under-line` 为 baseline minus 2 points，并将 K2 标为 completed。那时才是 Claude review 的正确暂停点。
