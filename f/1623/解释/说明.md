# v1623 项目导览说明归档

本版主题是把用户要求的 mini-kv 通俗解释沉淀为可维护项目文档，并用 CTest 防止后续维护时误删关键机理说明。它不是新业务功能版，也不是运行时 evidence 字段扩展版。

本版新增/修改的核心内容：

- `项目通俗说明/mini-kv-项目机制与价值说明.md`：用中文解释 mini-kv 的目标、价值、输入输出、WAL、Snapshot、TCP/RESP、只读证据命令、`CHECKJSON` / `SMOKEJSON` 边界和 Node/Java 消费模型。
- `项目通俗说明/README.md`：作为新文件夹索引，说明该目录只放通俗导览，不替代 README、CHANGELOG 或运行归档。
- `cmake/check_project_orientation_doc.cmake`：新增 CMake 脚本型 CTest，读取说明文档并断言 `flowchart LR`、`sequenceDiagram`、`CommandProcessor`、`WAL`、`Snapshot`、`TCP / RESP`、`CHECKJSON LOAD`、`SMOKEJSON`、`read_only`、`execution_allowed`、`order_authoritative`、`managed audit`、`Node / Java` 等关键片段仍然存在。
- `README.md`、`docs/CHANGELOG.md`、`START_HERE.md`、`docs/production-excellence-progress.md`：记录 v1623 的导览维护范围。

## 边界说明

本版没有新增或打开任何运行时执行能力：

- 不新增 WAL 写路径。
- 不新增 Snapshot/restore 执行入口。
- 不新增 active shard router。
- 不新增 write routing。
- 不读取 credential value。
- 不解析 raw endpoint URL。
- 不打开 managed audit connection。
- 不启动 Node/Java/mini-kv 上游进程。
- 不执行 `LOAD`、`COMPACT`、`RESTORE`、`SETNXEX`。

本版的价值是维护面：让新读者能更快理解 mini-kv，同时让这份解释被测试保护，避免未来只剩散乱版本流水账而失去项目机理导览。
