# v1622 质量收口归档说明

本版不是新业务功能版，而是 v1621 production-excellence K6 的质量补救版。目标是把快速巡检发现的三类问题一次收口：远端 CI 的 `clang-format changed files` 红点、`git show --check v1621` 报出的三个 Markdown EOF 空行问题、以及本地 full CTest 中 `cli_log_level_flag_smoke` 依赖平台 stdin 行为导致的 337/338 豁免。

本版没有新增 WAL、snapshot、restore、router、write routing、credential、raw endpoint、Node/Java runtime execution 相关能力。TCP idle/command timeout 行为沿用 v1621，只做格式和测试闭环修复。

## 截图 01：format and whitespace

文件：`../图片/01-format-and-whitespace.png`

对应命令：

```powershell
D:\CLION\CLion 2025.1.1\bin\clang\win\x64\bin\clang-format.exe --dry-run --Werror include\minikv\tcp_server.hpp src\tcp_server.cpp tests\tcp_server_timeout_limit_tests.cpp tests\tcp_test_support.hpp
git diff --check
```

结果意义：

- changed-file clang-format dry-run 退出码为 0，覆盖 v1621 远端 CI 失败的四个 C++ 文件。
- `git diff --check` 退出码为 0，说明本次工作区没有 whitespace error。
- 输出中只保留 Git 的 CRLF 提示，它不是 `--check` 错误，不影响提交质量。
