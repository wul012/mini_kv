# mini-kv v1610 命令归档说明

本目录记录 production-excellence K1 sanitizer 诊断引导版本的关键验证截图。v1610 是工程质量版本，不新增运行时命令，不移动历史归档，不修改 release fixture，不打开 router，不写 store，不触 WAL，不授予 execution authority。

## 01-cmake-configure.png

对应命令：

```powershell
$cmake -S . -B cmake-build-v1610 -G 'MinGW Makefiles' -DCMAKE_BUILD_TYPE=Debug
```

结果：默认 Debug CMake 配置和生成通过，仍只出现既有 Windows 长路径 warning。

意义：证明 `MINIKV_SANITIZE` 默认关闭时不影响普通本地构建，也不改变 `MINIKV_CURRENT_ARCHIVE_VERSION` 的 frozen runtime fixture 输出路径。

## 02-cmake-build.png

对应命令：

```powershell
$cmake --build cmake-build-v1610 -- -j 4
```

结果：Debug MinGW 构建完成，`minikv` 库、CLI/server/client 和全部 CTest 可执行目标完成编译链接。

意义：证明新增 CMake 选项和 CI YAML 断言没有破坏默认目标集合。

## 03-focused-ctest.png

对应命令：

```powershell
$ctest --test-dir cmake-build-v1610 -R 'dependabot_config_tests|command_tests|smokejson_command_receipt_tests|runtime_smoke_evidence_tests|metrics_file_tests' --output-on-failure --timeout 120 --progress
```

结果：5/5 通过。

意义：focused tests 覆盖本版最相关的风险点：CI 配置守卫、命令路径、runtime smoke evidence、metrics 相关目标和默认运行时只读边界。

## 04-full-ctest.png

对应命令：

```powershell
$ctest --test-dir cmake-build-v1610 --output-on-failure --timeout 120 --progress
```

结果：334/334 通过，0 失败。

意义：全量 CTest 证明 v1610 没有破坏命令行为、fixture、WAL、snapshot、RESP、TCP、恢复证据和治理 receipt 边界。

## 05-sanitizer-configure-local-limit.png

对应命令：

```powershell
$cmake -S . -B cmake-build-v1610-sanitize -G 'MinGW Makefiles' -DCMAKE_BUILD_TYPE=Debug -DMINIKV_SANITIZE=ON
```

结果：Windows MinGW sanitizer 配置在 CMake 阶段快速失败，错误说明 CLion 捆绑 MinGW 不提供 `-lasan`/`-lubsan`，应使用 Ubuntu sanitizer CI job。

意义：证明本机 sanitizer runtime 限制已被收敛成可解释的配置期反馈，避免维护者在长构建末尾才遇到 linker failure。
