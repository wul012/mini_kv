# mini-kv v1619 命令归档说明

本轮归档对应 production-excellence K4 的 shard-readiness 拆分收尾。版本主题是把 `src/shard_readiness.cpp` 从巨大聚合入口拆成公开 fixture-path shim、SHARDJSON formatter、digest 证据链 helper、boundary/catalog helper 和内部连接头，同时保持 `SHARDJSON`、`CHECKJSON SHARDJSON`、CLI/TCP 行为、fixture parity、只读边界、no router、no write、no WAL、no execution 全部不变。

## 截图清单

1. `01-cmake-configure.png`
   - 命令：`cmake -S . -B cmake-build-v1619 -G Ninja -DCMAKE_BUILD_TYPE=Debug`
   - 意义：证明拆分后的 CMake source list 可以配置成功，新 shard-readiness translation units 被纳入 `minikv` 目标。

2. `02-build.png`
   - 命令：`cmake --build cmake-build-v1619 --config Debug --parallel 4`
   - 意义：证明 public shim、internal helper、digest module、boundary validators、JSON formatter 之间的符号和 include 关系正确，拆分后可以完整链接 CLI/server/tests。

3. `03-focused-tests.png`
   - 命令：`ctest --test-dir cmake-build-v1619 -C Debug --output-on-failure --timeout 120 -R "shard_readiness_tests|shard_readiness_post_closeout_continuity_tests|command_tests|readonly_fixture_tests|runtime_smoke_evidence_tests|tcp_resp_tests"`
   - 意义：盯住 K4 直接风险，证明 `SHARDJSON` fixture parity、post-closeout continuity、command catalog、read-only fixture、runtime smoke 和 TCP/RESP 入口没有因为拆分漂移。

4. `04-full-ctest.png`
   - 命令：`ctest --test-dir cmake-build-v1619 -C Debug --output-on-failure --timeout 120 --progress`
   - 意义：证明全量 337 条 CTest 仍然通过，WAL、snapshot、RESP、runtime evidence、治理证据链和历史 fixture 没有被链接拆分误伤。

5. `05-format-diff-check.png`
   - 命令：`clang-format --dry-run --Werror ...` 与 `git diff --check`
   - 意义：证明本轮新增/修改 shard-readiness C++ 文件满足 changed-file formatting 约束，文档和源码没有 trailing whitespace；同时记录 Git 的 line-ending 提醒仅为本机 CRLF 工作区提示，不是 diff check 失败。

6. `06-cli-tcp-smoke.png`
   - 命令：真实 CLI 执行 `SHARDJSON`、`CHECKJSON SHARDJSON`、`SHARDROUTE sample-key`、`COMMANDSJSON`；真实 TCP server 执行 `PING`、`CHECKJSON SHARDJSON`、`SHARDROUTEJSON tcp-key-v1619`、`QUIT`
   - 意义：证明拆分不只在单元测试里成立，CLI 和 TCP 两条用户入口仍能读到只读 shard readiness / route preview 证据，且 `read_only=true`、`execution_allowed=false`、no router/no write/no WAL/no execution 边界仍然可见。

## 边界说明

- 本版不启动 Java、Node 或其他 sibling service。
- 本版不读取 Node 运行时输出、不扫描 sibling archive、不移动历史 `a/` 到 `f/` 归档根。
- 本版不安装 active shard router、不打开 write routing、不执行 restore/load/compact、不触发 WAL 写入。
- TCP smoke 启动的 `minikv_server.exe` 仅用于本轮验证，验证结束后已停止。
- `src/shard_readiness_boundary_field_names.cpp` 未在 K4 内转换为新表结构；该文件作为 E9 后续维护迁移处理，未来若改动必须单独证明 fixture byte-for-byte parity。
