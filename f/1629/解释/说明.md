# mini-kv v1629 OSFS 课程设计层命令归档说明

本归档记录 v1629 新增 OSFS 用户态二级文件系统模拟层的关键验证命令。截图通过 Playwright MCP 打开本地临时 HTTP 渲染页后生成，最终只保留 PNG 证据；原始日志和 HTML 渲染页属于本轮临时材料，收尾时删除。

## 01-cmake-configure.png

- 命令：`cmake -S . -B cmake-build-v1629 -G "MinGW Makefiles"`
- 结果：配置与生成完成，构建目录为 `D:/C/mini-kv/cmake-build-v1629`。
- 意义：证明新增 `minikv_osfs`、`osfs_tests`、`osfs_cli_smoke` 和 CMake smoke 脚本能进入项目构建图。
- 备注：本机仍有既有长测试目标对象路径 warning，属于历史长文件名问题；全量构建与全量 CTest 已通过，未构成本版阻塞。

## 02-build-osfs-targets.png

- 命令：`cmake --build cmake-build-v1629 --target minikv_osfs minikv_osfs_tests -j 4`
- 结果：`minikv_osfs` 与 `minikv_osfs_tests` 均构建成功。
- 意义：证明 OSFS 可执行入口和 API 测试目标能独立构建，不依赖启动 `minikv_server`，也不改变原 KV CLI/TCP 运行入口。

## 03-osfs-focused-tests.png

- 命令：`ctest --test-dir cmake-build-v1629 -R "^osfs_(tests|cli_smoke)$" --output-on-failure`
- 结果：`osfs_tests` 与 `osfs_cli_smoke` 2/2 通过。
- 意义：`osfs_tests` 验证 superblock、bitmap、inode、目录项、权限、持久化重开和句柄式 read/write；`osfs_cli_smoke` 验证真实 `minikv_osfs` 脚本演示路径。

## 04-osfs-cli-smoke.png

- 命令：将脚本输入管道传给 `minikv_osfs --disk <Temp中文路径> --format --blocks 96`。
- 结果：程序在 `C:\Users\吴林东\AppData\Local\Temp\...` 中文路径下成功创建磁盘镜像，完成 `CREATE`、`OPEN`、`WRITE`、`READ`、`CHMOD`、`LOGIN`、权限拒绝和 `DIR` 输出。
- 意义：证明 v1629 的 Windows `wmain`/`-municode` 处理确实修复了中文用户名路径问题；目录行 `report 1 15 0600 0 27` 对应课程要求里的文件名、物理地址、保护码和文件长度。

## 05-full-ctest-summary.png

- 命令：`ctest --test-dir cmake-build-v1629 --output-on-failure`
- 结果：全量 344/344 CTest 通过。
- 意义：证明新增 OSFS 没有破坏原 KV store、command、WAL、snapshot、RESP、TCP、runtime receipt、readonly fixture 和治理质量测试。v1629 是独立课程模块，不污染原 mini-kv 主线。
