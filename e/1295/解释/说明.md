## 04-cli-tcp-smoke.png

对应命令：

```powershell
"CHECKJSON SHARDROUTEPRODUCTIONLIVECAPTURERELEASEEVIDENCEREVIEWNONPARTICIPATIONJSON" | .\cmake-build-v1295\minikv_cli.exe
.\cmake-build-v1295\minikv_server.exe 6404 127.0.0.1 tmp-v1295-smoke3.wal
"CHECKJSON SHARDROUTEPRODUCTIONLIVECAPTURERELEASEEVIDENCEREVIEWNONPARTICIPATIONJSON" | .\cmake-build-v1295\minikv_client.exe 127.0.0.1 6404 5000
```

结果：

- CLI 输出 `read_only:true`、`execution_allowed:false`、`write_command:false`、`touches_wal:false`、`append_when_enabled:false`。
- TCP 输出连接到 `127.0.0.1:6404`，在 WAL enabled 的服务端上下文中仍然输出 `touches_wal:false`、`append_when_enabled:false`。
- 本轮 smoke 启动的服务进程 PID 为 `29868`，验证结束后已停止，截图中记录 `SERVER_STOPPED=true`。

意义：

- 证明新命令不仅在单元测试中只读，在真实 CLI 和 TCP 输入路径下也保持 no execution、no write、no WAL append。
- TCP smoke 使用临时 WAL 仅用于服务端上下文验证，后续清理阶段删除临时 WAL 和 stdout/stderr log。
