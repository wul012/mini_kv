# mini-kv v150 命令归档说明

本版归档对象是 `SHARDJSON` live-read gate prerequisite evidence。v150 不打开 live-read gate，不启动长期服务，不运行 runtime probe，不安装 active router，不创建分片目录，不写入存储，也不执行 `LOAD`、`COMPACT`、`RESTORE` 或 `SETNXEX`。它只把 v149 `consumerHandoff` 证据冻结成 `fixtures/release/shard-readiness-v149.json`，再在 rolling current 中新增 `liveReadGatePlan`，记录后续如需 live read 必须先具备的 service owner、port list、smoke target、fail-closed 行为和 cleanup 责任。

截图使用 Playwright MCP 生成。由于 MCP 不允许直接打开 `file://` 页面，本轮临时启动了一个只读本地 HTTP 页面服务承载日志渲染页；截图完成后已关闭该服务。

## 截图

- `01-cmake-configure.png`
  - 命令：`cmake -S . -B cmake-build-v150 -G Ninja`
  - 结果：CMake 配置和生成成功。
  - 意义：证明 v150 构建树可以识别新增的 live-read gate 前置计划字段和版本化 v149 fixture。
- `02-target-build-ctest.png`
  - 命令：构建 `minikv_shard_readiness_tests`、`minikv_command_tests`，随后运行 `ctest -R '^(command_tests|shard_readiness_tests)$'`。
  - 结果：2/2 通过。
  - 意义：证明 current v150 fixture 等于 runtime 输出，v149 frozen fixture 被保留，`liveReadGatePlan` 明确 no live-read gate、no runtime probe、no service start、no router、no write、no execution。
- `03-full-build-ctest.png`
  - 命令：全量 `cmake --build cmake-build-v150` 和全量 `ctest --output-on-failure`。
  - 结果：71/71 通过。
  - 意义：证明 v150 没有破坏既有存储、WAL、snapshot、restore、TCP/RESP、runtime evidence 和 credential resolver non-participation 证据链。
- `04-tcp-shardjson-live-read-gate-plan-smoke.png`
  - 命令：临时启动 `minikv_server.exe 6420 127.0.0.1`，用真实 `minikv_client.exe` 读取 `HEALTH`、`COMMANDSJSON`、`EXPLAINJSON SHARDJSON`、`CHECKJSON SHARDJSON extra`、`SHARDJSON`、`QUIT`。
  - 结果：真实 TCP 输出包含 `releaseVersion=v150`、`status=live-read-gate-prerequisite-read-only`、`previousConsumedFixturePath=fixtures/release/shard-readiness-v149.json`、`consumerHandoff.frozenReleaseVersion=v149`、`liveReadGatePlan.planMode=service-lifecycle-prerequisite-only`、`liveReadGateAllowed=false`、`runtimeProbeAllowed=false`、`startsServices=false`、`requiresServiceOwner=true`、`requiresPortList=true`、`requiresSmokeTarget=true`、`requiresCleanup=true`、`routerActivationAllowed=false`、`writeRoutingAllowed=false`、`executionAllowed=false`、`writeCommandsAllowed=false` 和 `activeRouterInstalled=false`。服务端验证后已停止，`6420` 没有 LISTENING 残留。
  - 意义：证明 v150 通过真实命令路径暴露 live-read gate 前置计划，但没有把计划误变成运行授权。

## 边界

v150 仍是只读 evidence release。`liveReadGatePlan` 是给后续计划或消费端看的前置条件清单，不是启动入口：后续如需 live read，仍必须先有独立计划写清服务 owner、启动命令、端口、只读 smoke 目标、fail-closed 行为和停止清理责任。mini-kv 不读取或修改 Node 归档，不启动 Java/Node，不成为 Java order authority，也不承担 audit authority。
