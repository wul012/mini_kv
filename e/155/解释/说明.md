# mini-kv v155 命令归档说明

本版归档对象是 `SHARDJSON` 的 runtime execution approval-gate input precheck。v155 先把 v154 的 mini-kv-side runtime execution artifact candidate 冻结为 `fixtures/release/shard-readiness-v154.json`，再在 rolling current 中声明 Node v397-v399 之后仍缺少三项 approval-gate input：Node-approved runtime window、correlated operator approval record、complete cross-project runtime execution packet。

这不是 runtime approval，也不是 runtime execution packet。current evidence 仍保持 `approvalGateInputCount=0`、`missingApprovalGateInputCount=3`、`acceptedRuntimeExecutionArtifactCount=0`、`missingRuntimeExecutionArtifactCount=6`、`runtimeExecutionPacketExecutable=false`、`startsJavaService=false`、`startsMiniKvService=false`、`runtimeProbeAllowed=false`、`liveReadAllowed=false`、`routerActivationAllowed=false`、`writeRoutingAllowed=false`、`executionAllowed=false`。归档截图由 Playwright MCP 从临时只读 HTTP 渲染页生成；截图完成后该临时 HTTP 服务已关闭。

## 截图

- `01-cmake-configure-generate-fixture.png`
  - 命令：使用 CLion CMake 工具链配置 `cmake-build-v155`，构建 `minikv_cli`，再用 stdin 执行 `SHARDJSON` / `QUIT` 生成 `fixtures/release/shard-readiness.json`。
  - 结果：current fixture 更新为 `releaseVersion=v155`、`status=runtime-execution-approval-gate-input-precheck-blocked-read-only`、`evidenceDigest=fnv1a64:edb1c4d8534461ab`；v154 frozen fixture 保留 `status=mini-kv-runtime-execution-artifact-candidate-no-runtime-read-only`、`evidenceDigest=fnv1a64:edbe9c546c0d780a`。
  - 意义：证明 v155 current 由真实 CLI 命令路径生成，v154 candidate 被版本化冻结，不依赖 rolling current 作为历史 baseline。
- `02-target-build-ctest.png`
  - 命令：构建 `minikv_shard_readiness_tests`、`minikv_command_tests`，随后运行 `ctest -R '^(command_tests|shard_readiness_tests)$'`。
  - 结果：2/2 通过。
  - 意义：证明 current v155、v154 frozen candidate、`miniKvRuntimeExecutionArtifactCandidateFreeze` 和 `runtimeExecutionApprovalGateInputPrecheck` 的关键边界字段都通过定向断言。

  - 结果：71/71 通过。
  - 意义：证明新增 approval-gate precheck 没有破坏既有存储、WAL、snapshot、restore、TCP/RESP、runtime evidence 和 credential resolver non-participation 证据链。
- `04-tcp-shardjson-approval-gate-input-precheck-smoke.png`

  - 结果：真实 TCP 输出包含 `releaseVersion=v155`、`status=runtime-execution-approval-gate-input-precheck-blocked-read-only`、`previousConsumedFixturePath=fixtures/release/shard-readiness-v154.json`、`miniKvRuntimeExecutionArtifactCandidateFreeze.frozenReleaseVersion=v154`、`runtimeExecutionApprovalGateInputPrecheck.precheckMode=blocked-missing-approval-gate-inputs`、`approvalGateInputCount=0`、`missingApprovalGateInputCount=3`，并保持 runtime packet、runtime probe、live read、router、write、execution 全部为 false。验证服务只运行在临时验证端口 `6426`，候选端口 `6424` 全程无 LISTENING socket；验证结束后 `6426` 也无 LISTENING socket。
  - 意义：证明 approval-gate precheck 通过真实 TCP 只读路径暴露，但没有启动 candidate port，没有把 v154 candidate 或 Node v397-v399 review/archive 误升级为 runtime approval。

## 边界

v155 仍是只读 evidence release。`runtimeExecutionApprovalGateInputPrecheck` 的含义是“approval gate 输入仍缺 3/3，必须 fail closed”，不是“可以执行 runtime packet”。后续若要推进 Node v400 或更晚的 approval input intake，仍必须同时提供 Node-approved runtime window、correlated operator approval record 和 complete cross-project runtime execution packet，并在独立 gate 中证明 Java/mini-kv 的 owner、port、GET-only smoke、cleanup proof 和 stop-only-owned-process 规则完整成立。
