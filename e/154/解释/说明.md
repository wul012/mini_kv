# mini-kv v154 命令归档说明

本版归档对象是 `SHARDJSON` 的 mini-kv 侧 runtime execution artifact candidate。v154 先把 v153 的 blocked runtime execution artifact intake preflight 冻结为 `fixtures/release/shard-readiness-v153.json`，再在 rolling current 中声明 mini-kv 侧候选信息：候选 loopback 端口、GET-only smoke 命令、service owner 和 cleanup rules。

这些信息只是候选证据，不是 runtime approval。current evidence 仍明确保持 `acceptedRuntimeExecutionArtifactCount=0`、`missingAcceptedRuntimeExecutionArtifactCount=6`、`runtimeExecutionPacketExecutable=false`、`startsMiniKvService=false`、`runtimeProbeAllowed=false`、`liveReadAllowed=false`、`routerActivationAllowed=false`、`writeRoutingAllowed=false`、`executionAllowed=false`。归档截图由 Playwright MCP 从临时只读 HTTP 渲染页生成；截图完成后该临时 HTTP 服务已关闭。

  - 结果：current fixture 更新为 `releaseVersion=v154`、`status=mini-kv-runtime-execution-artifact-candidate-no-runtime-read-only`、`evidenceDigest=fnv1a64:edbe9c546c0d780a`；v153 frozen fixture 保留 `status=runtime-execution-artifact-intake-preflight-blocked-read-only`、`evidenceDigest=fnv1a64:ca09095f3f3e4b2c`。
  - 意义：证明 v154 current 不是手写 JSON，而是真实 CLI 命令路径生成；v153 blocked preflight 被版本化冻结为本版 baseline。

## 边界

v154 仍是只读 evidence release。`miniKvRuntimeExecutionArtifactCandidate` 的含义是“mini-kv 可以提供候选行给后续完整 runtime execution packet 评审”，不是“Node/Java/mini-kv 已经可以执行 runtime live read”。后续若要推进 Node v397 或更晚的 runtime execution packet，仍必须另行提供并批准完整六项 artifact，并由独立 gate 证明 operator approval、跨项目 packet、Java/mini-kv 端口绑定、GET-only smoke、owner confirmation 和 cleanup proof 完整成立。
