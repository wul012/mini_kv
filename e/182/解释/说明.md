# mini-kv v182 命令归档说明

v182 冻结 v181 route catalog integrity window，并把只读 Node route/catalog 兼容窗口扩展到 Node v462 test migration。它只发布版本化证据，不改变 `SHARDJSON`、fixture path、服务启动、runtime probe、active router、write routing、LOAD/RESTORE/COMPACT 或执行许可。

## 01-cmake-configure-build.png

- 命令：使用 CLion CMake/Ninja 配置并构建 `cmake-build-v182`，并在测试更新后追加 rebuild。
- 结果：配置、构建和增量构建均成功。
- 意义：确认 v182 源码与测试可编译链接。

## 02-generate-shard-readiness-fixture.png

- 命令：`@('SHARDJSON','QUIT') | .\cmake-build-v182\minikv_cli.exe`，刷新 rolling fixture。
- 结果：`releaseVersion=v182`，status 为 `node-route-catalog-test-migration-window-read-only`，窗口覆盖 Node v433-v462，count/span 为 30，digest 为 `fnv1a64:e4a40fd2e1d1d741`。
- 意义：证明 v462 test migration 已进入只读兼容窗口。

## 03-targeted-ctest.png

- 命令：定向运行 `command_tests` 和 `shard_readiness_tests`。
- 结果：2/2 通过。
- 意义：验证 SHARDJSON、冻结 v181 fixture、v462 window、30 span 和 no-execution 边界。

## 04-full-ctest.png

- 命令：运行全量 71 条 CTest。
- 结果：71/71 通过。
- 意义：确认窗口扩展未扰动存储、WAL、TCP、恢复和各类 receipt 合约。

## 05-tcp-smoke.png

- 命令：在 `127.0.0.1:6455` 启动服务端，通过客户端读取 `SHARDJSON`。
- 结果：真实 TCP 响应包含 v182、v433-v462 window、count/span 30、`readOnly=true`、`executionAllowed=false` 和 digest `fnv1a64:e4a40fd2e1d1d741`；服务端已停止，端口已释放。
- 意义：证明真实客户端路径仍为只读证据，不引入运行或写入。
