# mini-kv v147 命令归档说明

本版归档对象是 `SHARDJSON` active shard prototype prerequisite evidence。v147 的重点不是启用分片，而是把 Node 已消费的 mini-kv v146 证据冻结为 `fixtures/release/shard-readiness-v146.json`，并在 current `fixtures/release/shard-readiness.json` 中新增 `activePrototypePlan`，明确 active shard prototype 仍然只是前置计划字段。

截图使用 Playwright MCP 生成；由于 MCP 截图需要浏览器页面承载日志，本轮临时启动了本地只读 HTTP 页面服务，截图完成后已关闭。

## 截图

- `01-cmake-configure.png`
  - 命令：`cmake -S D:\C\mini-kv -B D:\C\mini-kv\cmake-build-v147 -G Ninja ...`
  - 结果：CMake 配置和生成成功。
  - 意义：证明 v147 的 active prototype prerequisite 字段和冻结 v146 fixture 可被当前工具链识别。

- `02-target-build-ctest.png`
  - 命令：构建 `minikv_shard_readiness_tests`、`minikv_command_tests`，随后定向运行 `ctest -R ^(command_tests|shard_readiness_tests)$`。
  - 结果：2/2 通过。
  - 意义：证明 current v147 fixture 等于 runtime 输出，v146 frozen fixture 保留，且 `SHARDJSON` 仍然是只读、非 WAL、拒绝多余参数的命令。

- `03-full-build-ctest.png`
  - 命令：全量 `cmake --build` 和全量 `ctest --output-on-failure`。
  - 结果：71/71 通过。
  - 意义：证明 v147 没有破坏既有 WAL、snapshot、runtime evidence、credential resolver non-participation、TCP/RESP、restore evidence 和其他回归链。

- `04-tcp-shardjson-active-prototype-plan-smoke.png`
  - 命令：临时启动 `minikv_server.exe 6417 127.0.0.1`，用真实 `minikv_client.exe` 读取 `HEALTH`、`COMMANDSJSON`、`EXPLAINJSON SHARDJSON`、`CHECKJSON SHARDJSON extra`、`SHARDJSON`、`QUIT`。
  - 结果：真实 TCP 输出包含 `releaseVersion=v147`、`status=active-prototype-prerequisite-read-only`、`historicalFallback.previousConsumedFixturePath=fixtures/release/shard-readiness-v146.json`、`activePrototypePlan.planMode=prerequisite-only`、`activePrototypePlan.activeShardPrototypeAllowed=false`、`routerActivationAllowed=false` 和 `writeRoutingAllowed=false`。服务端验证后已停止，`6417` 没有 LISTENING 残留。
  - 意义：证明 v147 证据不仅在 fixture 中存在，真实命令路径也能读到 active prototype 仍未启用的边界。

## 边界

v147 仍不创建分片目录，不安装 active router，不启动多进程，不写存储，不执行 `LOAD`、`COMPACT`、`RESTORE`、`SETNXEX`，也不让 mini-kv 成为订单或审计权威。`activePrototypePlan` 只是控制面读到后的前置计划说明，不是执行入口。
