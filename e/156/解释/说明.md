# mini-kv v156 命令归档说明

本版归档对象是 `SHARDJSON` 的 final mini-kv approval gate input。v156 先把 v155 approval-gate input precheck 冻结为 `fixtures/release/shard-readiness-v155.json`，再在 rolling current 中声明 mini-kv 本侧最终输入已经完整：loopback port、GET-only `SHARDJSON` smoke、service owner confirmation、stop-only-owned-process cleanup rules。

这不是 runtime approval，也不是完整 cross-project packet。current evidence 仍保持 `runtimeGateApprovalPresent=false`、`nodeApprovedRuntimeWindowPresent=false`、`correlatedOperatorApprovalRecordPresent=false`、`completeCrossProjectRuntimeExecutionPacketPresent=false`、`cleanupProofPresent=false`、`runtimeExecutionPacketExecutable=false`、`startsJavaService=false`、`startsMiniKvService=false`、`runtimeProbeAllowed=false`、`liveReadAllowed=false`、`routerActivationAllowed=false`、`writeRoutingAllowed=false`、`executionAllowed=false`。归档截图由 Playwright MCP 从临时只读 HTTP 渲染页生成；截图完成后该临时 HTTP 服务已关闭。

## 边界

v156 仍是只读 evidence release。`miniKvFinalApprovalGateInput` 的含义是“mini-kv 本侧输入已完整，可供 Node v401 或更晚 intake”，不是“可以执行 runtime packet”。后续若要推进真实 runtime execution，仍必须同时提供 Node-approved runtime window、correlated operator approval record 和 complete cross-project runtime execution packet，并在独立 gate 中证明 Java/mini-kv owner、port、GET-only smoke、cleanup proof 和 stop-only-owned-process 规则完整成立。
