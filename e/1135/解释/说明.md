# mini-kv v1135 命令归档说明

本轮归档对应 mini-kv v1111-v1135：`operator value supply approval profile section renderer split` 证据闭环。截图由最终命令日志渲染生成，并通过 Playwright MCP 截图保存。由于 Playwright 阻止直接打开 `file://` 页面，本轮临时启动了仅绑定 `127.0.0.1` 的静态页面服务器用于截图渲染；该服务器只服务归档页面，收尾时停止并删除临时日志。

## 01-cmake-configure.png

命令：

```powershell
$cmake -S . -B cmake-build-v1135 -G Ninja
```

结果：CMake 配置完成，生成 `cmake-build-v1135` 构建目录。日志中保留了本机已有的 Windows 路径长度警告；该警告不影响配置结果，也不是源码或测试失败。

意义：证明本版新增源文件、测试目标、命令入口和 fixture 相关改动能够进入 CMake/Ninja 构建图。

## 02-cmake-build.png

命令：

```powershell
$cmake --build cmake-build-v1135
```

结果：最终稳定构建完成；最后一次归档运行显示 `ninja: no work to do.`，说明源码、测试、文档稳定后没有遗留待编译目标。

意义：证明后续 CTest、CLI smoke、TCP smoke 使用的是本版稳定构建产物，不依赖旧 exe。

## 06-tcp-smoke.png

命令：

```text

minikv_client.exe --port 6735
SHARDROUTEVALUESUPPLYAPPROVALPROFILESECTIONJSON
QUIT
```

结果：真实 TCP server/client 通道返回新命令 JSON；release 为 `v1135`，stage 为 `route-preview-operator-value-supply-approval-profile-section-closeout-summary`，`25/25`、链完整、验证通过。验证结束后服务端已停止。

意义：证明新命令不是仅在单进程 CLI 中可见，也能通过 TCP 查询并保持同一组只读边界。由于聚合 `SHARDJSON` 已非常大，TCP smoke 只覆盖新命令；聚合 `SHARDJSON` 由 CLI smoke 和 CTest 覆盖，避免客户端行长度上限干扰判断。
