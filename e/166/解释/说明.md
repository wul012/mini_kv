# mini-kv v166 命令归档说明

本目录记录 mini-kv v166 的最终闭环验证。v166 冻结 v165 release catalog 证据，新增 `shardReadinessReleaseCatalogAudit`，证明 v161-v164 catalog 连续、无重复、digest/fixture/计数完整。该审计只用于只读消费，不打开 active router、不允许 write routing、不启动 Java 或 mini-kv runtime、不执行 LOAD/RESTORE/COMPACT/SETNXEX。

## 01-cmake-configure-build.png

命令：

```powershell
cmake -S . -B cmake-build-v166 -G Ninja
cmake --build cmake-build-v166
```

结果：配置和构建成功。`src/shard_readiness_release_catalog.cpp` 中新增的 catalog audit formatter 已进入构建图，CLI/server/client 与测试目标均生成成功。

意义：证明 v166 的 release catalog audit 是可编译实现，且没有破坏现有目标链接。

## 02-generate-shard-readiness-fixture.png

命令：

```powershell
"SHARDJSON`nQUIT`n" | .\cmake-build-v166\minikv_cli.exe
```

结果：由真实 `SHARDJSON` 输出重新生成 `fixtures/release/shard-readiness.json`。关键字段为：

- `releaseVersion=v166`
- `status=shard-readiness-release-catalog-audit-read-only`
- `evidenceDigest=fnv1a64:4bd18a01790dc5f1`
- `auditMode=shard-readiness-release-catalog-consistency-read-only`
- `expectedReleaseCount=4`
- `observedReleaseCount=4`
- `boundaryCatalogVersion=read-only-boundary-fields.v7`
- `fieldCount=381`
- `groupCount=20`
- `executionAllowed=false`
- `activeRouterInstalled=false`
- `writeRoutingAllowed=false`

意义：证明 v166 rolling fixture 来自命令输出，catalog audit 只审计版本化证据链，不改变命令、路径、路由或执行边界。

## 05-tcp-smoke.png

命令：临时启动：

```powershell
minikv_server.exe 6439 127.0.0.1
```

然后通过 client 发送：

```text
SHARDJSON
GET missing:v166
QUIT
```

结果：TCP client 读到 v166 `SHARDJSON`，`evidenceDigest=fnv1a64:4bd18a01790dc5f1`，`auditMode=shard-readiness-release-catalog-consistency-read-only`，`executionAllowed=false`，`writeRoutingAllowed=false`，`activeRouterInstalled=false`；缺失 key 返回 `(nil)`；验证结束后停止 server，并确认 `6439` 无 LISTENING 端口。

意义：证明真实 TCP 只读路径也能消费 v166 audit，但不会打开写路由、runtime probe、服务联动或执行路径。
