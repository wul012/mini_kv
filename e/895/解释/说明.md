# mini-kv v895 命令截图说明

本目录归档 v881-v895 candidate document request package closeout 批次的关键命令证据。截图使用 Playwright MCP 从本地临时 HTTP 页面生成；临时 HTTP server 仅用于截图，截图完成后已停止。

## 01-cmake-configure-build.png

对应命令：

```powershell
cmake -S . -B cmake-build-v881-v895 -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build cmake-build-v881-v895 --target minikv_cli minikv_server minikv_client minikv_candidate_request_package_tests minikv_command_tests minikv_shard_readiness_tests
```

结果：配置和目标构建通过。CMake 输出中出现既有长路径 warning，属于旧测试目标路径长度提示，不影响本轮新增短命名模块构建。

意义：证明新 `shard_preview_candidate_request_package*` 模块、命令入口、server/client 和关键测试目标可以在本机 CLion/MinGW toolchain 下构建。

## 02-focused-ctest.png

对应命令：

```powershell
ctest --test-dir cmake-build-v881-v895 -R "candidate_request_package|command_tests|shard_readiness_tests|line_editor_tests" --output-on-failure
```

结果：聚焦测试全部通过。

意义：证明 v881-v895 新链路的 stage/item/check/gate/validation、命令 catalog、`SHARDJSON` 集成和 CLI 补全都保持一致。

## 03-full-ctest.png

对应命令：

```powershell
ctest --test-dir cmake-build-v881-v895 --output-on-failure
```

结果：154 条 CTest 全部通过。

意义：证明新增 request package 证据链没有破坏旧 WAL、snapshot、restore、TCP、fixture、只读边界和历史 evidence 测试。

## 04-cli-smoke.png

对应命令：

```powershell
COMMANDS
SHARDROUTEVALUESUPPLYSIGNEDAPPROVALCAPTUREARTIFACTDRAFTTEXTPACKAGECANDIDATEDOCUMENTREQUESTPACKAGECLOSEOUTJSON
SHARDJSON
```

结果：`COMMANDS` 计数为 59；新命令输出 `candidateDocumentRequestPackageReleaseVersion=v895`、15 个 request item、15 个 acceptance check、38 个 gate、20 个 requested candidate fields；`realCandidateDocumentPresent=false`、`writeRoutingAllowed=false`、`executionAllowed=false`；`SHARDJSON` 为 v895 且包含 `shardPreviewCandidateDocumentRequestPackageCloseout`。

意义：证明新增命令可由真实 CLI 读取，且只是请求包证据，不创建/导入/评价/接受 candidate document。

## 05-tcp-smoke.png

对应命令：

```powershell
minikv_server.exe 6395 127.0.0.1 e/895/_logs/tcp-smoke.wal
minikv_client.exe 127.0.0.1 6395 3000
```

结果：TCP PING 成功；新 request package 命令和 `SHARDJSON` 关键字段可经 TCP client 读取；`writeRoutingAllowed=false`、`executionAllowed=false`；server PID 已记录并在 smoke 后停止。

意义：证明真实 TCP 路径仍能读取新只读证据，同时没有留下运行中的 mini-kv server。
