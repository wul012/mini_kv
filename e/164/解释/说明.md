# mini-kv v164 命令归档说明

本目录记录 mini-kv v164 的最终闭环验证。v164 是维护拆分版本：冻结 v163 的 `slotTablePreviewAudit` 证据，拆出 `shard_readiness_slot_preview_audit` formatter，并新增 `slotTablePreviewAuditMaintenance` 只读维护证据。它不打开 active router、不允许 write routing、不启动 Java 或 mini-kv runtime、不执行 LOAD/RESTORE/COMPACT/SETNXEX。

## 01-cmake-configure-build.png

命令：使用 CLion 捆绑 CMake/Ninja 工具链执行：

```powershell
cmake -S . -B cmake-build-v164 -G Ninja
cmake --build cmake-build-v164
```

结果：配置和构建成功。新文件 `src/shard_readiness_slot_preview_audit.cpp` 已参与链接，`minikv_cli.exe`、`minikv_server.exe`、`minikv_client.exe` 和测试目标均生成成功。

意义：证明 v164 的 formatter 拆分不是文档级变化，而是进入实际 C++ 构建图的可编译实现。

## 02-generate-shard-readiness-fixture.png

命令：

```powershell
"SHARDJSON`nQUIT`n" | .\cmake-build-v164\minikv_cli.exe
```

结果：由真实 `SHARDJSON` 输出重新生成 `fixtures/release/shard-readiness.json`。关键字段为：

- `releaseVersion=v164`
- `status=slot-table-preview-audit-maintenance-read-only`
- `evidenceDigest=fnv1a64:9935ccb959b5a3b8`
- `maintenanceMode=slot-table-preview-audit-formatter-split-contract-preserving`
- `boundaryCatalogVersion=read-only-boundary-fields.v5`
- `fieldCount=337`
- `groupCount=18`
- `readOnly=true`
- `executionAllowed=false`
- `activeRouterInstalled=false`
- `writeRoutingAllowed=false`

意义：证明当前 rolling fixture 来自命令输出，且新维护字段只表达拆分证据，不改变只读边界。

## 03-targeted-ctest.png

命令：

```powershell
ctest --test-dir cmake-build-v164 -R "^(command_tests|shard_readiness_tests)$" --output-on-failure
```

结果：定向测试 2/2 通过。

意义：`command_tests` 验证 `CommandProcessor` 输出仍与 fixture 语义一致；`shard_readiness_tests` 验证 v164 当前证据、v163 frozen fixture、维护字段、catalog v5 计数和 no router/no write/no execution 边界。


结果：全量 CTest 71/71 通过。

意义：证明 formatter 拆分没有破坏 WAL、snapshot、TCP、restore、runtime smoke、credential resolver non-participation receipts 等既有行为面。

## 05-tcp-smoke.png

命令：临时启动：

```powershell
minikv_server.exe 6437 127.0.0.1
```

然后通过 client 发送：

```text
SHARDJSON
GET missing:v164
QUIT
```

结果：TCP client 读到 v164 `SHARDJSON`，`evidenceDigest=fnv1a64:9935ccb959b5a3b8`，`executionAllowed=false`，`writeRoutingAllowed=false`，`activeRouterInstalled=false`；缺失 key 返回 `(nil)`；验证结束后停止 server，并确认 `6437` 无 LISTENING 端口。

意义：证明真实 TCP 路径也只能读取 v164 证据，不会打开写路由、runtime probe、服务联动或执行路径。
