本版冻结 v180 历史 formatter maintenance 证据，并把只读 Node route/catalog 兼容窗口扩展到 Node v460。v181 仍只是证据：不改变 `SHARDJSON` 命令、fixture path、服务启动、runtime probe、active router、write routing、LOAD/RESTORE/COMPACT 或执行许可。

- 命令：使用 CLion CMake/Ninja 配置并构建 `cmake-build-v181`，并在测试更新后追加 rebuild。
- 结果：配置、构建和增量构建均成功。
- 意义：确认 v181 C++ 源码、测试源码和 route/catalog window 更新可以编译链接。

## 02-generate-shard-readiness-fixture.png

- 命令：`@('SHARDJSON','QUIT') | .\cmake-build-v181\minikv_cli.exe`，并刷新 `fixtures/release/shard-readiness.json`。
- 结果：当前 fixture 为 `releaseVersion=v181`，status 为 `node-route-catalog-integrity-window-read-only`，window 覆盖 Node v433-v460，共 28 个版本，computed span 为 28，digest 为 `fnv1a64:58e9b51084ad91e5`。
- 意义：证明 v181 rolling fixture 已经包含 Node v460 catalog integrity 的只读兼容证据。

## 03-targeted-ctest.png

- 命令：`ctest --test-dir cmake-build-v181 -R '^(command_tests|shard_readiness_tests)$' --output-on-failure`。

- 意义：验证 `SHARDJSON` 命令、v180 frozen fixture、v460 window、28 span、source frozen digest 和 no-execution 边界。

## 04-full-ctest.png

- 命令：`ctest --test-dir cmake-build-v181 --output-on-failure`。
- 结果：全量 71 条 CTest 全部通过。

## 05-tcp-smoke.png

- 命令：在 `127.0.0.1:6454` 启动 `minikv_server.exe`，通过客户端发送 `PING`、`SHARDJSON`、`QUIT`。
