# mini-kv v165 命令归档说明

本目录记录 mini-kv v165 的最终闭环验证。v165 冻结 v164 的 slot preview audit maintenance 证据，新增 `shardReadinessReleaseCatalog`，把 v161-v164 的 shard-readiness 证据链整理成版本化 catalog。该 catalog 只用于只读消费，不打开 active router、不允许 write routing、不启动 Java 或 mini-kv runtime、不执行 LOAD/RESTORE/COMPACT/SETNXEX。

## 01-cmake-configure-build.png

命令：

```powershell
cmake -S . -B cmake-build-v165 -G Ninja
cmake --build cmake-build-v165
```

结果：配置和构建成功。新模块 `src/shard_readiness_release_catalog.cpp` 已进入 CMake 构建图，CLI/server/client 与测试目标均生成成功。

意义：证明 v165 的 release catalog 是可编译实现，不是单纯文档或 fixture 修改。

## 02-generate-shard-readiness-fixture.png

命令：

```powershell
"SHARDJSON`nQUIT`n" | .\cmake-build-v165\minikv_cli.exe
```

结果：由真实 `SHARDJSON` 输出重新生成 `fixtures/release/shard-readiness.json`。关键字段为：

- `releaseVersion=v165`
- `status=shard-readiness-release-catalog-read-only`
- `evidenceDigest=fnv1a64:5e94e2a9b438d2ad`
- `catalogMode=versioned-shard-readiness-release-catalog-read-only`
- `catalogedReleaseCount=4`
- `latestCatalogedDigest=fnv1a64:9935ccb959b5a3b8`
- `boundaryCatalogVersion=read-only-boundary-fields.v6`
- `fieldCount=357`
- `groupCount=19`
- `executionAllowed=false`
- `activeRouterInstalled=false`
- `writeRoutingAllowed=false`

意义：证明 v165 rolling fixture 来自命令输出，catalog 只是版本化只读证据，不改变命令、路径或执行边界。

## 03-targeted-ctest.png

命令：

```powershell
ctest --test-dir cmake-build-v165 -R "^(command_tests|shard_readiness_tests)$" --output-on-failure
```

结果：定向测试 2/2 通过。

意义：`command_tests` 验证 `CommandProcessor` 输出与 fixture 合同一致；`shard_readiness_tests` 验证 v165 catalog、v164 frozen fixture、v431 preserved flag、boundary catalog v6 计数和 no router/no write/no execution 字段。

## 04-full-ctest.png

命令：


结果：全量 CTest 71/71 通过。

意义：证明新增 release catalog 没有破坏 WAL、snapshot、TCP、restore、runtime smoke、credential resolver non-participation receipts 等既有行为面。

## 05-tcp-smoke.png

命令：临时启动：

```powershell
minikv_server.exe 6438 127.0.0.1
```

然后通过 client 发送：

```text
SHARDJSON
GET missing:v165
QUIT
```

结果：TCP client 读到 v165 `SHARDJSON`，`evidenceDigest=fnv1a64:5e94e2a9b438d2ad`，`catalogMode=versioned-shard-readiness-release-catalog-read-only`，`executionAllowed=false`，`writeRoutingAllowed=false`，`activeRouterInstalled=false`；缺失 key 返回 `(nil)`；验证结束后停止 server，并确认 `6438` 无 LISTENING 端口。

意义：证明真实 TCP 只读路径也能消费 v165 catalog，但不会打开写路由、runtime probe、服务联动或执行路径。
