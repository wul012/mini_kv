# mini-kv v159 命令归档说明

本轮先只读确认 Node 当前计划为 `docs/plans3/v418-post-sandbox-endpoint-credential-resolver-route-group-split-roadmap.md`。该计划是 Node route registration refactor，不新增 evidence gate，不启动 Java / mini-kv。mini-kv v159 因此冻结 v158，并新增 route-group split compatibility 只读证据。

## 01-cmake-configure-generate-fixture.png

命令：

```powershell
cmake -S . -B cmake-build-v159 -G Ninja
cmake --build cmake-build-v159 --target minikv_cli
minikv_cli.exe stdin: SHARDJSON
```

结果：CMake 配置成功，`minikv_cli.exe` 构建成功，当前 `fixtures/release/shard-readiness.json` 由真实 CLI 路径生成。生成后的 `releaseVersion=v159`，`status=node-route-group-split-compatibility-read-only`，`evidenceDigest=fnv1a64:0741c3475ee371e2`，`miniKvContractChangedForNodeRouteSplit=false`，`executionAllowed=false`。

意义：证明 current fixture 来自当前 C++ 实现；Node route-group split 不要求 mini-kv 改 `SHARDJSON` 契约或 runtime 行为。

## 02-target-build-ctest.png

命令：

```powershell
cmake --build cmake-build-v159 --target minikv_shard_readiness_tests minikv_command_tests
ctest --test-dir cmake-build-v159 -R "^(command_tests|shard_readiness_tests)$" --output-on-failure
```

结果：定向构建成功，`command_tests` 与 `shard_readiness_tests` 2/2 通过。

意义：证明 v159 route split compatibility 字段、v158 frozen fixture、no router、no write、no execution 在核心测试中被锁住。

## 03-full-build-ctest.png

意义：证明 v159 的新小模块和证据推进没有破坏 WAL、snapshot、restore、runtime evidence、credential resolver receipt、TCP/RESP 等既有测试面。

## 04-tcp-smoke-cleanup.png

命令：

```powershell
minikv_server.exe 6430 127.0.0.1
minikv_client.exe 127.0.0.1 6430
```

客户端输入：

```text
SHARDJSON
GET v159:route-split:probe
QUIT
```

结果：真实 TCP 路径读到 v159 `SHARDJSON`，包含 `nodeRouteGroupSplitCompatibility`，`miniKvContractChangedForNodeRouteSplit=false`，`nodeAddsEvidenceGate=false`，`routerActivationAllowed=false`，`writeRoutingAllowed=false`，`executionAllowed=false`。`GET v159:route-split:probe` 返回 `(nil)`。声明的 approval port `6424` 全程未监听；临时 smoke port `6430` 验证后已停止并确认不再监听。

意义：证明本版只读证据可以通过真实服务读取，但不会让 Node 路由拆分变成 mini-kv runtime approval、router、write 或 execution。
