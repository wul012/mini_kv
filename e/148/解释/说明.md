# mini-kv v148 命令归档说明

本版归档对象是 `SHARDJSON` activePrototypePlan freeze evidence。v148 的重点不是启用分片，而是把 v147 的 `activePrototypePlan` 证据冻结为 `fixtures/release/shard-readiness-v147.json`，并在 current `fixtures/release/shard-readiness.json` 中新增 `activePrototypePlanFreeze`，继续证明 frozen baseline 和 current evidence 都保持只读、no router、no write、no execution。

截图使用 Playwright MCP 生成；由于 MCP 截图需要浏览器页面承载日志，本轮临时启动了本地只读 HTTP 页面服务，截图完成后已关闭。

  - 结果：CMake 配置和生成成功。
  - 意义：证明 v148 的 frozen activePrototypePlan evidence 字段和冻结 v147 fixture 可被当前工具链识别。

  - 结果：2/2 通过。
  - 意义：证明 current v148 fixture 等于 runtime 输出，v147 frozen fixture 保留，且 `SHARDJSON` 仍然是只读、非 WAL、拒绝多余参数的命令。

  - 结果：真实 TCP 输出包含 `releaseVersion=v148`、`status=active-prototype-plan-frozen-read-only`、`historicalFallback.previousConsumedFixturePath=fixtures/release/shard-readiness-v147.json`、`activePrototypePlan.writeRoutingAllowed=false`、`activePrototypePlanFreeze.frozenReleaseVersion=v147`、`frozenRouterActivationAllowed=false`、`frozenWriteRoutingAllowed=false`、`executionAllowed=false`、`writeCommandsAllowed=false` 和 `activeRouterInstalled=false`。服务端验证后已停止，`6418` 没有 LISTENING 残留。

## 边界

v148 仍不创建分片目录，不安装 active router，不启动多进程，不写存储，不执行 `LOAD`、`COMPACT`、`RESTORE`、`SETNXEX`，也不让 mini-kv 成为订单或审计权威。`activePrototypePlanFreeze` 是给 Node/Java 后续只读消费的冻结证据说明，不是执行入口。
