# mini-kv v149 命令归档说明

本版归档对象是 `SHARDJSON` frozen evidence handoff。v149 不启用分片原型、不安装 router、不创建分片目录、不写入存储，也不执行 `LOAD`、`COMPACT`、`RESTORE` 或 `SETNXEX`。它只把 v148 当前证据冻结成 `fixtures/release/shard-readiness-v148.json`，再让滚动 current 输出新增 `consumerHandoff` 字段，告诉下游 Node/Java 只能消费版本化 fixture，并在缺失冻结证据时 fail closed。

截图使用 Playwright MCP 生成。由于 MCP 不允许直接打开 `file://` 页面，本轮临时启动了一个只读本地 HTTP 页面服务承载日志渲染页；截图完成后已关闭该服务。

## 截图

- `01-cmake-configure.png`
  - 命令：`cmake -S . -B cmake-build-v149 -G Ninja`
  - 结果：CMake 配置和生成成功。
  - 意义：证明 v149 构建树可以识别新增的冻结证据交接字段和版本化 fixture。
- `02-target-build-ctest.png`
  - 命令：构建 `minikv_shard_readiness_tests`、`minikv_command_tests`，随后运行 `ctest -R '^(command_tests|shard_readiness_tests)$'`。
  - 结果：2/2 通过。
  - 意义：证明 current v149 fixture 等于 runtime 输出，v148 frozen fixture 被保留，`consumerHandoff` 明确 no live read、no router、no write、no execution。
- `03-full-build-ctest.png`
  - 命令：全量 `cmake --build cmake-build-v149` 和全量 `ctest --output-on-failure`。
  - 结果：71/71 通过。
  - 意义：证明 v149 没有破坏既有存储、WAL、snapshot、restore、TCP/RESP、runtime evidence 和 credential resolver non-participation 证据链。
- `04-tcp-shardjson-consumer-handoff-smoke.png`
  - 命令：临时启动 `minikv_server.exe 6419 127.0.0.1`，用真实 `minikv_client.exe` 读取 `HEALTH`、`COMMANDSJSON`、`EXPLAINJSON SHARDJSON`、`CHECKJSON SHARDJSON extra`、`SHARDJSON`、`QUIT`。
  - 结果：真实 TCP 输出包含 `releaseVersion=v149`、`status=frozen-evidence-handoff-read-only`、`previousConsumedFixturePath=fixtures/release/shard-readiness-v148.json`、`consumerHandoff.handoffMode=frozen-evidence-only`、`readyForNodeConsumption=true`、`startsServices=false`、`routerActivationAllowed=false`、`writeRoutingAllowed=false`、`executionAllowed=false`、`writeCommandsAllowed=false` 和 `activeRouterInstalled=false`。服务端验证后已停止，`6419` 没有 LISTENING 残留。
  - 意义：证明 v149 不是只在 fixture 文件里成立，真实命令路径也只暴露冻结证据交接，不触发服务启动、router、write routing 或执行授权。

## 边界

v149 仍是只读 evidence release。`consumerHandoff` 是给下游消费端的验收提示，不是运行入口：下游应读取 `fixtures/release/shard-readiness-v148.json`，校验 release/status/readOnly/executionAllowed/no-router/no-write 字段，并在冻结证据缺失时 fail closed。mini-kv 不读取或修改 Node 归档，不成为 Java order authority，也不承担 audit authority。
