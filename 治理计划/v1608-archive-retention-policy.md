# mini-kv v1608 Archive Retention Policy

本文件是 mini-kv 的本地归档保留策略。它吸收 Node v2114 pointer 的风险提醒：下游项目可能已经把 mini-kv 历史归档路径、release fixture 路径和内容 digest 写进自己的治理证据里。因此，mini-kv 不能用“搬目录”“重命名”“压缩替换旧 PNG”“删除旧截图”的方式处理归档膨胀。

## 1. 当前基线

v1608 只读核对到的归档规模如下：

| 归档根 | 文件数 | 字节数 | 状态 |
|---|---:|---:|---|
| `a/` | 461 | 21,367,665 | historical frozen |
| `b/` | 45 | 3,812,703 | historical frozen |
| `c/` | 336 | 31,130,191 | historical frozen |
| `d/` | 152 | 13,506,894 | historical frozen |
| `e/` | 5153 | 1,065,416,900 | historical frozen, high-volume |
| `f/` | 25 | 979,961 | current active root |

这里的核心判断是：`e/` 的体积已经很大，但它仍然是证据链的一部分，不是临时目录。只要没有单独的 owner approval、外部备份和迁移索引，任何历史归档根都必须保留原路径。

## 2. 禁止动作

以下动作默认禁止：

- 移动、重命名、合并、重排 `a/` 到 `f/` 任意历史版本目录。
- 删除旧截图、旧解释、旧日志、旧 JSON evidence。
- 用压缩版、缩略版、重新渲染版替换已经归档的 PNG。
- 把 `e/<version>/图片/` 或 `e/<version>/解释/` 移到新的根目录。
- 改写历史 fixture 路径，尤其是 `fixtures/release/*.json`。
- 为了本地整洁而破坏 Node / Java / mini-kv 之间已经形成的绝对路径或 digest 证据。

这些禁止项不表示仓库永远不能治理磁盘体积，而是说明治理必须先有迁移方案和引用更新方案，不能在普通功能版本里顺手做。

## 3. 允许动作

以下动作可以在后续普通治理版本中执行：

- 给历史归档根补只读索引，例如统计文件数、字节数、版本范围、最大文件来源。
- 在 README 或治理计划中标记 `e/` 为 historical frozen root。
- 新版本继续写入当前 active root `f/`，直到它达到明确拥挤阈值。
- 当 `f/` 达到拥挤阈值时，创建新 sibling root，例如 `g/<version>/图片/` 与 `g/<version>/解释/说明.md`，并先更新 AGENTS / README / 治理计划。
- 对未来新版本的截图做尺寸和数量节制，例如只保留关键命令截图，不为重复 smoke 输出生成多张几乎相同的图片。
- 对未来新增证据使用更清晰的解释文档，让图片数量下降但可审计性不下降。

## 4. active root 轮转规则

后续归档根轮转遵守三个条件：

1. 先判断当前 active root 是否已经拥挤。拥挤信号包括文件数过高、单根目录超过约 1GB、版本解释难以检索、截图命名冲突增多。
2. 如果需要轮转，只新增 sibling root，不搬旧内容。比如从 `f/` 之后进入 `g/`，但 `f/` 保持原样。
3. 轮转版本必须更新 `AGENTS.md`、`README.md` 或本治理目录索引，写清楚新 root 的起始版本和旧 root 的 frozen 状态。

## 5. 截图生成约束

后续生成命令归档截图时，优先遵守“少而准”：

- 配置、构建、测试、真实 CLI/TCP smoke、关键 fixture 校验可以截图。
- 对同一命令的重复成功输出，不重复截图；解释文档可以引用同一张图。
- 如果文档版本没有 C++ 行为变化、不运行构建、不创建命令截图，应在最终回复里说明原因。
- 如果截图失败可以回退到文本渲染 PNG，但要在说明中记录回退原因。

## 6. 需要单独审批的迁移

如果未来确实要减少仓库体积，必须先写单独迁移方案，至少包含：

- 哪些外部引用会受影响，尤其是 Node 的绝对路径与 digest。
- 旧路径到新位置的 manifest。
- 迁移前完整备份位置。
- 迁移后如何验证 Node / Java / mini-kv 的证据引用仍可解释。
- 是否需要保留旧路径占位文件或跳转索引。

在没有这个方案和明确授权之前，普通版本只允许冻结、索引、轮转，不允许迁移。

一句话总结：mini-kv 的归档治理先保路径稳定，再控制未来增长；`e/` 的 1.1GB 是需要被标记和索引的历史证据，不是可以顺手清理的临时垃圾。
